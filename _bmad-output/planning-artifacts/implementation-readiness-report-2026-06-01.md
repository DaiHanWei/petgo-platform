### 3.2 UX 覆盖缺口（PRD 有、UX markdown 未描述）

最新 PRD 新增的大量 V1 功能在 UX **markdown** 中缺少行为规格（虽然效果图 HTML 已画出对应屏）：

- FR-0A~0H 登录与引导全流程（软性登录浮层、宠物状态三选、档案提示条）——UX 无对应 Flow
- FR-26 迷你主页预览卡（bottom sheet）——UX 无
- FR-29~33 兽医端工作台（独立 Tab Bar、接单、30 分钟评分门）——UX 无
- FR-23/24 点赞与两级评论、FR-28 内容详情页、FR-25 举报——UX 无
- FR-34 通知中心铃铛、FR-38 推送深链接——UX 无
- FR-27 双语（id/en）——UX「Accessibility Floor」仅以 `[ASSUMPTION] V1.1 English fallback` 提及，与 PRD「V1 即双语」冲突

> 也就是说：**UX markdown 是旧版骨架，新 PRD 的一半功能在 UX 行为文档里没有落点**。效果图 HTML 补上了画面，但缺交互/状态/文案规格。

## 4. 技术选型分析（TECH_FRAMEWORK）

### 4.1 优点
- **多云拓扑职责清晰**：Cloudflare(边缘) + 德国(后端/DB) + 阿里雅加达(媒体/CDN) + 腾讯(IM) + Google(AI/Gemini)，跨云交互被收敛到极少数点（"IM→OSS 存档复制""Gemini 签名 URL 拉 OSS 图"）。
- **模块化单体**而非过早微服务，模块边界（auth/triage/consult/content/profile/notify/moderation/vet）与 PRD FR 一一映射，未来拆分线清晰。
- **难点有兜底**：AI 假阴性靠**独立于模型的确定性安全规则层**（高危症状→强制升红）是架构必需件，判断正确且符合 PRD 对抗性评审的 critical。
- **三层媒体存储模型**（OSS 公开桶 / OSS 私密桶 / IM 托管）把隐私边界讲清楚了，并正确指出"档案不可引用 IM URL，需存档时复制到私密桶"。
