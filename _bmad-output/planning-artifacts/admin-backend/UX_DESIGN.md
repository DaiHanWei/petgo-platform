# TailTopia 管理后台 — 轻量视觉规范（Design Tokens）

> **定位**：这不是完整 UX 工作流产物，而是一份**给内部后台用的轻量视觉规范 + 设计令牌**，目的有二：① 全后台视觉统一（避免 25 个故事各写内联样式漂移）；② 带上 App 品牌识别（**淡紫色**）。
> **设计哲学**：**紫做品牌强调色，底子保持后台中性、高信息密度、桌面办公**——「基于 App 主题色」而非「等于 App」。App 是消费级俏皮调性（圆角大、留白多、动效活泼），后台要密集表格、克制留白、状态色清晰。
> **色值源真相**：App `petgo_app/lib/core/theme/colors.dart`（常量名 `mint*` 是遗留别名，实际值是 violet）。落地时以该文件为准对齐。

## 1. 颜色令牌（admin.css `:root` 变量）

### 品牌强调（取自 App，紫）
| 令牌 | 值 | 用途 |
|---|---|---|
| `--brand` | `#845EC9` | 主按钮、激活态导航、链接、选中 |
| `--brand-strong` | `#6C48AE` | 按下态、hover 深化 |
| `--brand-tint` | `#F8F2FF` | 选中行/标签柔填充、浅紫底 |
| `--brand-line` | `#DCD2F7` | 紫浅描边（强调区块） |

### 中性底（后台密集，非 App 俏皮）
| 令牌 | 值 | 用途 |
|---|---|---|
| `--bg` | `#F5F6F8` | 页面画布（中性浅灰，非 App 米白/紫底） |
| `--surface` | `#FFFFFF` | 卡片/表格/表单面 |
| `--sidebar` | `#2E2A45` | 侧栏底（App ink 深墨紫，承品牌但克制） |
| `--sidebar-active` | `#845EC9` | 侧栏激活项底/左条 |
| `--ink` | `#2E2A45` | 标题/正文主 |

## 2. 排版与密度（后台 ≠ App）
- 基础字号 **13–14px**（比 App 紧）；表格行高紧凑（~36–40px）；标题 16–18px。
- 字体沿用 `system-ui, sans-serif`（既有 admin 已用）。
- 圆角克制：卡/按钮 **6–8px**（非 App 的大圆角）；不要 App 的立体「下唇」按钮、不要吉祥物 Momo、不要活泼动效。
- 间距尺度：4 / 8 / 12 / 16 / 24（密集优先）。

## 3. 组件基线（统一到 admin.css，禁内联各写各）
- **布局**：左侧深墨紫固定侧栏（导航，激活项紫底/左紫条）+ 右侧白底内容区 + 顶栏（面包屑 + 语言切换 zh/en + 当前账号 + 退出）。沿用既有 `admin/layout.html` fragment，仅换配色到令牌。
- **表格**：斑马纹可选、表头浅灰、行 hover 浅紫 `--brand-tint`、密集；批量操作用行首复选框 + 顶部批量条（HTMX）。
- **状态徽章 `.badge`**：圆角胶囊 + 状态底/文字（见状态色表），用于兽医资质 6 态、账号状态、工单状态、举报类型。
- **按钮**：主 `.btn-primary`（紫底白字）、次 `.btn`（白底紫边）、危险 `.btn-danger`（红，用于封禁/删除/下架，必带二次确认）。
- **筛选条 `.filter-bar`**：一行内的下拉/搜索框，提交走 HTMX 局部刷新结果区。
- **空态/错误**：内联错误条（红底）+ 友好空态文案（i18n），非堆栈。
- **确认弹层**：危险操作（封禁/删除/下架）统一 `fragments/confirm-dialog.html`，必填原因区。

## 4. 双语（i18n）
- 所有可见文案走 `#{admin.*}` 键（见架构 §Patterns）；语言切换在顶栏，Cookie 记忆；zh-CN / en 两键集一一对应。

## 5. 不做（与 App 对齐姿态相反，明确收口）
- ❌ 移动端响应式（桌面 only）
- ❌ App 的吉祥物 Momo / 立体按钮 / 活泼动效 / 大圆角米白底
- ❌ 复杂主题切换（仅亮色；语言切换有，主题不做）

## 6. 落地方式
本规范落到**单一共享 `static/admin/admin.css`**（`:root` 令牌 + 组件类），既有 `templates/admin/*.html` 去内联样式改引用类；新页面一律用这些类与令牌。归入 **Epic 1 Story 1.6**（i18n 基建 + admin 设计令牌 + 既有模板统一）。
