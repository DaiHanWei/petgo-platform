# TailTopia · Core Pages — UX 行为清单

> 把**交互 / 流程 / 状态 / 时序规则 / 校验 / 门控**从视觉中单独抽出。
> 视觉（配色/间距/组件外观）见 `core-pages-reference-解读.md`；本文只讲「怎么动」。
> 来源：`preview-core-pages.html` 内的 onclick 跳转、状态标记、内嵌脚本与文案逻辑。
> 配套：拆分后的单页在 `pages/`（`pages/index.html` 为索引）。

## 一、全局 UX 模式

| 模式 | 行为 |
|---|---|
| **底部 TabBar** | 5 项 + 中间凸起「+」发布。激活态 Pop Art 错位（红下层+紫上层）；「+」无错位。`data-notab` 页（引导/弹窗/全屏态）不显示 TabBar。 |
| **登录门控** | 游客可浏览 Feed/详情；触碰核心功能 → 弹 P-04 登录引导（说好处，不说「不支持」）。登录后**回跳**触发功能的来源页。 |
| **Bottom Sheet** | 登录引导、通知权限、存档确认、里程碑详情、兽医状态切换均为底部抽屉，slideUp 进入，可下滑/点遮罩关闭。 |
| **空态 / 错误态** | Feed、通知、时间线各有空态（图标+引导 CTA）；Feed/网络有错误态（重试 + error code）。 |
| **危险操作** | 注销、删档、登出走红色态 + 二次确认（删账还需键入邮箱）。 |
| **强制阅读** | 红色分诊结果按钮 5 秒倒计时后才可点。 |

## 二、认证 & 引导流（UX）

```
P-01 Splash（自动）
  └→ P-02 访客 Feed
       ├ 点 AppBar「Masuk」───────┐
       ├ 点需登录功能 → P-04 登录引导弹窗
       └ 下滑 → P-03 软性登录浮层（Daftar Gratis / Masuk / Lanjut lihat dulu）
                                    ↓
                            P-05 Google 登录
                                    ↓ (OAuth 成功)
              ┌── 老用户 → 回上次所在页
              └── 新用户 → P-06 昵称确认（1/2）
                              ↓ Lanjut
                          P-07 宠物状态选择（2/2）
                ┌── 有宠物 → P-08 建档引导 → P-30 建档 → P-31 成功
                └── 没宠物 → 直接进 P-10 首页
              （流程中适时 P-09 通知权限请求）
```

**关键交互/规则**
- P-02 访客与 P-10 登录态内容相同；差别：访客 AppBar 右侧是「Masuk」按钮，登录态是铃铛。
- P-06 昵称：预填 Google 名，可编辑，**上限 30 字**，实时计数；提示「可随时在设置改」。
- P-07：二选一单选（有宠物 / 没有但喜欢动物），决定后续分流。
- P-09 通知权限：「Aktifkan」请求系统权限 /「Tidak」跳过；被拒后兜底见 6-4（我的页「去设置」引导）。

## 三、AI 分诊流（UX）

```
P-17 问诊 Tab ──「Mulai Diagnosa」──→ P-18 症状上传
                                          ↓「Analisis Sekarang」
                              AI 处理（约 15s，含等待/超时/降级态）
                                          ↓
            ┌─────────────┬─────────────┬─────────────┐
        🟢 P-19b 绿       🟡 P-19 黄      🔴 P-19c 红
        居家护理          建议24h内咨询    紧急半屏强提醒
        无紧急CTA         + 条件倒计时     + 5秒锁 + 单「我已知晓」
            │                 │
            │            「立即咨询兽医」→ P-21 等待匹配
            └─「仍想咨询医生」→ P-21
```

**关键交互/规则**
- P-18：症状描述 **≤500 字**、照片 **≤3 张**，可逐张删除；提交前需见**免责声明前置位**。
- **三态分级**由后端安全规则层决定，前端只渲染：
  - 🟢 绿：仅居家观察建议，**不提供紧急/导航 CTA**。
  - 🟡 黄：症状摘要 + AI 建议 + **条件倒计时协议块**（观察指标 / 时间窗口 / 触发条件三要素）。
  - 🔴 红：全屏红 + 呼吸动效 + 三步急救；按钮 `Saya Mengerti (5..0)` **倒计时锁定**，关闭后**零兽医 CTA、零导航**（已去导航化，F3·2026-06-08）；无障碍 ASSERTIVE 打断式播报。

## 四、兽医问诊流（UX）

**用户侧**
```
P-17 / P-20 入口 ──「Mulai Konsultasi」──→ P-21 等待匹配
   · 1 分钟倒计时；可「Batalkan permintaan」
   · 超时弹层：继续等待 / 先用 AI（独立态）
   · 无在线兽医 → 软引导「暂无兽医在线」
                          ↓ 匹配成功
                     P-22 图文对话
   · 摘要条可展开；✓✓ 已读；对方 typing 三点
   · 「Akhiri」结束 → P-23 评分 → P-25 存档确认
   · PENDING_CLOSE：结束后 30 分钟内仍可发消息（带状态标签）
   · 异常终态：会话「已中断」+「重新发起」
```
- **P-23 评分门**：1–5 星**必填** + ≤100 字**选填**；可「Lewati」跳过；未评分会补弹。
- **P-25 存档**：明确「存进宠物档案、私人、不分享给任何人」；可「保存」/「删除不保存」。

**兽医侧**
```
V-01b 兽医登录（邮箱+密码，无 Google；被封禁登录显示「账号已停用」）
   ↓
V-01/V-02 队列（按 红/黄/绿 优先级）
   ↓「Lihat Detail」
V-03 案例详情（接单前预览 + 接单倒计时）──「Ambil Kasus」──→ V-05 兽医对话
   · 工具栏：建议模板（可一键发送）/ 药物列表 / 病历 / 紧急
   ↓ 结束
V-08 最终诊断表单（诊断/建议/是否需药/药物/复查时间/恶化征兆/紧急时限）
   ↓
V-06 历史（可按 评分 / 分诊类型 筛选）
V-07 我的 + V-ST 状态切换（Online/Sibuk/Offline，影响是否接新单）
```

## 五、社区内容流（UX）

```
P-10 Feed ──点卡片──→ P-12 内容详情
   · 筛选 chips：Semua/Kucing/Anjing/Lainnya（宠物状态硬过滤）
   · 长按卡片 → context menu（举报/分享）
```
- **P-12 详情**：多图左右滑（角标 x/y）；❤️ 点赞（**仅详情页有，Feed 卡无**）；两级评论 + 二级回复；底部评论输入。
- **「···」菜单按身份分支**：自己 → 删除（二次确认，**无编辑**）；他人 → 举报（选类型 sheet + toast）；未登录举报 → 触发登录。
- **P-38 发布**：类型 chips（日常/Tips/快乐时刻）+ **宠物选择必填** + 图 ≤9 + 文 ≤1000 + 事件日期；发布按钮**满足必填才启用**。
  ```
  发布 → P-39b 审核中（AI 审核约5s）
            ├ 通过 → P-39 发布成功
            └ 不通过 → P-39c 被拒（列原因 +「修正内容」重提）
  ```
- **他人迷你主页卡**（3-8）：点他人头像 → bottom sheet（头像+昵称+发布数+「筹备中」措辞）。〔原型缺，见补充清单〕

## 六、成长档案 / 里程碑流（UX）

```
P-27 档案（护照）
  · Timeline ↔ Kalender(P-28) 切换
  · 健康事件条目：🏥 + 日期 + 评级 + 症状摘要
  · 分享 FAB → 分享 sheet（WhatsApp/IG/FB/复制链接）
  · 编辑入口 → P-32（种类、生日锁定不可改；底部删档危险区）
  · 对外 → P-36 名片 H5（公开摘要 + 引导下载）

P-33 里程碑列表（5/30，L/M/S 三级）
  ├ 点徽章 → P-33b 详情抽屉
  │    · 系统类：说明+条件
  │    · 打卡类：「去发布」(→P-38 预选日历，回填自动完成) / 「已打卡」(选已有记录关联，已关联置灰)
  ├ P-34 徽章图鉴（30 枚，已解锁彩色/锁定灰化虚线）
  └ 达成 → P-35 三级庆祝动效（S 半屏~1.5s / M 全屏~3s / L 开宝箱~4s）
            └ L 级 → 推送(P-13) + 分享卡(P-36)
```

**完成事件**由后端订阅既有事件（建档/发布/分享/问诊等）自动触发，前端被动渲染解锁。

## 七、通知 / 推送 / 设置（UX）

- **P-13 通知中心**：铃铛入口（首页右上，未读红点角标）；按 今天/昨天 分组；类型：兽医回复 / 点赞 / 评论 / 里程碑 / 生日提醒；点条目深链到对应页。空态 P-13b。
- **深链**：推送点击按路由表落地到 P-22 / P-12+评论锚点 / P-38 / P-27 / 里程碑等；「正在查看该页」时抑制对应推送。
- **6-5 版本更新**：冷启动检测 → 推荐更新（可关）/ 强制更新（不可关，挡操作）+ 跳商店。〔原型缺，见补充清单〕
- **P-41 设置**：账户（编辑资料 / 通知开关 / 语言）·外观（深色模式开关）·隐私（档案公开开关 / 隐私政策 / 条款）·危险区（登出 / 删账）。
- **P-43 注销**：列永久删除项 + **30 天冷静期**（期内重新登录可撤销）+ **须键入邮箱**才激活删除按钮。
- **P-40 我的**：用户信息 + 宠物 mini 卡（→P-27）+ 我的发帖网格；编辑资料抽屉（**email 只读**）。

## 八、时序 / 校验 / 门控 速查

| 项 | 规则 |
|---|---|
| 昵称长度 | ≤ 30 字，实时计数 |
| 症状描述 | ≤ 500 字；照片 ≤ 3 |
| 发布 | 宠物必选；图 ≤ 9；文 ≤ 1000；满足才启用发布键 |
| 评分 | 1–5 星必填，评语 ≤ 100 字选填，可跳过+补弹 |
| 匹配等待 | 1 分钟倒计时，超时给「继续等待/先用 AI」 |
| 会话结束 | PENDING_CLOSE 30 分钟内可继续发消息 |
| 红色分诊 | 按钮 5 秒锁定才可点；关闭后零 CTA/导航 |
| 内容审核 | 发布后 AI 审核约 5s，分 通过 / 被拒 |
| 删账 | 30 天冷静期 + 键入邮箱二次确认 |
| 内容 | 可删不可编辑；点赞仅详情页 |
| 档案 | 单账号单宠物；种类/生日创建后锁定 |
