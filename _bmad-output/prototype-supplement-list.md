# 原型补充清单（交设计侧）

> 基于 `prototype-vs-story-coverage.md` 的核对结论。原型 `preview-core-pages.html`（V1.0.0）已覆盖绝大多数页面，本清单列出**需补画的页面**。
> 风格基线：薄荷绿/紫 `#845EC9` + Pop Art `#F0425A`，Poppins，印尼语文案，浅色态；分诊三色 🟢/🟡/🔴。新页请沿用现有 component（卡片圆角 12px、Bottom Sheet slideUp、ibtn 等）。
> 优先级：**P0 = 必补**（story 整页缺失）· **P1 = 建议补**（独立子态/弹层，影响验收完整性）· **P2 = 可选**（元素级，能在既有页内增补）。

---

## P0 · 必补（原型整页缺失）

### 1. 他人迷你主页预览卡 〔Story 3-8〕
- **形态**：点 Feed/详情页里他人头像或昵称 → 弹出 bottom sheet（非整页）。
- **必含元素**：
  - 用户头像 + 昵称
  - 「TA 发布了 X 条内容」统计行
  - 「主页筹备中」类措辞（V1 不做完整他人主页，用克制措辞说明，不写「功能不支持」）
  - 关闭/下滑收起
- **态**：正常 / 该用户已注销或不可见（占位措辞）。
- **备注**：措辞口吻参照 P-04 登录引导（说状态、不说「不支持」）。

### 2. App 版本更新提醒弹窗 〔Story 6-5〕
- **形态**：冷启动检测后的通用 Dialog（叠加在任意页上）。
- **必含 2 态**：
  - **推荐更新**：标题 + 版本说明 + 「立即更新」(跳商店) + 「稍后」可关闭
  - **强制更新**：同上但**无关闭**，仅「立即更新」，挡住后续操作
- **元素**：跳商店 CTA、版本号/更新内容文案位。
- **备注**：纯弹窗，无需独立路由页。

## P1 · 建议补（独立子态/弹层，按口径属灰区，影响验收完整性）

### 3. 排队超时弹层 〔Story 5-3〕
- 在 P-21 等待匹配页之上，**1 分钟超时后**弹出：
  - 文案：未匹配到在线兽医
  - 两个出口：「继续等待」/「先用 AI 分诊」
- 另需 **「暂无兽医在线」软引导态**（发起咨询时即检测离线）。

### 4. 会话异常终态 〔Story 5-7〕
- **用户侧（P-22 变体）**：会话「已中断」终态标签 + 「重新发起」按钮。
- **兽医侧（V-01b 变体）**：被封禁踢回登录页时显示「账号已停用」提示。

### 5. 健康事件时间线条目样式 〔Story 2-5〕
- 在 P-27/P-28 时间线里，问诊记录条目的专属样式：
  - 🏥 标识 + 日期 + AI 评级徽章（🟢/🟡/🔴）+ 症状摘要一行
  - 与「快乐时刻」条目视觉区分（参照现有 P-28 红色咨询条）。
