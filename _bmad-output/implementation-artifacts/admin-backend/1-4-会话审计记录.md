# Story 1.4: 会话审计记录

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

> **范围域**：管理后台 V1.0.0 · Epic 1（后台地基）· Story 1.4。**建立在已完成的 Story 1.1/1.2 之上**（admin_accounts + 服务端会话 + 紧急账密 formLogin + Lark OAuth 回调 + adminFilterChain + AdminSessionGuardFilter）。
> **产物归属**：仅后端 `petgo-backend`（**无 Flutter 侧**）。brownfield 扩 `com.tailtopia.admin`。
> **与 1.3 区分**：1.3 = **操作**审计（写操作哈希链）；本故事 = **会话**审计（登录/登出/过期，仅超管可见）。两表两入口，互不复用。

## Story

As a **超级管理员**,
I want **查看所有后台认证会话的审计记录（含紧急账密使用）**,
so that **能发现异常登录、复盘谁在何时何地从哪个 IP 进出后台（满足 AB-0B 合规留痕）**。

## Acceptance Criteria

> 验证层：**L0** 编译/单测/MockMvc 切片（mock repo，无需 DB）· **L1** 集成（Docker postgres+redis，验迁移/写入/门控真跑）· **L2** 外部凭证（本故事无 L2）。

1. **AC1（建表，L0 编译 + L1 迁移）**：新增 Flyway `V33__init_admin_session_logs.sql` 建 `admin_session_logs`（`id` / `account_id`(Long，FK 语义指向 admin_accounts) / `login_at`(timestamptz UTC) / `logout_at`(timestamptz UTC, nullable) / `ip`(varchar) / `created_at`）；JPA 实体 `@Column` 与列精确匹配（`ddl-auto=validate` 启动不报错）。

2. **AC2（登录写入，L1）**：任一后台认证会话**登录成功**即写一条 `admin_session_logs`（`account_id` + `login_at`=now + `ip`=请求源 IP，`logout_at`=null）——**紧急账密（formLogin）与 Lark OAuth 回调两条路径都写**（分别挂在 1.1 的 formLogin successHandler 与 1.2 的 `AdminLarkLoginController` 建会话处）；会话与该 session log 行建立关联（如 session 属性存 sessionLogId）以便登出/过期回填。

3. **AC3（登出/过期回填，L1）**：会话**登出**（`/admin/logout` 成功）或**因停用/移出白名单被守卫失效**（`AdminSessionGuardFilter` invalidate 路径）或**闲置 8h 过期**时，回填对应行 `logout_at`=now；无法精确关联到行时（如纯超时由容器回收）至少保证「登出」与「守卫失效」两类可回填，超时由 `HttpSessionListener#sessionDestroyed` 兜底回填。

4. **AC4（只可追加不可删，L1，A6/NFR4）**：`admin_session_logs` **不可经任何 UI 删除**；`logout_at` 回填是对既有行的合法补全（非篡改 append-only 语义，允许一次性 null→时间戳）；仓库接口**不暴露** `delete*`；DB 层禁 DELETE（触发器阻断 DELETE；UPDATE 仅允许 `logout_at`/`updated_at`——或简化为不建 UPDATE 限制但仓库窄接口仅提供回填方法）。

5. **AC5（仅超管可见，L0 门控 + L1）**：`/admin/session-logs` 会话审计页**仅 `SUPER_ADMIN` 可见**（`@PreAuthorize("hasRole('SUPER_ADMIN')")`）；普通后台账号（STAFF）访问 → 403 越权（前端不渲染入口仅体验，服务端方法级权威）。

6. **AC6（查询展示，L1）**：会话审计页列出会话记录（账号身份/`login_at`/`logout_at`(或「进行中」)/`ip`），按 `login_at` 倒序分页；可选按账号/日期范围筛选（HTMX 片段）。**展示 IP 不算敏感泄漏**（运营审计正当用途），但页面与日志仍不外泄令牌/密码。

7. **AC7（回归绿，L0/L1）**：`mvn -B compile` + L0 全绿；本地 L1 全量回归绿；既有登录/登出/会话守卫（1.1/1.2）行为不回归（写 session log 是叠加副作用，不改变认证结果）。

## Dev Notes

### 架构约束（必须遵守）
- 后端 = Spring Boot 4 / Java 21 / Maven；包 `com.tailtopia.admin.audit.{web,service,domain,repository}`（会话日志与操作日志同属 audit 子域）。[Source: admin-backend/architecture.md#Project Structure → admin/audit{AdminSessionLog}]
- 后台 SSR MVC + HTMX，路由 `/admin/**` 返视图/片段不返 JSON；CSRF 开。[Source: architecture.md#API & Communication Patterns]
- **会话日志 append-only、永久、仅超管可读**（A6 / NFR4）：应用无删除权 + DB 禁 DELETE；`logout_at` 回填是合法补全。[Source: architecture.md#Data Architecture（admin_session_logs）/ Authentication & Security A6]
- 时间 `timestamptz` UTC；新表 snake_case 复数 `admin_` 前缀。[Source: architecture.md#Naming Patterns]
- 授权方法级权威（A5）：`@PreAuthorize("hasRole('SUPER_ADMIN')")`；前端隐藏非安全边界。[Source: architecture.md#Process Patterns / Authentication & Security A5]

### 既有代码基线（READ，1.1/1.2 产出）
- `SecurityConfig.adminFilterChain`：formLogin `successHandler`（紧急账密，AC2 写 login）、`logout().logoutSuccessUrl(...)`（AC3 回填，需改为/补 `logoutSuccessHandler`）。⚠️ 文件含法律页 WIP，提交时仅纳入 admin 增量。[Source: SecurityConfig.java:77-96]
- `AdminLarkLoginController.callback`：Lark 建会话成功处（`securityContextRepository.saveContext` 后），AC2 在此写 login + 存 sessionLogId。[Source: AdminLarkLoginController.java:83-92]
- `AdminSessionGuardFilter`：停用/移出白名单 → `session.invalidate()` 前是 AC3 回填点。[Source: AdminSessionGuardFilter.java:48-56]
- `AdminUserDetails.getAdminAccountId()`：写 session log 的 `account_id` 源。[Source: AdminUserDetails.java:40-42]
- 1.3（同批，先行）建立 `admin/audit` 子域 + append-only 触发器范式 + 窄仓库接口范式——本故事**复用同一范式**（DB 触发器阻断 DELETE、窄接口）；若 1.3 未先合并，本故事自带触发器写法亦可。
- 当前 Flyway：1.3 占 V32；本故事 **V33**。[来源：`db/migration/` + 1.3 分配]

### 关键边界 / 防坑
- **sessionLogId 关联**：登录写行后把行 id 存 `HttpSession`（如 `SESSION_LOG_ID`），登出/守卫失效/超时都靠它回填——这是把「容器会话生命周期」桥到「审计行」的关键；漏存则 logout_at 永远为 null。
- **超时回填靠 `HttpSessionListener`**：闲置 8h 过期由容器销毁 session，无请求触发 → 必须用 `sessionDestroyed` 监听器兜底（在监听器内取 SecurityContext/属性里的 sessionLogId 回填）。注意监听器中无 Spring 事务上下文，需经 service 自管事务。
- **源 IP 与 CF 前置**：后台经 `admin.tailtopia.id` + Cloudflare（架构 F-B/A4），`getRemoteAddr()` 会是 CF 回源 IP → 优先 `CF-Connecting-IP`，回退 `X-Forwarded-For` 首段，再回退 `getRemoteAddr`；统一工具方法，勿各处自取。
- **不改认证结果**：写 session log 是叠加副作用，successHandler/回调原有跳转（`/admin/dashboard`）与失败处理不变；写失败不应阻断登录（捕获并告警/降级，避免审计故障锁死后台）。
- **HTMX 片段 vs 整页**：依 `HX-Request` 区分。

### 范围边界（不做 / 避免前向依赖）
- **本故事不做**：操作审计（→ 1.3）；STAFF 权限装载/账号管理 UI（→ 1.5，本故事门控仅 `hasRole('SUPER_ADMIN')` 无需 permission_code，故无前向依赖）；双语外化（→ 1.6，新页文案先中文）。
- 本故事**只**交付：会话日志表/实体（V33）+ 两登录路径写入 + 三类登出回填 + 仅超管查询页。
