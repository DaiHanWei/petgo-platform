# Story 2.4: 编辑兽医账号与重置密码

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

> **范围域**：管理后台 V1.0.0 · Epic 2 · Story 2.4（AB-2C 编辑 + AB-2D 重置密码）。
> **产物归属**：仅后端 `petgo-backend`（**无 Flutter 侧**）。SSR + HTMX，`/admin/vets`，不返 JSON。
> **brownfield 关键**：`POST /admin/vets/{id}/password` 重置 + `AdminVetService.resetPassword` **已存在**（Story 5.1）。**编辑账号（显示名/邮箱/手机号）是新增能力**；本故事补编辑 + 给重置补审计/一次性显示硬化。

## Story

As a **运营成员**,
I want **编辑兽医的显示名/登录邮箱/联系手机号，并能重置其密码**,
so that **维护账号信息且不中断兽医进行中的会话，重置凭证安全交付**。

## Acceptance Criteria

> 验证层：**L0** MockMvc 切片 + 单测 · **L1** 集成（Docker postgres+redis，验编辑落库/不中断会话/审计）。

1. **AC1（编辑字段，L0/L1）**：可编辑 **显示名 / 登录邮箱 / 联系手机号**；邮箱格式校验 + 唯一校验（改成已被他人占用 → 拒绝「该登录邮箱已被占用」）；保存生效。
2. **AC2（不中断进行中会话，L1）**：编辑账号资料**不触碰会话状态**——有进行中（IN_PROGRESS/PENDING_CLOSE）会话的兽医被编辑后，其会话保持原状（不调用任何中断逻辑）。
3. **AC3（编辑写审计，L1）**：编辑成功同事务调 `AdminAuditService.record(actorAccountId, "VET_UPDATED", "VET_ACCOUNT", vetId, summary)`（summary 含变更字段，**手机号脱敏/不落明文**）。
4. **AC4（重置密码一次性显示，L1）**：重置生成**新临时密码**（系统生成或运营输入，二选一见防坑），界面**仅显示一次**；**绝不写入审计 summary / 应用日志 / 列表**；旧凭证立即失效（既有 `resetPassword` 重算 BCrypt）。
5. **AC5（重置写审计但不含密码，L1）**：重置成功调 `AdminAuditService.record(actorAccountId, "VET_PASSWORD_RESET", "VET_ACCOUNT", vetId, summary)`，summary 仅记「已重置密码」之类**不含任何密码字符**。
6. **AC6（门控，L0/L1）**：编辑端点 `@PreAuthorize("hasAuthority('vet.create')")`（账号资料管理能力）；重置端点 `@PreAuthorize("hasAuthority('vet.reset_password')")`。
7. **AC7（回归绿，L0/L1）**：既有列表/开户/封禁不回归；`mvn -B compile` + admin 单测全绿；本地 L1 全量回归绿。

## Dev Notes

### 架构约束（必须遵守）
- 写操作强制审计（Story 1.3 `AdminAuditService.record`）；`VET_UPDATED`/`VET_PASSWORD_RESET` UPPER_SNAKE 过去式。**一次性敏感值（重置密码）界面只显示一次，绝不入审计 summary/日志**。[Source: architecture.md#Format Patterns「一次性敏感值」/ Communication Patterns]
- 授权：编辑 `vet.create`、重置 `vet.reset_password`（附录 B，Story 1.5 基础）。[Source: PRD 附录 B]
- 编辑不中断进行中会话（PRD AB-2C 明确）。[Source: PRD#AB-2C]

### 既有代码基线（READ 过，brownfield）
- **现状**：`AdminWebController#resetVetPassword(id, newPassword)`（`POST /{id}/password` → `AdminVetService.resetPassword` → `VetAccountService.resetPassword`：≥8 位校验 + 重算 BCrypt + save，**无审计、无门控、新密码由运营在表单明文输入未做一次性回显硬化**）；**无任何编辑账号资料的端点/方法**；`VetAccountService` 有 `getById`、`setStatus`（封禁用，本故事不碰）。
- **本故事改什么**：新增编辑（form/service/controller/模板）；给重置补 `@PreAuthorize` + 审计 + 一次性回显。
- **不可破坏**：① `resetPassword` 既有 ≥8 位校验 + BCrypt 重算逻辑保留；② 封禁 `setStatus`/中断逻辑**绝不**被编辑路径触达（AC2 核心）；③ 列表/开户端点不动；④ `username`=登录邮箱 口径沿用 2.3（不加 email 列、不改名）。

### 关键边界 / 防坑
- **编辑邮箱唯一校验须排除自身**：`existsByUsername(newEmail) && !newEmail.equals(current.username)` 才算冲突；否则改其它字段时会误判自占用。
- **不中断会话是硬约束**：编辑路径**禁止**调用 `ConsultInterruptService`/`presence.goOffline`/`setStatus`。测试须断言编辑后会话状态不变（mock 或 L1 真跑一个 IN_PROGRESS 会话）。
- **重置密码的「系统生成 vs 运营输入」**：既有是运营在输入框填新密码。AB-2D 文案是「生成新临时密码，界面仅显示一次」。两种实现皆可：(a) 保留运营输入（运营本就知道，回显=确认）；(b) 后端生成随机临时密码并一次性回显。**推荐 (b) 更贴 PRD「生成」语义**：后端生成 → 一次性回显 → 不入库明文/不入审计。dev 二选一并在 Completion 记录；无论哪种，**审计/日志零密码字符**。
- **HTMX/redirect**：编辑保存后可 redirect `/admin/vets` 或 HTMX 局部刷新该行；重置回显新密码须在响应内（不可 redirect 丢失一次性值）——重置走整页/片段回显，不 redirect。

### Flyway
- 本故事**无新迁移**（编辑复用 2.3 的 `contact_phone`(V38) + 既有列）。如确需新列从 **V38 之后**顺延并说明（不预期）。

### 范围边界（不做）
- 不做封禁/解封（→ 2.5）、资质（→ 2.7）、双语外化（→ 1.6，新文案先中文）、用户侧任何展示。
- 本故事**只**交付：编辑（显示名/邮箱/手机号，不中断会话）+ 重置密码硬化（一次性 + 审计无密码）+ 双端点门控。

### 测试标准
- L0：`AdminWebControllerTest`（mock service/audit）：编辑成功（`VET_UPDATED` 审计 + 不调中断 service）、邮箱占用拒绝、缺 `vet.create`/`vet.reset_password` 403、重置成功（一次性回显 + `VET_PASSWORD_RESET` 审计 summary 无密码）。`VetAccountService.updateProfile` 单测（唯一校验排除自身 / 不改 status）。
- L1（本地 docker）：编辑落库；有 IN_PROGRESS 会话的兽医被编辑后会话不变（真跑）；重置后旧密码失效。云端只 L0。
