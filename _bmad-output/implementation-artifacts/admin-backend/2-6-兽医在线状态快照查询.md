# Story 2.6: 兽医在线状态快照查询

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

> **范围域**：管理后台 V1.0.0 · Epic 2 · Story 2.6（AB-2F）。**纯只读**，无写操作、无审计。
> **产物归属**：仅后端 `petgo-backend`（**无 Flutter 侧**）。SSR + HTMX，`/admin/vets/online`，不返 JSON。
> **brownfield 关键**：兽医 Redis 在线态机制 **已存在**（Story 5.2 `VetPresenceService`）。本故事**只新增一个只读快照视图**，读既有 presence，不改在线态写入逻辑。

## Story

As a **运营成员**,
I want **手动刷新查看各兽医的在线/离线快照，并知道这是上次刷新时刻的快照**,
so that **判断当前接诊能力，且明确数据非实时**。

## Acceptance Criteria

> 验证层：**L0** MockMvc 切片（mock presence/vet service）· **L1** 集成（Docker postgres+redis，验真实读 Redis 在线态）。

1. **AC1（只读快照视图，L1）**：`/admin/vets/online` 列出兽医及其在线态（读既有 `VetPresenceService.statusOf` → ONLINE/BUSY/OFFLINE）；页面**不自动刷新**（无轮询/无定时刷新脚本）。
2. **AC2（手动刷新，L1，HTMX）**：提供「刷新」按钮，点击重新读取 Redis 在线态并刷新快照（`hx-get` 局部刷新结果区或整页 GET 重载均可）。
3. **AC3（最后查询时间，L0/L1）**：页面顶部显示「最后查询时间：HH:MM:SS」（取本次请求渲染时刻，按运营时区展示，标注时区），提示数据为快照。
4. **AC4（无写操作 + 门控，L0/L1）**：本视图**无任何写操作**（不改在线态、不改账号）；端点 `@PreAuthorize("hasAuthority('vet.view')")` 门控；**不写审计**（只读）。
5. **AC5（回归绿，L0/L1）**：既有 `VetPresenceService`、兽医上线/心跳/接单置忙逻辑不回归；`mvn -B compile` + 单测全绿；本地 L1 全量回归绿。

## Dev Notes

### 架构约束（必须遵守）
- 在线快照**读既有 Redis 兽医在线态**（AB-2F 只读不写），经 `VetPresenceService` 接口，**禁跨 repo / 禁直连 Redis 绕过 service**。[Source: architecture.md#Communication Patterns「会话/在线态」]
- 路由 `/admin/vets/online`，HTMX 片段返 HTML、不返 JSON；按运营时区在视图层格式化时间（存储/逻辑仍 UTC）。[Source: architecture.md#Format Patterns「时间显示」]
- 只读 → **不写审计**（审计仅写操作）；`@PreAuthorize("hasAuthority('vet.view')")`。[Source: architecture.md#Communication Patterns（审计仅写操作）；PRD 附录 B]

### 既有代码基线（READ 过，brownfield）
- **现状**：`VetPresenceService`（Redis ZSET `vet:online` + SET `vet:busy`，TTL 3min 惰性清理）：`statusOf(vetId)`→`VetPresenceStatus{ONLINE,BUSY,OFFLINE}`、`isOnline`、`onlineVetIds`、`anyOnline`；写入由兽医侧 `VetPresenceController`（goOnline/heartbeat/logout）+ 接单 `goBusy`/`goAvailable` 维护。`AdminVetService` 已注入 `VetPresenceService`（5.1 封禁用 goOffline）。
- **本故事改什么**：新增只读 `onlineSnapshot()` + `/admin/vets/online` 页 + 模板。
- **不可破坏**：① **绝不**在 admin 只读路径调用任何 presence 写方法（goOnline/goOffline/goBusy/heartbeat/pruneStale 之外的写）——`statusOf` 内部仅读 score/member，安全；② 既有兽医在线态写入与 TTL 逻辑不动；③ 2.2 列表的在线列与本专页共用 `statusOf`（口径一致，勿各写一份判定）。

### 关键边界 / 防坑
- **快照≠实时**：本页**禁加自动轮询/SSE/定时刷新**（PRD 明确「页面不自动刷新，手动刷新」）；刷新=用户点按钮触发一次 GET。
- **`statusOf` 含惰性清理副作用？**：`statusOf`→`isOnline`→只读 `score`，**不触发 pruneStale**（pruneStale 在 `anyOnline`/`onlineVetIds` 内）。逐行 `statusOf` 不写 Redis，符合「只读」。如需顺带清幽灵在线，**勿**在 admin 页触发写清理（保持纯读）；幽灵态由兽医侧 TTL 自然过期，admin 只如实反映。
- **BUSY 展示**：AB-2F 是在线/离线二态视图，但既有有 BUSY 子态——可显示「在线（忙碌）」或归入「在线」；与 2.2 列表口径统一（2.2 BUSY 归「在线」筛选）。
- **最后查询时间**：取请求渲染时刻；若走 HTMX 局部刷新，片段也要回带新的 queriedAt（否则刷新后时间不更新）。

### Flyway
- 本故事**无新迁移**（纯读 Redis + 既有 vet_accounts）。

### 范围边界（不做）
- 不做实时推送/自动刷新、不做在线态写入/修改、不做审计、不做双语外化（→ 1.6，新文案先中文）。
- 本故事**只**交付：只读在线快照专页（手动刷新 + 最后查询时间）+ vet.view 门控。

### 测试标准
- L0：`AdminWebControllerTest`（mock `AdminVetService`/`VetPresenceService`）：渲染含最后查询时间 + 各兽医在线态；HTMX 刷新返片段；无 `vet.view` 403；断言只读（never invoke presence 写方法）。
- L1（本地 docker）：goOnline 几个兽医 → 快照显示在线；TTL 过期/logout → 离线。云端只 L0。

- 新增 `admin/dto/VetOnlineSnapshot`、`templates/admin/vet-online.html`；修改 `admin/service/AdminVetService`、`admin/web/AdminWebController`。
