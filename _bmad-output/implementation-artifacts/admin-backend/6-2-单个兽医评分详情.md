# Story 6.2: 单个兽医评分详情

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

> **范围域**：管理后台 V1.0.0 · Epic 6（兽医评分质量看板）· Story 6.2（AB-6B）。**纯只读详情**，与 6.1 互不依赖（无前向依赖）。
> **产物归属**：仅后端 `petgo-backend`（**无 Flutter 侧**）。brownfield **扩/规整既有页**。
> **brownfield 现状**：既有 `GET /admin/vets/{id}/ratings` → `admin/vet-ratings.html`（Story 5.6）**已上线**，展示该兽医「均分 + 已评历史（星级/评价/时间）」。**本故事的增量 = 补「未评问诊单列 + 标注原因」**（既有页只有已评列表，没有未评区）。
> **隐私边界**：V1.0.0 评分**不对 App 用户/兽医公开**，仅运营可见（沿用 `ConsultRatingQueryService`「仅运营可见」语义）。

## Story

As a **运营成员**,
I want **打开某个兽医的评分详情页，既看到全部已评问诊（日期/1-5 星/文字评价），也看到未评问诊并标明原因**,
so that **我能深入分析该兽医的服务表现，并区分「真没评」与「评分门超时」等情况（FR-33 / AB-6B）**。

## Acceptance Criteria

> 验证层：**L0** 编译/单测/MockMvc 切片（mock service，无需 DB）· **L1** 集成（Docker postgres+redis，验已评/未评分区与原因标注真跑）· **L2**（本故事无）。

1. **AC1（已评历史完整展示，L0/L1）**：详情页展示该兽医**已评问诊完整历史**：日期、1-5 星、文字评价（无评价显示「—」），按时间倒序。沿用既有 `vet-ratings.html` 的已评列表与均分摘要，**不回退**。
   **Given** 某兽医有若干评分
   **When** 打开 `/admin/vets/{id}/ratings`
   **Then** 列出全部已评条目（日期/星级/评价），均分摘要与底层一致
2. **AC2（未评问诊单列 + 原因，L0/L1）**：**新增**「未评问诊」区，单独列出该兽医未获评分的问诊，每条**标注原因**；原因取自会话终态，映射见 Dev Notes「未评原因映射」。
   **Given** 某兽医有未评问诊（CLOSED 无评分 / 中断等）
   **When** 打开详情页
   **Then** 未评区逐条列出（含会话标识/日期 + 原因标签），与已评区分离清晰
3. **AC3（原因如实映射，不杜撰，L0/L1）**：未评原因按**实际数据模型**标注，**不得**编造数据中不存在的类别。当前模型唯一的「无评分自动关闭」终态是 `closed_reason=UNRATED`（30min 评分门超时）；中断为 `INTERRUPTED`。映射与「用户未评/30 分钟超时」的对应关系见 Dev Notes，差异需在 Completion Notes 记录。
   **Given** UNRATED 关闭且无补评分的会话 / INTERRUPTED 会话
   **When** 渲染未评区
   **Then** 前者标「30 分钟超时自动关闭（用户未在评分窗口内评分）」，后者标「会话中断未评（兽医封禁中断）」或按 Dev Notes 约定
4. **AC4（纯只读，L0）**：详情页**无任何运营操作功能**（无编辑/删除/隐藏/回复/导出按钮），仅查看（PRD AB-6B「纯只读」）。
   **Given** 详情页
   **When** 渲染
   **Then** 页面不含任何对评分/会话产生写副作用的控件
5. **AC5（隐私与门控，L0/L1）**：端点经 `@PreAuthorize("hasAuthority('rating.view')")` 门控（横切依赖 Story 1.5）；评分/未评数据**绝不**出现在任何 App 用户/兽医可达接口。
   **Given** 无 `rating.view` 权限的后台账号
   **When** 访问该详情页
   **Then** 方法级拒绝 403（前端隐藏非安全边界）
6. **AC6（跨模块边界，L0）**：补充的未评数据**只经既有/扩展的 consult `service` 接口**取得，`com.tailtopia.admin`（含 rating/既有 AdminVetService）**绝不**直接注入 `consult.repository` / `vet.repository`（禁跨 repo）。
7. **AC7（回归绿，L0/L1）**：既有 5.6 单兽医评分页行为不回归（已评列表 + 均分仍正常）；`mvn -B compile` + L0 全绿；本地 L1 全量回归绿。

### 🟡 未评原因映射（关键，AC3 —— 不杜撰）
PRD AB-6B 文案列了两类原因：「用户未评分 / 30 分钟超时自动关闭」。**但当前数据模型里二者同源**——会话从 `PENDING_CLOSE` 只有两条出口：`closeRated()`（已评）或 `closeUnrated()`（30min 评分门超时、用户未在窗口内评分）。**不存在**独立的「用户主动拒评」终态。建议映射：
- **`status=CLOSED` 且 `closed_reason=UNRATED` 且无评分行** → 标「30 分钟超时自动关闭（用户未在评分窗口内评分）」。这一类同时覆盖 PRD 所说的「用户未评分」与「30 分钟超时」（实现上是同一事件）。
- **`status=INTERRUPTED`** → 标「会话中断未评（兽医封禁中断，不进入评分）」。这是 PRD 文案未单列、但数据中真实存在的第三类未评会话；**如实呈现**，不要塞进上面两类。
- **`closed_reason=RATED` 或存在评分行** → 属已评区，不进未评区（注意 `UNRATED` 后**补评分**的会话有评分行 → 归已评，**以「是否存在评分行」为权威**，不以 `closed_reason`）。
- ⚠️ 实现**不得**编造「用户主动拒评」这类数据里没有的标签。PRD 文案与实现的差异在 Completion Notes 记录，若产品坚持二分需回填 PRD / 排后续故事补数据字段。

### 关键边界 / 防坑
- **已评判定以评分行为权威**：`UNRATED` 关闭后补评分的会话有评分行 → 算已评、不进未评，避免重复计入两区。
- **隐私**：未评/已评均仅运营可见，切勿泄露到任何 `/api/v1` 或 App 可达响应。
- **纯只读**：模板不得出现任何写控件（含「下架/隐藏评价」之类未来想象功能）；AC4 要求页面零写副作用。
- **门控就位**：`@PreAuthorize("hasAuthority('rating.view')")` 依赖 Story 1.5 的 `@EnableMethodSecurity` 与 permission authority 装配；若 1.5 未落地，注解先就位（待生效），且**不弱于**既有 `ROLE_ADMIN` 链门控；在 Completion Notes 标注生效层级。
- **会话标识展示**：未评区列「会话标识」用内部 `sessionId` 即可（运营内部工具）；不展示 IM 正文/AI 上下文/用户媒体（NFR5 数据边界——本页本就只取元数据 + 评分）。
