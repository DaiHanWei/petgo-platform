# Story 2.2: 兽医账号列表与搜索筛选

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

> **范围域**：管理后台 V1.0.0 · Epic 2 · Story 2.2（AB-2A）。**依赖 Story 2.1**（`vet_qualifications` 资质状态）。
> **产物归属**：仅后端 `petgo-backend`（**无 Flutter 侧**）。SSR + HTMX，`/admin/vets`，不返 JSON。
> **brownfield 关键**：`/admin/vets` 列表页 + `AdminVetService.list()` + `VetAdminView` + `vets.html` **已存在**（Epic 5 Story 5.1 建）。本故事**扩列表的展示列与筛选/搜索**，不重建页面。

## Story

As a **运营成员**,
I want **按账号状态、资质状态、在线状态多维筛选并按姓名/邮箱搜索兽医账号列表**,
so that **在兽医增多后仍能快速定位目标兽医**。

## Acceptance Criteria

> 验证层：**L0** MockMvc 切片（mock service，无 DB）· **L1** 集成（Docker postgres+redis，验真实筛选/资质 join/在线快照）。

1. **AC1（列表展示扩列，L1）**：`/admin/vets` 列表行展示：**姓名（displayName）/ 登录邮箱 / 账号状态（活跃·已封禁）/ 资质状态（6 态，读 2.1 `vet_qualifications`）/ 在线快照（读既有 Redis presence）/ 评分均分（读既有 consult 评分）/ 创建日期**。`VetAdminView` 扩展承载这些字段（绝不含 `passwordHash`）。
2. **AC2（多维筛选，L1，HTMX）**：支持按 **账号状态**(ACTIVE/BANNED) + **资质状态**(6 态) + **在线状态**(ONLINE/OFFLINE，BUSY 归入「在线」展示)筛选；筛选条件经 query 传入（`/admin/vets?accountStatus=&qualStatus=&online=`）；`HX-Request` 时返回 `admin/vets :: rows` 片段，整页请求返完整视图。
3. **AC3（姓名/邮箱搜索，L1，HTMX）**：支持按姓名或登录邮箱关键词搜索（`/admin/vets?q=`，大小写不敏感、子串匹配）；与 AC2 筛选可叠加；同走 HTMX 片段刷新。
4. **AC4（空态 + 门控，L0/L1）**：无匹配结果显示空态文案；列表页与片段端点受 `@PreAuthorize("hasAuthority('vet.view')")` 门控（无权限 403）。
5. **AC5（回归绿，L0/L1）**：既有开户/重置密码/封禁/评分查看入口（5.1/5.6/5.7）在扩列后仍可用、不回归；`mvn -B compile` + admin 单测全绿；本地 L1 全量回归绿。

## Dev Notes

### 架构约束（必须遵守）
- 路由 `/admin/vets`（**绝不** `/api/v1`）；HTMX 端点返 HTML 片段（`HX-Request` 区分），**不返 JSON**；筛选走 query。[Source: architecture.md#Naming Patterns / Format Patterns（增量）]
- 授权方法级 `@PreAuthorize("hasAuthority('vet.view')")`（权威），模板按权限隐藏入口仅体验（`sec:authorize`）。`vet.view` 权限基础由 Story 1.5 提供；本故事按其约定加注解（前向兼容）。[Source: architecture.md#Process Patterns；PRD 附录 B]
- 跨模块只经既有 service（presence/评分/资质），**禁跨 repo**；在线快照读既有 Redis presence（AB-2F 只读）。[Source: architecture.md#Communication Patterns]
- 本故事**只读**，无写操作 → **不写审计**（审计仅写操作）。

### 既有代码基线（READ 过，brownfield）
- **现状**：`AdminWebController#vets(Model)`（`GET /admin/vets` 无参，`adminVetService.list()` 取全量）；`AdminVetService.list()` → `vetAccounts.listAll().map(VetAdminView::from)`；`VetAdminView(id, username, displayName, status, createdAt)`（仅 5 字段，无资质/在线/均分）；`vets.html` 有开户卡 + 表格（列：ID/昵称/账号/状态/评分(链接)/重置/封禁），表格**未抽 fragment、无筛选条、无 HTMX**。
- 可复用 service：`VetPresenceService.statusOf(vetId)`→`VetPresenceStatus{ONLINE,BUSY,OFFLINE}`；`ConsultRatingQueryService.forVet(vetId)`→`VetRatingsView`（含均分）；Story 2.1 的 `VetQualificationService.getStatus(vetId)`。
- **本故事改什么**：扩 `VetAdminView`（+资质/在线/均分）、`AdminVetService.list(filter)`（组装 + 过滤）、`AdminWebController#vets`（接参 + HTMX 分支 + 门控）、`vets.html`（筛选条 + rows fragment + 新列）。
- **不可破坏**：① 开户表单 POST `/admin/vets`、重置 `/{id}/password`、封禁 `/{id}/status`、评分 `/{id}/ratings` 既有端点与行为不变（封禁逻辑在 2.5 改，本故事不动）；② 列表绝不含 `passwordHash`；③ 既有 `createdVetId` 开户成功回显逻辑保留。

### 关键边界 / 防坑
- **均分批量取的性能**：`ConsultRatingQueryService.forVet` 逐条返回完整评分明细，列表逐行调会拉大量明细。V1 兽医量小可接受；**但优先**为列表新增一个轻量均分聚合查询（如 `ConsultRatingQueryService.avgFor(vetIds)` 或单兽医均分），避免在列表页加载全部评分文字。若新增聚合，仍经 consult service 接口（不跨 repo）。标注：未评兽医均分为空（显示「—」）。
- **在线快照语义**：`statusOf` 读 Redis 即时态，但 AB-2F 强调「快照、非实时」——列表加载时取一次即可，**不在列表页加自动刷新轮询**（自动刷新/最后查询时间是 2.6 专页的事）。
- **HTMX fragment 边界**：`rows` fragment 必须只含 `<tr>` 集合（含空态行），筛选条/表头留在整页；`hx-target` 指向 `tbody#vet-rows`。CSRF 对 GET 不需要，但本页若有内联 POST 表单仍带 token（沿用既有）。
- **筛选默认**：无参 = 全部（不过滤），保持既有「列全量」行为不变。

### Flyway
- 本故事**无新迁移**（资质状态读 2.1 的 V34；账号读既有 vet_accounts）。如确需新列（不预期）从 **V38** 顺延（Epic 2 列增统一从 V38 起，见 sprint 约定）。

### 范围边界（不做）
- 不做封禁逻辑改造（→ 2.5）、在线快照专页/最后查询时间/手动刷新（→ 2.6）、资质录入审核（→ 2.7）、双语外化（→ 1.6，新文案先中文）。
- 本故事**只**交付：列表扩列 + 多维筛选 + 搜索（HTMX 片段）+ vet.view 门控。

### Completion Notes List

- **门控口径**：用 `hasRole('SUPER_ADMIN') or hasAuthority('vet.view')`（非 story 字面的 `hasAuthority('vet.view')`）——保持与 1.5 隐式全权选型一致，否则超管（无细粒度 authority）会被锁死。
- **均分取法**：复用 `ConsultRatingQueryService.forVet`（含全部明细），count 0 → 均分 null（显示「—」）。防坑提到可加轻量聚合查询；V1 兽医/评分量小，暂复用既有接口（标注），不新增 consult 查询以免越界改它模块。
- **在线维度**：ONLINE 含 BUSY（均视为「在线」展示/筛选）；OFFLINE 仅离线。在线快照列表加载时取一次，不加轮询（自动刷新属 2.6）。

- **i18n**：新增 21 键×2（资质 6 态 admin.qual.* + 在线态 + 筛选/列），动态键 `#{'admin.qual.' + ${v.qualStatus()}}`；key 对齐测试通过。
- **只读无审计**：列表为只读，不写审计；不动开户/重置/封禁/评分既有端点（封禁逻辑改造在 2.5）。`VetTestSupport.newActiveVet` 仍为 CERTIFIED（2.1 起）。
