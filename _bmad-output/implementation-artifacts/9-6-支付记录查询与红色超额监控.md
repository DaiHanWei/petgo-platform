# Story 9.6: 支付记录查询与红色超额监控

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

> **所属**：V1.1 Epic 9 第六个 Story（**后端 + admin 模板**）。承接 9-1 权限范式。交付两块：① **支付记录通用查询**（AB-8E）——按用户跨类型只读查 `payment_intents`（VET_CONSULT/PAWCOIN_TOPUP/AI_UNLOCK/ID_HD 四类支付统一）；② **红色超额只读监控**（AB-7A）——按用户聚合 RED 分诊计数，人工标记 待核查/已处理，**无自动拦截**（阈值 OPEN，仅埋点观测，OQ-11）。
> **关键事实**：`payment_intents`（V60）已含 user_id/purpose/channel/amount/status/created_at（idx_user）→ 直接查。RED 计数源 `triage_tasks.danger_level='RED'` group by user_id。标记需持久化 → V81 建 `red_overage_reviews`（user 维度单行）。
> **权限**：9-1 未含。本 story 加 `payment.view`（支付记录查询）+ `risk.view`（红色超额监控查看 + 标记，内部注记低危同门控），常量+i18n。

## Story

As a **客服/运营**,
I want **跨类型支付记录查询 + 红色超额只读监控**,
so that **我能排查与观测（AB-8E/7A）**。

## Acceptance Criteria

> **验证层**：**L0**（行映射/聚合组装单测、`mvn compile`）· **L1**（真 pg：跨类型查、RED 聚合、标记落库 + 审计、schema validate）。无 L2。

### AC1 — 支付记录按用户跨类型只读查

**Given** `payment.view` 门控 + 按 userId 查 `payment_intents`（四类 purpose 统一）
**When** 输入 userId 查询
**Then** 返回该用户全部支付意图（token/purpose/channel/amount/status/时间，倒序），只读；无 userId → 空/提示
> 验证层：**L1**（造多类型支付 → 全返、倒序、跨类型）+ **L0**（行映射单测）。

### AC2 — 红色超额只读监控 + 人工标记（无自动拦截）

**Given** `risk.view` 门控 + 按用户 RED 分诊计数（desc）+ 复核态（`red_overage_reviews`，Flyway V81）
**When** 查看 / 标记 TO_VERIFY / RESOLVED（可备注）
**Then** 展示各用户 RED 计数 + 复核态；标记落 `red_overage_reviews` + `AdminAuditService` 审计；**绝不自动拦截/限流/封禁**（AB-7A，纯观测 + 人工注记）
> 验证层：**L1**（RED 聚合计数正确 + 标记落库 + 审计；被标记用户无任何自动处置）+ **L0**（聚合/标记校验单测）。
