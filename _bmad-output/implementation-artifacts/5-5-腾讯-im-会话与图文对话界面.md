# Story 5.5: 腾讯 IM 会话与图文对话界面

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a **用户与兽医**,
I want **实时图文对话并附专业辅助**,
so that **我能获得专业指引、兽医能高效判断**。

> 本 Story 是 Epic 5 的**核心实时段**：兽医在「待接单」接单 → 后端经腾讯 IM 建会话、状态 `WAITING→IN_PROGRESS`、双方进入图文对话界面、切 Tab 后台保持连接消息连续（NFR-5）、兽医侧 FR-5 辅助工具（AI 参考回复 + 历史摘要冷启动）、用户侧免责提示。
>
> 🔄 **PRD V1.0.0 修订（F4 · 2026-06-08）：** V1.0.0 全程无视频——会话仅文字 + 图片，删除视频聊天能力。
>
> 做：接单流程（待接单列表 → 接受 → IM 建会话 → 状态迁移 → 双端进入对话）、腾讯 IM Flutter SDK 直连对话界面（文字/图片）、后台保连消息连续、兽医 FR-5 辅助（AI 参考回复，历史摘要冷启动空库仅给 AI 参考）、用户侧免责提示、接单时兽医置 BUSY。
> 不做：结束会话 / PENDING_CLOSE / 评分（5.6）、封禁中断（5.7）、IM→OSS 存档（5.6/Epic 2）、推送本身（Epic 6，本 Story 用 IM 系统消息 + 留推送事件）。
>
> 依赖前序：5.2（在线态 / 待接单队列基础 + BUSY 枚举位）、5.3（`consult_sessions` WAITING + 队列 + `ConsultAcceptedEvent` 占位 + 「同时仅 1 个」）、5.4（AI 上下文卡数据，挂本 Story 对话顶部）。本 Story **首次引入腾讯 IM**——架构明确「后端经 IM REST/回调建会话、发系统消息，后端不持长连接；客户端腾讯 IM Flutter SDK 直连」。**含 L2**（真实腾讯 IM appid + UserSig + 真机）。

## Acceptance Criteria

> **验证层标注**：**L0**（状态迁移单元 / DTO / lint） · **L1**（需 Docker + pg + redis：接单迁移 WAITING→IN_PROGRESS、出队、BUSY 置位、辅助工具数据） · **L2**（需**真实腾讯 IM appid/SecretKey、UserSig 签发、真机**：建 IM 会话、收发文字/图片、后台保连消息连续、IM 系统消息）。
> 本 Story 实时对话**主体在 L2**（真实 IM 不可在纯 Docker 验）；状态机迁移与队列在 L1；辅助工具数据 L1，渲染 L2。

### AC1 — 接单 → IM 建会话 + WAITING→IN_PROGRESS

**Given** 兽医在「待接单」点击「接受」
**When** 会话创建
**Then** 后端经腾讯 IM 建会话、状态迁移 WAITING→IN_PROGRESS、自动移入「进行中」，用户侧收到「兽医已接受你的问诊，点击开始对话」（FR-30 接单流程）
> 验证层：状态迁移 WAITING→IN_PROGRESS + 出待接单队列 + 兽医置 BUSY = **L1**；腾讯 IM 建会话（后端 REST 建会话/填 im_conversation_id）+ 用户侧「兽医已接受」IM 系统消息 = **L2**（真实 IM）。**并发抢单**：多兽医同时点「接受」→ 仅一人成功（DB 乐观锁/状态 CAS），其余得「已被接走」= **L1**。

### AC2 — 进行中图文对话 + 后台保连消息连续

> 🔄 **PRD V1.0.0 修订（F4 · 2026-06-08）：** **V1.0.0 全程无视频**——会话仅支持文字 + 图片，删除视频聊天能力（视频 ≤60s 客户端校验整条移除）。视频聊天随收费模式后置。

**Given** 进行中会话
**When** 双方对话
**Then** 支持文字/图片（腾讯 IM 传输）；用户切其他 Tab 时会话后台保持连接不中断、切回恢复消息连续（NFR-5）
> 验证层：**L2**（真实 IM 收发文字/图片；切 Tab 后台保连、切回消息连续）。图片客户端压缩（≤10MB）= **L2**。聊天媒体由腾讯 IM 托管（架构媒体三层③），**不**经 OSS、**不**经后端中转。

### AC3 — 兽医 FR-5 辅助工具 + 用户侧免责提示

**Given** 兽医接单后
**When** 查看辅助工具
**Then** 展示 FR-5 辅助：按症状关键词自动匹配历史判断摘要（冷启动数据库为空时仅展示 AI 参考回复）、用户侧自动展示免责提示「本次建议为参考，最终决策权在您」（FR-5、NFR-9）
> 验证层：AI 参考回复 + 历史摘要匹配（冷启动空库 → 仅 AI 参考回复）数据 = **L1**（G-2：冷启动仅 AI 参考回复，历史匹配为后置增强）；兽医侧辅助面板渲染 + 用户侧免责提示展示 = **L2**。免责提示文案双语、克制（UX-DR14：免责声明克制）。

## Dev Notes

### 关键架构约定（本 Story 必须落实）

- **后端编排 IM、不持长连接**：建会话/系统消息/封禁中断在 Java 侧经 IM REST/回调；**实时收发由客户端腾讯 IM Flutter SDK 直连**（架构 §API & Communication / Frontend / Integration Points）。后端只编排，不做消息中转。
- **聊天媒体留 IM（媒体三层③）**：聊天图由腾讯 IM 托管，**不经 OSS、不经后端**（架构 §Data Architecture 媒体三层）。存档时（5.6）才复制必要图到私密桶。
  > 🔄 **PRD V1.0.0 修订（F4 · 2026-06-08）：** V1.0.0 IM 仅图片，无视频（架构媒体三层③「IM 图/视频」→仅图片同步收敛）；VOD/转码本就 V1 不做，随视频能力一并后置至收费模式。
- **会话状态机 WAITING→IN_PROGRESS 在本 Story**：接单是这一迁移的唯一触发；CAS 防并发抢单（架构状态机在 Java 侧）。BUSY 子态（5.2 留位）在接单时置位、结束/中断时回 ONLINE/OFFLINE。
- **FR-5 辅助冷启动（G-2）**：历史判断摘要库冷启动为空，V1 仅给 AI 参考回复，历史匹配为 consult 内查询后置增强。AI 参考回复仅供兽医参考、不自动发用户（NFR-9 专业辅助不替代判断）。
- **免责提示（NFR-9 / UX-DR14）**：用户侧常驻「本次建议为参考，最终决策权在您」，克制、双语。

### 会话状态机专项

> 本 Story 是状态机的**实时核心段**：`WAITING →(接单,本Story) IN_PROGRESS`。接单：CAS 迁移 + 填 vet_id + 出队 + IM 建会话 + 兽医 BUSY + 系统消息 + `ConsultAcceptedEvent`。进行中：双端 IM SDK 直连对话。**结束（IN_PROGRESS→PENDING_CLOSE）在 5.6**、**封禁中断（IN_PROGRESS→INTERRUPTED）在 5.7**、**存档（CLOSED→OSS 复制）在 5.6/Epic 2**——本 Story 均不做，但 `ImCallbackController` 骨架与 `im_conversation_id` 为它们铺好基础。

### 强制护栏（架构 §Enforcement —— 违反即返工）

- **后端不持 IM 长连接 / 不中转聊天媒体**；聊天媒体留 IM，不落 OSS、不落后端、不入日志。
- **禁 MQ / 通用缓存 / 新中间件**；状态迁移用 DB CAS；BUSY 用 5.2 Redis 在线态（不新增 Redis 用途）。
- **IM SecretKey / UserSig 签发**：SecretKey env 注入、绝不下发客户端、绝不入日志；UserSig 短时。
- **角色门控**：接单 `role=vet`；对话双方各自 role；`/im/callback` 签名校验 + 内网/白名单。
- **AI 参考回复不自动发**、免责前置——专业辅助不替代兽医判断（NFR-9）。

### 范围边界（防 scope creep —— 本 Story 明确不做）

- ❌ 不做结束会话 / PENDING_CLOSE / 30min 评分门 / 评分（Story 5.6）。
- ❌ 不做封禁 → INTERRUPTED 中断（Story 5.7，本 Story 仅 `ImCallbackController` 骨架）。
- ❌ 不做 IM→OSS 存档复制（5.6 / Epic 2 `ImToOssArchiver`）。
- ❌ 不做推送本身（Epic 6 / 6.2，本 Story 发 `ConsultAcceptedEvent` + IM 系统消息）。
- ❌ 不做历史摘要匹配算法（G-2 后置增强，本 Story 冷启动返回空）。
- ❌ 不做视频聊天 / VOD / 视频转码（🔄 **F4 · 2026-06-08**：V1.0.0 全程无视频，会话仅文字 + 图片，视频随收费模式后置）。
- ✅ 只做：接单(CAS+IM 建会话+BUSY)+ WAITING→IN_PROGRESS + 双端 IM SDK **图文**对话 + 后台保连消息连续 + UserSig 签发 + IM 回调骨架 + FR-5 辅助(AI 参考回复 + 冷启动空历史)+ 用户免责提示。

- **UserSig 签发与 TTL**：`GET /api/v1/im/usersig` 按 JWT role 为自身 IM 账号签 UserSig；TTL `IM_USERSIG_TTL_SECONDS` 默认 86400s。SecretKey **仅服务端持有、绝不下发客户端、绝不入日志**；`UserSig` record 不含 SecretKey。
- **C2C vs Group 选型**：接口 `createConversation(userImId, vetImId)` 抽象，stub 返回本地会话 id；真实选型（建议 C2C 单聊，一对一咨询）在 live 实现时落定（L2）。
- **IM 模式开关**：`petgo.im.mode=stub`（默认，免凭证验状态机/接单 CAS/BUSY/查询）/ `live`（真实 IM，L2）。**live 未实现 → `ImConfig` 启动即抛错**，避免用桩冒充真实 IM 或伪造凭证。
- **BUSY 子态**：`VetPresenceService` 加 `vet:busy` SET，`goBusy`/`goAvailable`/`statusOf`（BUSY 优先于 ONLINE）；接单 goBusy，结束/中断（5.6/5.7）goAvailable，goOffline 清 busy。
- **FR-5 辅助冷启动（G-2）**：`assist` 返回基于会话 AI 上下文的**模板化** AI 参考回复（确定性，免凭证可验）+ **空历史摘要**。真实 Gemini 生成更自然参考回复属 L2 增强。AI 参考回复**不自动发**（前端「采用」填输入框，NFR-9）。
- **聊天媒体留 IM**：后端不持长连接、不中转聊天媒体、不落 OSS/日志；实时收发由客户端腾讯 IM Flutter SDK 直连（占位 `ImChatPlaceholder`，真机接入）。
- **IM 回调骨架**：`POST /im/callback`（permitAll + `verifyCallback` token 校验），存档(5.6)/中断(5.7) 事件分发在此接。
