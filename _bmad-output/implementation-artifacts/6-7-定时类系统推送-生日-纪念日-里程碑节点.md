## Story

As a **有宠物档案的用户**,
I want **在宠物生日前、陪伴纪念日、重大里程碑节点收到系统提醒**,
so that **我不会错过记录这些特别时刻，与宠物的成长被珍视**。

> 本 Story 实现三类**定时触发的系统推送**：① 宠物生日提醒（FR-40，扫 `pet_profiles.birthday` 前 1 天）② 陪伴纪念日提醒（FR-41，扫 `pet_profiles.created_at + {30,100,365} 天`）③ L级里程碑节点推送（FR-42 中**仅"系统推送提醒"切片**，如「第一个生日」节点提醒）。三类均经 **Story 6.1** 的 `NotificationService.send` 下发——落 `notifications` 一行 + Redis 角标自增 + 经腾讯 IM 离线通道投递；**未授权系统推送的用户当天仅落 App 内通知中心（6.6 展示）**，不投递系统推送。
> **依赖**：① **Story 6.1** 的 `NotificationService` / `NotificationType`（已新增 `PET_BIRTHDAY`/`COMPANION_ANNIVERSARY`/`MILESTONE_NODE`）/ 深链路由表（生日→「+发布」预选成长日历、纪念日→成长档案 Tab、L级里程碑节点→里程碑列表页壳）就绪；② **Story 2.2** 的 `pet_profiles.pet_type` / `birthday`；③ `pet_profiles.created_at`（2.2 建表已有）。
> **明确不做里程碑系统本体**：里程碑清单（猫30/狗30/通用15）、徽章、三级庆祝动效、打卡、列表页本体、自动完成事件订阅、L级分享卡 —— 全部属**里程碑 mini-epic（决策 F2，排 Epic 8 / 1.0.x-1.1.0）**，本 Story 仅做**「系统推送提醒」这一切片**，不依赖里程碑表落地。

## Acceptance Criteria

> **验证层标注**：每条 AC 末尾标注验证层级与所需本地环境——
> **L0 静态**（编译/lint/纯单测，无需 DB） · **L1 集成/运行时**（需 Docker daemon + postgres + redis：扫描 + 去重落库 + 角标自增 + IM 离线投递触发） · **L2 端到端/外部**（真实 APNs/FCM 经腾讯 IM、真机收到系统推送、点击深链直达）。
> **定时扫描 + 去重逻辑可在 L0 金标单测验证**（注入"当前日期"+ 假 repository，断言扫出/跳过/不重复触发的集合）；**真投递经 IM 为 L1/L2**。

### AC1 — 生日提醒：宠物生日前 1 天系统自动触发，每年仅一次（FR-40）

**Given** 用户宠物档案填写了生日（`pet_profiles.birthday`）
**When** 每日定时扫描命中"明天是该宠物生日"（按 month/day 比对，含 2/29 兜底）且本年度该节点未推过
**Then** 经 `NotificationService.send(type=PET_BIRTHDAY, ...)` 下发「[宠物名] 明天就 X 岁啦！记录一条特别的快乐时刻吧 🎂」，点击深链跳「+发布」预选成长日历类型；落 `notifications` + 角标；未授权系统推送用户仅落通知中心
**And** 档案**未填写生日**字段时**不触发**；**每年该宠物生日仅推一次**（DB 去重标记）
> 验证层：**L0**（扫描判定"明天是生日 && 本年未推 && 生日非空 ⇒ 入投递集；已推/生日空/非明天 ⇒ 跳过"纯单测，注入当前日期 + 假 profiles + 假去重记录全覆盖，含 2/29→平年 2/28 兜底）+ **L1**（真扫 pg + 写 `notifications` + 去重标记落库 + 角标自增 + IM 离线投递触发，需 pg+redis）+ **L2**（真机收到生日系统推送、点击直达「+发布」预选成长日历，真实 IM/APNs/FCM）。

### AC2 — 陪伴纪念日：建档满 30/100/365 天触发，每节点仅一次（FR-41）

**Given** 用户创建宠物档案满 30 / 100 / 365 天（以 `pet_profiles.created_at` 为基准）
**When** 每日定时扫描命中"今天恰为某节点天数（now::date − created_at::date ∈ {30,100,365}）"且该节点未推过
**Then** 经 `NotificationService.send(type=COMPANION_ANNIVERSARY, ...)` 下发「你和 [宠物名] 在一起 X 天了 🎉 快去看看 TA 的成长故事」，点击深链跳成长档案 Tab；落 `notifications` + 角标；同时出现在通知中心（已授权系统推送者同时收系统推送）
**And** **每个节点（30/100/365）仅触发一次**（DB 去重标记，按 `profile × node`）；365 之后不再有 V1 节点
> 验证层：**L0**（扫描判定"天数差 ∈ {30,100,365} && 该 profile×node 未推 ⇒ 入投递集"纯单测，注入当前日期 + 假 created_at 边界（29/30/31、99/100/101 天）+ 假去重记录全覆盖）+ **L1**（真扫 pg + 去重落库 + 角标 + IM 投递触发）+ **L2**（真机收纪念日系统推送、点击直达成长档案 Tab）。

### AC3 — L级里程碑节点推送：仅"系统推送提醒"切片，不做里程碑本体（FR-42 部分）

**Given** 某 L级里程碑节点对应一个**可由日期推算的系统提醒时点**（V1 范围：如「第一个生日 🎂」节点 = 生日当天，触发方式标"系统推送 + 用户当天发布"）
**When** 每日定时扫描命中该节点时点且未推过
**Then** 经 `NotificationService.send(type=MILESTONE_NODE, ...)` 下发节点提醒，点击深链跳成长档案 Tab → 里程碑列表页（壳，6.1 已落）；落 `notifications` + 角标
**And** **本 Story 仅推送，不写里程碑完成态 / 不发徽章 / 不触发庆祝动效 / 不建里程碑表**——这些属里程碑 mini-epic（F2）；节点**每节点仅推一次**（DB 去重）
> 验证层：**L0**（节点时点扫描 + 去重判定纯单测；并断言**本 Story 不写任何里程碑完成/徽章数据**——边界守护测试）+ **L1**（写 `notifications` + 去重 + 角标 + IM 投递触发）+ **L2**（真机收里程碑节点推送、点击落里程碑列表页壳）。
> **范围守护**：里程碑清单/徽章/动效/列表页本体/打卡/自动完成订阅一律**不在本 Story**（决策 F2，Epic 8 mini-epic）。

## Dev Notes

### 关键架构约定（沿用全局）

- **目录/分层**：后端 `com.petgo.notify.{schedule,service,domain,repository,dto}`——新增 `notify/schedule/ScheduledPushJob`（`@Scheduled` 扫描）+ 去重 `scheduled_push_marks` 表与 repository；复用 6.1 `NotificationService` / `NotificationType` / `shared/im` 离线投递。前端**无新本体**，复用 `core/router/deep_link_routes`（6.1）+ 通知中心（6.6）。
- **命名映射链**：DB `snake_case`(`scheduled_push_marks`) ↔ Java/Dart `camelCase` ↔ JSON `camelCase`；枚举 UPPER_SNAKE；时间戳 timestamptz UTC；Flyway `V<序号>__<描述>.sql` 按执行顺序顺延（决策 E2）。
- **错误/日志**：单条投递失败仅记日志、不阻塞、不外泄 PII/健康数据/令牌（SLF4J JSON）。

### 定时推送专项约定（决策 F5 硬约束）

- **Spring 原生 `@Scheduled` 每日扫描 + `@Async` 逐条投递 + DB 去重标记位**——同 6.2「批量在线兽医循环走 @Async」既有范式。≤500 DAU 单机日扫足够。
- **DB 标记"当年/该节点是否已推"去重**：`scheduled_push_marks` 唯一约束 `(pet_profile_id, push_kind, node_key)` 为去重单一事实源——`node_key` 生日=年份、纪念日=天数、里程碑=节点 id。
- **生日扫 `pet_profiles.birthday`、纪念日扫 `pet_profiles.created_at + {30,100,365}d`**（决策 F5 明文规定的扫描口径）。
- **档案未填生日不触发**（FR-40）；**每节点仅一次**（FR-41 节点 / FR-42 节点 / FR-40 按年）。
- **未授权推送用户当天仅落通知中心**：依赖 6.1 `NotificationService.send` 既有「落 `notifications` + 角标 + IM 投递（未授权设备投递失败仅记日志）」行为，本 Story 无需特判。

### 强制护栏（违反即返工）

- **禁 Quartz / Kafka / RabbitMQ / 任何调度或消息中间件**——只用 Spring 原生 `@Scheduled` + `@Async`（决策 F5 / CLAUDE.md 护栏：异步只用 `@Async` + DB 状态机/标记，禁引入 MQ/通用缓存层/新中间件）。
- **禁**用 Redis 当去重单一事实源（角标键仅未读计数用途；去重落 DB 唯一约束）。
- **禁**本 Story 写里程碑表本体 / 标完成 / 发徽章 / 触发动效（属 Epic 8 mini-epic，F2 范围守护）。
- **禁**推送 payload / 深链携带顺序 id（复用 6.1 `deepLinkToken` 不可枚举）或健康数据明文。
- 日志严禁记录 PII / 健康数据 / 令牌。
