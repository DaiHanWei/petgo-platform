# Story 7.1: 「我的」页面与用户信息编辑

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a **用户**,
I want **一个集中管理我的信息与发布的个人中心**,
so that **我能编辑资料、回看我发过的内容、并找到设置（语言/退出/注销/帮助）入口**。

> 本 Story 是 Epic 7 的入口聚合页（FR-20「我的」页面结构），**混合前后端**：后端补齐用户信息读写（头像/昵称编辑）+「我的发布」时间线聚合接口；前端搭「我的」Tab 五大区块壳与编辑交互。语言设置详细收口在 **7.2**、退出登录/账号注销在 **7.3**、无障碍达标在 **7.4**——本 Story 只放它们的**入口**，不实现其内部逻辑。宠物状态修改与档案编辑入口**复用 Epic 2 的 FR-39 编辑页 / FR-0F 状态选择界面**，本 Story 不重写编辑页，只接入口并保证双向同步（FR-21）。

## Acceptance Criteria

> **验证层标注**：每条 AC 末尾标注验证层级与所需本地环境——
> **L0 静态**（编译/lint，无需 DB） · **L1 集成/运行时**（需 Docker daemon + postgres + redis） · **L2 端到端/外部凭证**（需真实第三方或真机模拟器）。
> 本 Story 主要落 **L0 / L1**；头像替换走 STS 直传 OSS 的端到端验证需真机 + 真实 OSS = **L2**。

### AC1 — 「我的」页面五大区块就位

**Given** 登录用户进入「我的」Tab
**When** 查看页面
**Then** 展示用户信息（头像 + 昵称 + 编辑按钮）、宠物状态（展示 A/B/C + 修改入口）、我的发布、账号设置（退出登录）、账号注销入口、帮助与反馈（FR-20）
> 验证层：**L0**（区块布局/组件树 widget test）+ **L1**（页面数据来自 `GET /api/v1/me` 聚合，需后端起栈拉真实用户行）。退出登录/注销入口仅占位跳转，其逻辑在 7.3。

### AC2 — 用户信息编辑即时生效（头像 + 昵称）

**Given** 用户编辑资料
**When** 修改头像（可替换 Google 默认头像）或昵称（≤20 字）
**Then** 即时生效
> 验证层：**L1**（昵称改 `PATCH /api/v1/me`，Bean Validation `≤20` 字、超长返回 422 ProblemDetail，需后端起栈）+ **L2**（头像替换走客户端压缩→STS 直传①公开桶→回填 URL 的真机/真实 OSS 全链路；昵称即时生效可在 L0 widget 层验状态回填）。

### AC3 — 「我的发布」三类混合时间线倒序

**Given** 我的发布区块
**When** 展示
**Then** 以时间线倒序展示用户全部已发布内容（三类混合），单条目可进入详情页或删除（FR-36）
> 验证层：**L1**（`GET /api/v1/me/posts?cursor=...` 游标分页 20/批、仅返回当前用户 `author_id` 且 `deleted_at IS NULL` 的内容，倒序按 `created_at`；删除复用 FR-36 二次确认 + 级联软删，需后端起栈 + 种子数据）。空态/网络错误/已删除态走 UX-DR18。

### AC4 — 状态 A 用户的档案入口与跨页同步

**Given** 状态 A 用户
**When** 查看「我的」
**Then** 显示「编辑宠物档案」入口（复用 FR-39 编辑页）；宠物状态修改与成长档案 Tab 同步（FR-21）
> 验证层：**L1**（状态 A=有宠物档案 → 显示「编辑宠物档案」入口跳 FR-39 编辑页；状态 B/C 不显示该入口而显示状态选择入口；修改宠物状态后「我的」与成长档案 Tab 双向即时同步、首页 Feed 按新状态刷新——需后端 `pet_profiles` 真实行 + Riverpod 共享状态）。

## Tasks / Subtasks

> **按"后端子任务 / 前端子任务 / 联调验收"三段组织**。本 Story 前后端混合：后端补 `me` 聚合读 + 信息写接口，前端搭页面壳与编辑交互。建议顺序：后端 → 前端 → 联调。

### 🟦 后端子任务（petgo-backend / profile + auth）

- [ ] **B1. 「我的」聚合读接口 `GET /api/v1/me`** (AC: 1)
  - [ ] 在 `profile/web/ProfileApiController` 暴露 `GET /api/v1/me`：BFF 式一次返回用户信息（头像 URL/昵称）+ 当前宠物状态(A/B/C) + 是否有宠物档案标志（降印尼↔德国串行往返，见架构 §延迟缓解）。
  - [ ] 仅返回 JWT `sub` 对应用户自身数据；他人不可达（403 越权走 ProblemDetail）。
- [ ] **B2. 昵称编辑 `PATCH /api/v1/me`** (AC: 2)
  - [ ] `NicknameUpdateRequest`(record) + Bean Validation `@Size(max=20)`；超长→422 ProblemDetail `errors:[{field:"nickname",message:"≤20"}]`（权威校验在服务端，见架构 §校验时机）。
  - [ ] 写库 `users` 昵称列，`updated_at` 刷新（timestamptz UTC）；返回更新后资源。
- [ ] **B3. 头像替换链路** (AC: 2)
  - [ ] 客户端 STS 直传①公开桶（不经后端，见架构 §媒体三层）；后端 `PATCH /api/v1/me` 接收新头像 OSS URL 回填 `users.avatar_url`（仅存应用自有 URL，不存 Google 临时 URL）。
  - [ ] 上传链路复用 `shared/media`（含 EXIF GPS 剥离，G-4）。
- [ ] **B4. 「我的发布」时间线 `GET /api/v1/me/posts`** (AC: 3)
  - [ ] 经 `content` 模块 service 接口查询（**禁 profile 直 join content repository**，见架构 §模块边界）：`author_id = 当前用户` 且 `deleted_at IS NULL`，按 `created_at` 倒序，游标分页 `{items,nextCursor,hasMore}` 20/批。
  - [ ] 三类内容混合返回（统一 `content_posts` 结构 + 类型字段）；列表项含进入详情所需 id 与删除入口标志。
- [ ] **B5. 删除复用 FR-36** (AC: 3)
  - [ ] 「我的发布」删除直接复用 content 模块 FR-36 删除端点（二次确认 + 点赞/评论级联 + Feed 与时间线同步移除），**本 Story 不新写删除逻辑**。
- [ ] **B6. 宠物状态读写与档案入口判定** (AC: 4)
  - [ ] 状态读取与 `GET /api/v1/me` 合并；状态修改复用 FR-0F/FR-21 既有端点（**不新建**），发 `PetStatusChangedEvent` 让成长档案/Feed 订阅刷新。

### 🟩 前端子任务（petgo_app / features/me）

- [ ] **F1. 「我的」页面五区块壳** (AC: 1)
  - [ ] `lib/features/me/presentation/me_page.dart`：用户信息卡 / 宠物状态卡 / 我的发布 / 账号设置(退出登录入口) / 账号注销入口 / 帮助与反馈，自上而下五大区块。
  - [ ] 数据 `meProvider`（`AsyncValue<MeState>`）拉 `GET /api/v1/me`；loading 骨架 / error 顶部 banner / 空态按 UX-DR18 声明。
  - [ ] 退出登录、账号注销、语言设置、帮助反馈**仅放入口**（路由占位 → 7.2/7.3 落地），本 Story 不实现其逻辑。
- [ ] **F2. 资料编辑（头像 + 昵称）** (AC: 2)
  - [ ] 昵称编辑：行内/弹窗输入，客户端实时预校验 `≤20` 字（体验层）；提交 `PATCH /api/v1/me`，服务端 422 → 本地化文案（dio 拦截器映射 ProblemDetail）。
  - [ ] 头像替换：选图→客户端压缩(≤10MB)→STS 直传①公开桶→回填 URL→即时刷新 `meProvider`。可替换 Google 默认头像。
  - [ ] 状态即时回填（copyWith 不可变更新），无需手动刷新页面。
- [ ] **F3. 「我的发布」三类混合时间线** (AC: 3)
  - [ ] 倒序列表 + 无限滚动 20/批（复用 content Feed 分页组件）；单条目可点进详情页或触发 FR-36 删除（二次确认）。
  - [ ] 空态（emoji 空状态 UX-DR8）/ 删除后局部移除。
- [ ] **F4. 宠物状态卡与档案入口** (AC: 4)
  - [ ] 展示 A/B/C 状态 + 修改入口（复用 FR-0F 状态选择界面）；状态 A 额外显示「编辑宠物档案」入口跳 FR-39 编辑页。
  - [ ] 宠物状态用共享 Riverpod provider，使「我的」、成长档案 Tab、首页 Feed 三处同步（监听 `PetStatusChangedEvent` 对应的状态失效/刷新）。

### 🟨 联调验收子任务

- [ ] **J1. 信息编辑闭环** (AC: 2 / **L1**+L2)
  - [ ] 改昵称为 21 字 → 422 ProblemDetail 含 field=nickname；改为 ≤20 字 → 200 + 页面即时更新。头像替换在真机/真实 OSS 验全链路（L2）。
- [ ] **J2. 我的发布与删除** (AC: 3 / **L1**)
  - [ ] 种子三类内容 → `GET /api/v1/me/posts` 倒序 20/批；删一条 → Feed 与时间线同步消失，点赞评论一并删除。
- [ ] **J3. 状态同步** (AC: 4 / **L1**)
  - [ ] 在「我的」改宠物状态 → 成长档案 Tab 与首页 Feed 即时反映新状态。

## Dev Notes

### 关键架构约定（本 Story 必须落实）

- **目录**：后端 `com.petgo.profile/{web,service,domain,repository,dto,event}` + `com.petgo.auth/`（用户身份）；「我的发布」经 content service 接口拿，**禁跨模块 join**。前端 `lib/features/me/{data,domain,presentation}`，复用 content 的 Feed 分页组件与 profile 的编辑页。
- **命名映射链**：DB `snake_case`（`users.avatar_url`、`users.nickname`、`content_posts.author_id`）↔ JSON camelCase（`avatarUrl`/`nickname`/`authorId`）↔ Dart/Java camelCase。时间 UTC ISO-8601。
- **聚合读**：`GET /api/v1/me` 走 BFF 式一次返回，减少印尼↔德国串行往返。
- **状态**：Riverpod `AsyncValue<MeState>`，不可变 copyWith；宠物状态用跨页共享 provider 保证三处同步。
- **错误**：统一 ProblemDetail；dio 拦截器映射本地化文案；昵称 422 含 `errors[].field`。

### ⚖️ 合规攸关约束

- **PDP 数据主体权利可达（NFR-8）**：本 Story 的「我的」页面是 PDP「数据主体权利可达路径」的**承载入口**——账号注销入口（→7.3）、帮助与反馈、隐私政策链接必须从此页**可达**（注销逻辑在 7.3，但入口缺失即合规缺口）。本 Story 须确保入口存在且可点击进入对应流程占位。
- **头像/昵称即 PII**：`avatar_url`/`nickname` 属个人身份信息——**日志严禁记录昵称、头像签名/URL、令牌**（架构 §日志，SLF4J JSON 不落 PII）。头像走①公开桶（Feed 可见），但上传链路仍须 EXIF GPS 剥离（G-4）防定位泄漏。
- **越权隔离**：`GET/PATCH /api/v1/me` 只可操作 JWT `sub` 本人数据；任何以他人 id 越权读写 → 403 ProblemDetail，不得返回他人 PII。

### 无障碍（NFR-13 / UX-DR15）—— 本页须满足，正式横切验收在 7.4

- 编辑按钮、状态修改入口、列表项、退出/注销入口等可交互元素触摸目标 **≥44×44pt**。
- 文本/图标在 #FAF8F5 底色对比度 **≥4.5:1**；三色态宠物状态徽标用 icon+text+color **不依赖颜色单一**。
- 昵称编辑输入框、各入口须有可访问性语义标签（screen reader 可读）。

### 强制护栏

- **禁跨模块直接访问对方 repository**：「我的发布」必须经 content service 接口；宠物档案/状态经 profile/FR-21 既有端点——本 Story **不重写**编辑页与删除逻辑（复用 FR-39/FR-0F/FR-36）。
- 头像 OSS URL 只存应用自有 URL，不存第三方临时 URL。
- 异步只用 `@Async`+事件，**禁引入 MQ/通用缓存层**。

### 范围边界（本 Story 明确不做）

- ❌ 不实现语言设置内部逻辑与双语文案收口（7.2）——只放语言设置入口。
- ❌ 不实现退出登录与账号注销级联删除逻辑（7.3）——只放入口。
- ❌ 不重写宠物档案编辑页（FR-39，Epic 2）、状态选择界面（FR-0F）、内容删除（FR-36，Epic 3）——均复用。
- ❌ 不做全局无障碍审计门槛收口（7.4）——本页满足无障碍即可。
- ✅ 只做：「我的」五区块壳 + `GET /api/v1/me` 聚合 + 头像/昵称编辑 + 我的发布时间线 + 状态/档案入口与跨页同步。

### Project Structure Notes

- 后端：`com.petgo.profile/`（用户信息读写 + 我的发布聚合）+ `com.petgo.auth/`（身份）；`content` 模块 service 提供发布列表。
- 前端：`lib/features/me/{data,domain,presentation}`；`core/storage/prefs`（宠物状态/语言偏好非敏感缓存）；复用 `features/profile`（FR-39 编辑页）与 `features/content`（Feed 分页/删除）。
