# Story 7.2: 语言设置与双语文案收口

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a **印尼/英语用户**,
I want **在 id 与 en 间切换并看到完整本地化文案**,
so that **我能用母语顺畅使用 App**。

> 本 Story 是 Epic 7 的双语**收口**story（FR-27 / NFR-11 / UX-DR14），**前端重**。i18n 脚手架（`l10n.yaml`、`app_en.arb`/`app_id.arb`、`flutter_localizations`、跟随设备语言回退英语）已在 **Story 1.1** 落地——本 Story 不重建脚手架，而是：①补「我的→语言设置」手动切换并即时持久化生效；②**逐屏收口**前面各 Epic（auth/triage/consult/content/profile/notify/me）所有 UI/系统/错误文案的 id+en 双套，消除一切写死字符串；③按场景沉淀 Voice & Tone microcopy 双语库（空状态/分析中/分享引导/红色预警/免责声明等），遵守 UX-DR14 的 microcopy 规则。后端仅需保证错误信息可本地化（`Accept-Language` 驱动 ProblemDetail 文案，或前端按 type 映射）。UGC 内容**不受语言设置限制**，可任意语言发布。

## Acceptance Criteria

> **验证层标注**：每条 AC 末尾标注验证层级与所需本地环境——
> **L0 静态**（编译/lint/gen-l10n，无需 DB） · **L1 集成/运行时**（需 Docker daemon + postgres + redis） · **L2 端到端/外部凭证**（需真机模拟器切系统语言）。
> 本 Story 主体落 **L0**（文案双套、无写死字符串、microcopy 规则可静态/单元校验），切换持久化与逐屏渲染需真机 = **L2**；错误文案本地化需后端起栈 = **L1**。

### AC1 — 首次启动跟随设备语言（其他回退英语）

**Given** App 首次启动
**When** 读取设备语言
**Then** 设备印尼语→印尼语、英语→英语、其他→默认英语（FR-27）
> 验证层：**L0**（`MaterialApp` `supportedLocales=[id,en]` + `localeResolutionCallback` 其他→en 回退逻辑可单元验）+ **L2**（真机切系统语言为 id/en/第三语言，验首启分别落 id/en/en）。脚手架已在 1.1，本 Story 验回退分支正确。

### AC2 — 手动切换即时生效 + UGC 不受限

**Given** 用户在「**我的 → 右上角设置图标 → 设置页 → 语言设置**」
**When** 手动切换语言
**Then** 即时生效无需重启；UGC 内容不受语言设置限制可用任意语言发布（FR-27）
> 🔄 **PRD V1.0.0 修订（F8 · 2026-06-08）：** 语言切换入口路径层级 +1——由「我的 → 语言设置」改为「我的 → 右上角设置图标 → 设置页 → 语言设置」（依赖 7-1 新设置页）；功能逻辑（双语/跟随设备/即时生效/UGC 不受限）不变，仅入口路由调整。
> 验证层：**L0**（切换写 `core/storage/prefs` 持久化 + 覆盖设备语言、`localeProvider` 驱动全树重建可 widget test；入口路由「我的 → 设置页 → 语言设置」层级正确可 widget test）+ **L2**（真机切 id↔en 全 App 即时换文案无需重启、杀进程重启后保持上次选择；用印尼语 UI 下发英文帖子内容正常显示——UGC 不被改写）。

### AC3 — 全 App 文案双套收口 + microcopy 双语库

**Given** 全 App 文案
**When** 收口双语
**Then** 所有 UI 文案/系统提示/错误信息均提供 id + en 两套（NFR-11），按场景沉淀 Voice & Tone microcopy 双语库（空状态/分析中/分享引导/红色预警/免责声明等），规则：每条最多 1 emoji、问诊结果页克制不用感叹号、红色预警简短无歧义（UX-DR14）
> 验证层：**L0**（`app_en.arb`/`app_id.arb` 两套 key **完全对齐**无缺漏、`flutter analyze`/自定义 lint 扫无写死字符串、microcopy 规则用脚本/单元校验：每条 ≤1 emoji、问诊结果页 key 无 `!`/`！`、红色预警文案长度受限）+ **L1**（后端错误信息按 `Accept-Language` 或前端按 ProblemDetail `type` 映射 id/en，需起栈触发各类错误验文案）。

## Dev Notes

### 关键架构约定（本 Story 必须落实）

- **i18n 栈**：`flutter_localizations` + `intl` + `.arb`（id/en），跟随设备回退英语、即时切换（架构 §Frontend）。脚手架已在 **Story 1.1**——本 Story 复用，不重建。
- **目录**：前端 `lib/core/l10n/`（生成产物）+ `lib/l10n/{app_en.arb,app_id.arb}`（文案源）+ `core/storage/prefs`（语言偏好持久化，`shared_preferences`，非敏感）；语言设置页在 `features/me/presentation`。
- **错误本地化**：优先**前端按 ProblemDetail `type` 映射**（后端语言无关），dio 拦截器统一转本地化文案（架构 §Process Patterns）。`Accept-Language` 头可驱动后端 `detail` 作为补充。
- **状态**：`localeProvider` 驱动 `MaterialApp.locale`，Riverpod 不可变；切换触发全树重建。

### ⚖️ 合规攸关约束

- **PDP「真实隐私政策 + 目的告知」可读（NFR-8）**：隐私政策 / 服务条款 / 免责声明 / 注销说明等**合规文案必须 id+en 双套且语义等价**——不得一种语言完整、另一种缺译或语义偏移。免责声明（前置可留证，附录 C）双语文案在本 Story 收口，与登录页前置同意（FR-0D）、问诊免责前置一致。
- **红色预警文案不得因翻译产生歧义（安全攸关）**：红色等级预警（FR-3）的 id/en 双语文案必须**简短、无歧义、无修辞**——翻译错误可能误导紧急医疗判断。红色预警文案变更须经守门规则校验，且与"安全规则层只升不降""红色页零变现"语义一致（架构 §安全规则层）。
- **UGC 不本地化、不翻译、不改写**：前后端均不得按语言设置处理用户原创内容——保证表达完整性与可追溯。

### 无障碍（NFR-13 / UX-DR15）—— 文案侧配合，门槛验收在 7.4

- microcopy 须支持动态字体缩放（body ≤3 级），文案长度变化下不破布局——长文案（尤其印尼语通常更长）须验不溢出/不截断关键信息。
- 红色预警文案配 7.4 的 assertive 播报：文案本身须可被屏幕阅读器清晰朗读、无依赖颜色/图标才能理解的措辞。

### 强制护栏

- **零写死字符串**：所有面向用户文案走 `.arb`；`flutter analyze` + 自定义检查守门。
- **两套 key 完全对齐**：`app_en.arb`/`app_id.arb` key 集合必须一致，缺一即漏译，CI 应失败。
- microcopy 规则（≤1 emoji / 问诊页无感叹号 / 红色预警简短）以可执行测试固化，**不靠人工自觉**。
- 后端绝不在 ProblemDetail `detail` 内拼写死中文/混语；UGC 不经语言处理。

### 范围边界（本 Story 明确不做）

- ❌ 不重建 i18n 脚手架（`l10n.yaml`/delegates/supportedLocales 在 1.1）——只补语言设置页 + 逐屏收口 + microcopy 库。
- ❌ 不新增第三语言（V1 仅 id/en）。
- ❌ 不实现「我的」页面其余区块（7.1）、注销逻辑（7.3）、无障碍审计门槛（7.4）。
- ❌ 不翻译/改写任何 UGC。
- ✅ 只做：语言设置手动切换+持久化 + 回退分支 + 全 App 逐屏文案双套收口 + Voice & Tone microcopy 双语库 + 错误文案本地化映射。
