# Story 3.8: 他人迷你主页预览卡

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a **用户**,
I want **点击他人头像看到一张迷你主页卡**,
so that **我能感知对方身份，同时平台为未来主页留出升级位**。

> 本 Story 实现他人迷你主页预览卡（FR-26）——点他人头像/昵称从底部弹卡（头像/昵称/发布数 +「主页筹备中」措辞），无关注/查看主页按钮。**前端重、后端轻**：后端提供迷你主页投影端点（昵称/头像/发布数）；前端 bottom sheet 弹卡 + 措辞克制（不得技术性）+ 已注销作者不触发。
>
> **做**：① 点 Feed/详情他人头像或昵称 → 底部弹迷你主页卡（bottom sheet，不跳新页）；② 卡含头像+昵称、已发布内容数（「TA 发布了 X 条内容」）、文案「TA 的专属主页正在精心筹备中，更多精彩即将呈现 ✨」、关闭按钮；③ 无「关注」「查看主页」按钮；④ 措辞**不得**出现「功能开发中/敬请期待/暂不支持」等技术性表达；⑤ 已注销用户头像点击**不触发**迷你卡。
> **不做**：真实他人主页（V1 仅「筹备中」占位）；关注关系（V1 无关注）；自己头像点击（不弹卡或弹自己——按 FR-26 仅"他人"，自己头像行为本 Story 不涉，默认不弹）。
> **依赖前序**：Story 3.2（Feed 卡片作者头像/昵称 + 已注销作者头像不可点的匿名化投影）、Story 3.3（详情页作者信息区点击触发位）、Story 1.3（用户身份与发布数统计）、Story 7.3（注销匿名化——已注销作者投影「已注销用户」，本 Story 据此不触发）。

## Acceptance Criteria

> **验证层标注**：**L0**（编译/lint/widget test/Service 单元 + MockMvc） · **L1**（需 postgres+redis，验迷你主页投影 + 发布数统计 + 已注销过滤） · **L2**（真机，bottom sheet 弹出/手势关闭）。
> 投影/发布数/注销过滤为 **L1**；bottom sheet 布局/措辞/无关注按钮为前端 **L0 + L2**。无外部第三方依赖。

### AC1 — 点头像/昵称弹迷你主页卡（措辞克制，无关注/查看主页）

**Given** 用户在 Feed/详情页点击他人头像或昵称
**When** 触发预览
**Then** 从底部弹出迷你主页卡（bottom sheet，不跳转新页），含头像+昵称、已发布内容数（「TA 发布了 X 条内容」）、文案「TA 的专属主页正在精心筹备中，更多精彩即将呈现 ✨」、关闭按钮（FR-26）
**And** 不展示「关注」「查看主页」按钮，文案不得出现「功能开发中/敬请期待/暂不支持」等技术性表达
> 验证层：发布数统计 + 迷你主页投影端点为 **L1**（`GET` 返回 nickname/avatarUrl/postCount；postCount = 该用户未软删的公开内容数）；bottom sheet 自底弹（300ms spring，UX-DR11）/ 头像昵称/发布数文案 /「筹备中」措辞 / **无关注/查看主页按钮** / 措辞非技术性为前端 **L0 widget test**（断言不含禁用词）+ **L2 真机**（弹出/拖拽关闭 UX-DR12）。

### AC2 — 已注销作者不触发迷你卡

**Given** 已注销用户的内容作者
**When** 用户点击其「已注销用户」头像
**Then** 不触发迷你预览卡
> 验证层：**L0**（前端：作者投影标记为已注销（avatarUrl=默认占位 + nickname=「已注销用户」或 `isDeactivated` 标志）时，头像点击**不触发** bottom sheet）+ **L1**（后端：对已注销用户的迷你主页投影端点返回 404 或带 `isDeactivated`，前端据此不弹）。承接 NFR-8 注销匿名化。

## Dev Notes

### 关键架构约定（本 Story 必须落实）

**迷你主页卡模型**（FR-26）：点他人头像/昵称底部弹卡（头像/昵称/发布数 +「主页筹备中」文案）；**无关注/查看主页按钮**；**措辞不得出现技术性表达**。这是产品体验细节——「筹备中」要让用户感到"未来有更多"而非"功能没做"，措辞克制是验收点（widget test 断言禁用词）。

**已注销不触发**（NFR-8 + Story 7.3）：注销用户内容匿名化保留在 Feed，但点击其「已注销用户」头像**不触发** FR-26 迷你卡（已注销无主页可预览）。本 Story 据投影 `isDeactivated` 或前端匿名标记拦截。

**模块归属与数据边界**：迷你主页投影在 `profile` 或 `auth`（用户基本信息）；postCount 经 `content` service 获取（**禁直接 join content 表**，跨模块经 service，架构 §数据边界）。

**命名映射链**：`GET /api/v1/users/{userId}/mini-profile`；`MiniProfileResponse`(nickname/avatarUrl/postCount/isDeactivated)；DB snake_case ↔ JSON camelCase。

**前端**（架构 §Frontend）：bottom sheet 自底 300ms spring（UX-DR11）、拖拽/点背景关闭（UX-DR12）；`shared/widgets/mini_profile_sheet`（本 Story 首建该共享 widget）；文案双语 .arb（UX-DR14 措辞规则）。

### 强制护栏（架构 §Enforcement —— 违反即返工）

- **禁止擅自引入 MQ / 通用缓存层 / 新中间件**；投影直读 PostgreSQL（postCount COUNT，无缓存）。
- 模块间只经 service——postCount 经 content service，**禁 profile/auth 直接访问 content repository**。
- 已注销用户**不暴露个人身份信息**（NFR-8）；不触发迷你卡。
- 措辞克制：**前端文案禁用技术性表达**（功能开发中/敬请期待/暂不支持等），双语 .arb，每条 ≤1 emoji。
- 无关注关系、无查看主页（V1 无主页）——端点与 UI 均不含关注/主页帖字段。
- 对外标识：`userId` 在已授权读取场景可用数字 id（迷你主页是公开读取，可用 id；不涉枚举敏感数据）。

### 范围边界（防 scope creep —— 本 Story 明确不做）

- ❌ 真实他人主页 / 主页帖列表（V1 仅「筹备中」占位）。
- ❌ 关注关系 / 关注按钮 / 查看主页按钮（V1 无关注）。
- ❌ 自己头像点击行为（FR-26 仅"他人"，本 Story 默认不弹）。
- ❌ 注销匿名化机制本身（Story 7.3）——本 Story 据其标记不触发。
- ✅ 只做：迷你主页投影端点（nickname/avatar/postCount/isDeactivated）+ bottom sheet 弹卡（措辞克制，无关注/查看主页）+ 已注销不触发。
