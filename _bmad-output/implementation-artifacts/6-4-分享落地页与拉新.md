# Story 6.4: 分享落地页与拉新

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

> **所属**：V1.1 Epic 6 收尾 Story（FR-49C 分享拉新）。**薄 Story——大量复用既有基建**：`/p/{cardToken}` Thymeleaf 落地页 + 失效态 + 下载双 CTA 早在 **Story 2.6** 上线（`CardPageController`，已测），分享链接助手 `petCardShareUrl` + 分享范式 `shareFn` 早在 **Story 2.7** 落地（growth archive 分享 FAB）。本 Story 交付：**身份证页（6-2）加「分享」入口**（分享 `/p/{cardToken}` 拉新链接）+ 复用既有落地页/失效态/下载 CTA。
> ⚠️ **真·deferred deep link 第三方服务（Branch / Firebase Dynamic Links 等，安装后归因直达）明确后置**（sprint-status 决策：「Deferred Deep Link 服务选型(6-4 拉新，可后置)」）；本 Story 只做「已装 App-link 直达 / 未装落地页下载」的基础链路 + 分享入口，归因型 deferred 留后续。

## Story

As a **用户**,
I want **分享身份证链接、他人点开直达或走落地页拉新**,
so that **传播带来新用户（FR-49C）**。

## Acceptance Criteria

> **验证层**：**L0**（分享入口 widget 测 + 链接拼接单测）· **L1**（落地页已删/失效态——**已由 2.6 `CardPageControllerTest` 覆盖，本 Story 复用不重造**）· **L2**（真机拉新链路：装/未装点开、App-link 直达、下载引导）。

### AC1 — 身份证页分享入口（分享 `/p/{cardToken}` 拉新链接）

**Given** 身份证详情页（6-2）有档案 + `cardToken`
**When** 点「分享」
**Then** 经系统分享面板分享 `petCardShareUrl(cardToken)`（`s.tailtopia.id/p/{token}`，不可枚举 token，绝不暴露顺序 id）+ 身份证风味文案；iOS 传 `sharePositionOrigin`（照 2.7/memory `ios-share-needs-sharePositionOrigin`）
**And** 无 `cardToken` / 无档案 → 不渲染分享入口（不空指针）
> 验证层：**L0**（分享入口在有 cardToken 时渲染 + 点击调 `shareFn` 传正确 URL 的 widget 测；`petCardShareUrl` 拼接单测已存在于 2.6/2.7，复用）。

### AC2 — 已装直达 / 未装落地页引导下载（复用 2.6 落地页）

**Given** 分享链接 `/p/{cardToken}` + 既有 Thymeleaf 落地页（`CardPageController`）
**When** 已装 / 未装 App 点开
**Then** 已装 → App-link 直达 App 内页（既有 `tailtopia://card/{token}` 深链 → `/profile`）；未装 → 落地页（成长故事 6 区块）+ 下载双 CTA（iOS/Android）引导下载
> 验证层：**L2**（真机装/未装点开 + App-link 直达 + 下载引导——**既有基建，本 Story 不改后端**）。真·deferred 归因（安装后直达）留第三方服务后置。

### AC3 — 目标档案已删 / token 失效 → 失效态（非 500）

**Given** 分享链接指向已注销账号 / 不存在 token
**When** 点开落地页
**Then** 统一友好失效页（gone）+ noindex（防枚举），**非 500**
> 验证层：**L1**（**已由 Story 2.6 `CardPageControllerTest` 覆盖**：token 缺失 / 账号注销 → 404 友好页 + noindex，本 Story 复用不重造）。

## Dev Notes

### 复用既有基建（勿重造）

- **落地页 `/p/{cardToken}`**：`profile/web/CardPageController`（Story 2.6，Thymeleaf 直出，公开无鉴权）——6 区块成长故事 + 双 CTA 下载 + 失效态（token 缺失/注销 → gone 404 友好页 + noindex 防枚举）。本 Story **不改**。
- **分享链接助手**：`profile/domain/card_link.dart` 的 `petCardShareUrl(cardToken, {baseUrl=kH5BaseUrl})` → `s.tailtopia.id/p/{token}`。**非用户明确指令不得改默认子域**（memory `growth-archive-share-page`：s.tailtopia.id 待 CF 加 hostname；`petgo-app-base-url-default` 同约定）。
- **分享范式**：`profile/domain/share_service.dart` 的 `shareServiceProvider`（`shareFn(text, {sharePositionOrigin})`）——iOS **必传** origin（memory `ios-share-needs-sharePositionOrigin`：缺失 iPad 崩、iPhone 无反应）。照 growth archive 分享 FAB（Story 2.7）。
- **深链**：`tailtopia://card/{token}` → `/profile`（app_router 冷启动深链落点，已就绪）。「已装直达」= App-link。

### deferred deep link 后置（sprint 决策）

- 真·deferred deep link（安装后归因直达被分享的具体宠物）需第三方服务（Branch / Firebase Dynamic Links / Adjust）选型 + 接入——**明确后置**（sprint-status line：「Deferred Deep Link 服务选型(6-4 拉新，可后置)」）。本 Story 只做「已装 App-link 直达 / 未装落地页下载」基础链路 + 分享入口。归因型拉新数据待服务落地。

### 范围边界（本 Story 不做）

- ❌ 不改后端落地页 / 失效态 / 深链（2.6/6.1 已交付并测）。
- ❌ 不接入 deferred deep link 第三方服务（后置）。
- ❌ 不做 App-link（universal link / app link）平台配置（apple-app-site-association / assetlinks.json + intent filter）——属 L2 平台工程 + 需 s.tailtopia.id CF hostname 就位；本 Story 前端分享入口先行，平台侧留 L2/后续。
