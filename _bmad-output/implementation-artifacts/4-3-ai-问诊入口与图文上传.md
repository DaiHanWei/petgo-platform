# Story 4.3: AI 问诊入口与图文上传

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a **焦虑的宠物主**,
I want **从问诊模块进入 AI 分诊并上传照片与症状描述**,
so that **我能快速发起一次免费的初步分诊**。

> 本 Story 交付 Epic 4 的**用户入口与上传链路**——问诊 Tab 双入口卡（AI 与兽医平级、均免费，V1 无支付）、AI 上传页（≤3 张 JPG/PNG/HEIC，单张 ≤10MB + 文字症状，STS 直传私密桶）、等待态全屏 spinner、超时（>15s）与服务异常重试（复用上次提交不重传）+ 软引导兽医。本 Story 把 Story 4.1 的分诊契约接到真实用户 UI 上。
>
> 前置依赖：Story 4.1（`POST/GET /triage` 契约 + `triageResultProvider` 薄层）、Story 2.1（媒体基建——STS 直传私密桶 + 相机/相册权限模式），Story 1.x（Tab Bar 外壳 / 门控 / i18n / 设计 token）必须就位。本 Story **前端为主**（入口卡 + 上传页 + 等待/超时/异常态）；后端仅复用 4.1 契约与 2.1 STS，不新增端点。

## Acceptance Criteria

> **验证层标注**：每条 AC 末尾标注验证层级与所需本地环境——
> **L0 静态**（widget/lint/纯逻辑） · **L1 集成/运行时**（需 Docker daemon + postgres + redis，跑通 4.1 契约 + 打桩 Gemini） · **L2 端到端/外部凭证**（需真实 Gemini API key + STS/私密桶签名 URL + 真机：相机/相册权限、HEIC 拍照、真实直传）。
> 上传/权限/真机相机与真实 STS 直传 + 真实 Gemini 端到端落 **L2**；入口卡布局、字段校验、超时/异常态 UI 切换可在 **L0/L1**（打桩）验证。

### AC1 — 问诊 Tab 双入口卡（AI 与兽医平级、均免费）

**Given** 登录用户进入问诊 Tab
**When** 查看问诊首页
**Then** 展示双入口卡（「AI 智能分诊」+「在线兽医」平级，均标注「免费」；兽医卡在 Epic 5 接入前为占位）（FR-4A）
> 验证层：**L0**（widget test：两卡平级布局、均显「免费」、兽医卡占位且点击不进入未实现流）；未登录点问诊 Tab 触发 FR-0C 强登录引导（复用 Story 1.4/1.5 门控）= **L1**（需 auth）。

### AC2 — AI 上传页（≤3 图 JPG/PNG/HEIC ≤10MB + 文字 + 权限）

**Given** 用户点击 AI 智能分诊
**When** 进入上传页
**Then** 可上传 ≤3 张图（JPG/PNG/HEIC，单张 ≤10MB）+ 文字症状描述，V1 仅图文无视频（FR-1 媒体规格）
**And** 点击上传按场景触发相机/相册权限（复用 Story 2.1 权限模式）
> 验证层：图片数量上限/格式/单张 ≤10MB 客户端预校验 + 文字输入 = **L0**；STS 直传私密桶 + 真机相机/相册权限申请 + HEIC 拍照真实落桶 = **L2**（需 STS 凭证 + 真机）。

### AC3 — 提交等待态（全屏 Morandi blue spinner + 文案）

**Given** 用户提交分诊
**When** 等待结果
**Then** 显示全屏居中 Morandi blue spinner + 「正在为你的毛孩子分析，稍等一下～」（UX-DR9）
> 验证层：**L0**（提交后 `AsyncValue.loading` → 渲染全屏 spinner + 双语文案，引用主题 token 莫兰迪蓝 #7BA7BC，无硬编码）。

### AC4 — 超时（>15s）与服务异常重试（复用上次提交不重传）+ 软引导兽医

**Given** 用户提交分诊
**When** 等待结果
**And** 超时（>15s）显示「分析时间较长，请稍后重试」+「重新提交」；服务异常显示「AI 服务暂时不可用」+「重新提交」+ 软引导「或直接联系兽医」；重新提交复用上次内容不需重传（FR-1）
> 验证层：超时态/异常态文案与按钮 + "重新提交复用上次内容（已传图的私密桶对象 key 与文字保留、不重传图）" = **L0/L1**（打桩 Gemini 制造超时/FAILED → 验前端两态切换 + 重提交携带原 imageObjectKeys/symptomText）；真实超时观察落 **L2**（不可控触发，不强制）。

## Dev Notes

### 关键架构约定（本 Story 必须落实）

**分诊数据流（架构 §Integration Points 数据流-1）**：Flutter 压缩图 → **STS 直传②私密桶** → `POST /triage`(202+id) → …（4.1/4.2 后端）→ 客户端**短轮询**取结果。本 Story 实现该流的客户端首段（入口 + 上传 + 提交 + 等待/降级）。

**媒体（架构 §Data Architecture / Story 2.1）**：分诊图 = **私密桶**（仅短 TTL 签名 URL，本 Story 客户端只持有对象 key 不持签名 URL）；客户端 STS 直传不经后端；上传前压缩 ≤10MB；EXIF GPS 剥离（2.1 链路）。

**前端约定（架构 §Frontend）**：Riverpod `AsyncValue<T>` 三态（loading/data/error）；副作用进 provider；ProblemDetail→本地化文案；i18n .arb（id/en）即时切换；引用 `core/theme` token（莫兰迪蓝 #7BA7BC 问诊区），无硬编码色值/文案。

**可靠性降级（NFR-10）**：超时/异常重试，上传失败仅重传失败件，文字 session 内存保留无持久草稿；分诊提交带 Idempotency-Key。

### 🔒 安全攸关约束（不可协商）

> 本 Story 是用户进入生命安全分诊的**第一触点**，以下约束保障分诊不被入口层削弱。

- **免责声明前置可留证（NFR-9）**：上传/提交流程中前置展示 AI 分诊免责声明（小号字不干扰主流程），可留同意/告知记录——为后续 4.4 结果页免责承接打基础（本 Story 至少在上传页或提交前承载一次免责告知）。
- **降级必有兜底，不让用户卡死**：AI 超时/异常**必须**给出"重新提交"+"软引导兽医"出口（NFR-10/NFR-9），绝不出现无出口的死等或白屏（参 UX-DR18 状态完整性）。软引导兽医为软性、非红色态（红色态零变现/零兽医引流是 4.5 的约束，与此处普通失败软引导不冲突）。
- **私密图只走签名 URL / 对象 key**：分诊图直传私密桶；客户端只持对象 key，不缓存/不日志签名 URL；图内容不落本地明文持久化。
- **重提交不绕过限流/幂等**：重新提交仍走后端写端点限流 + Idempotency-Key（4.1），不在前端构造绕过路径。

### 强制护栏（架构 §Enforcement —— 违反即返工）

- 上传只走 2.1 STS 直传私密桶；不经后端中转图片；不引入新上传中间件。
- 引用主题 token / .arb 文案，无硬编码色值/字符串。
- 不在前端做 danger_level 判定/降级（绿/黄/红一律以后端落库值为准，前端只按 `dangerLevel` 路由——尤其红色不可被前端"提前判绿"绕过 4.2/4.5）。
- 文字仅 session 内存，无持久草稿；上传失败仅重传失败件。

### 范围边界（防 scope creep —— 本 Story 明确不做）

- ❌ 不做分诊后端/Gemini/状态机/安全规则层（**4.1/4.2**）。
- ❌ 不渲染绿/黄结果三态卡 / 黄色条件倒计时协议块（**4.4**）——DONE 后留交棒位。
- ❌ 不做红色半屏 overlay / 导航 / 无障碍播报（**4.5**）——红色结果留触发位。
- ❌ 不实现在线兽医真实入口（**Epic 5**）——兽医卡为占位、软引导为占位导向。
- ❌ 不做存档（FR-16，**2.5/4.4**）。
- ✅ 只做：问诊 Tab 双入口卡 + AI 上传页（图文校验 + STS 直传私密桶 + 权限）+ 等待 spinner + 超时/异常/降级态（复用上次提交不重传 + 软引导兽医）+ 免责前置承载 + 结果交棒占位。
