# Story 9.1: 后台权限点扩充与导航整改

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

> **所属**：V1.1 Epic 9（后台资金与运营治理）**开篇** Story。**brownfield 扩既有 Thymeleaf admin slice**（`com.tailtopia.admin`）。为 9-2~9-10 铺权限地基 + 清理导航。交付：① `AdminPermissions` 扩充 V1.1 新权限码（订单查看/导出、内容导出、举报者清单、虚拟账号、配置类）；② 补 Epic 4 遗漏的 `support.handle`/`refund.*` 权限 i18n 标签；③ 导航去重（「问诊异常」双入口 → 单入口）+「会话查询」→「历史会话查询」+ 兽医分组标签去除过时「（Epic 5）」。
> **关键事实**：后台权限码是**编译期常量**（`AdminPermissions.ALL`）+ `perm.<code>` i18n，`admin_account_permissions` join 表存 varchar 码**无 CHECK 约束** → **加权限码无需 Flyway 迁移**（纯代码 + i18n）。勾选 UI（`admin-accounts.html`）用 `th:each="p:${allPermissions}"` 自动渲染，加码即自动上架。
> **门控范式**：导航 `sec:authorize="hasRole('SUPER_ADMIN') or hasAuthority('<code>')"`；端点 `@PreAuthorize`。SUPER_ADMIN 隐式全权（不注入全集）。**9-2~9-10 的端点门控在各自 story 引用本 story 定义的码**——本 story 只定义码 + 上架勾选 + 导航整改，不建新端点/页面。

## Story

As a **超管**,
I want **新增细粒度权限点、整改兽医相关导航**,
so that **后台权限覆盖 V1.1 新功能、导航不重复（AB-1.1-29）**。

## Acceptance Criteria

> **验证层**：**L0**（`AdminPermissions.ALL`/`isValid` 单测、`mvn compile`）· **L1**（勾选/授权 STAFF 新码落库、既有端点门控回归、Thymeleaf 渲染无缺 key）。无 L2（后台无移动端视觉）。

### AC1 — 新权限码上架且可授予 STAFF

**Given** `AdminPermissions` 新增：`order.view`/`order.export`（兽医·AI 咨询订单查看/导出，9-3/9-4）、`content.export`（内容导出）、`content.view_reporters`（举报者清单）、`virtual_account.manage`（虚拟账号，9-8）、`config.view`/`config.edit`（定价/PawCoin/阈值配置，9-2/9-6）；退款三码 `refund.submit`/`refund.approve`/`refund.payout` 已在 Epic 4 存在（不重复加）
**When** SUPER_ADMIN 在账号页勾选新码授予某 STAFF 并保存
**Then** 新码进 `admin_account_permissions`；`AdminPermissions.isValid` 认可；勾选 UI 顺分组渲染新码（每码有 `perm.<code>` zh/en 标签，不露原始码）
> 验证层：**L0**（`ALL` 含新码 + `isValid` 真/假分支）+ **L1**（授予新码 → 落库 + 回读账号带该码）。

### AC2 — 补齐 Epic 4 遗漏的权限 i18n 标签

**Given** `support.handle`/`refund.submit`/`refund.approve`/`refund.payout` 常量已存在但**缺 `perm.*` i18n**（勾选 UI 现显原始码）
**When** 渲染账号权限勾选/标签
**Then** 四码均有 zh/en 标签
> 验证层：**L1**（`messages_zh_CN`/`_en` 含四 `perm.*` key；渲染无 `perm.support.handle` 原样字面）+ 人工核对。

### AC3 — 导航去重 +「会话查询」改名 + 兽医分组标签整改

**Given** `layout.html`：「问诊异常」当前双入口（兽医分组下一处 + 独立「问诊异常」分组一处，指向同一 `/admin/anomalies`）；「会话查询」；兽医分组标签「兽医（Epic 5）」
**When** 渲染侧导航
**Then** 「问诊异常」仅保留独立分组单入口（删兽医分组下重复项）；`admin.nav.sessions`「会话查询」→「历史会话查询」（zh）/「Session History」（en）；兽医分组标签去过时「（Epic 5）」
> 验证层：**L1**（渲染后 `/admin/anomalies` 链接在导航仅 1 处；nav 出现「历史会话查询」不出现「会话查询」）+ 人工核对。

### AC4 — 既有端点/导航门控零回归

**Given** 既有权限门控（`user.view`/`vet.view`/`consult.view_sessions`/`refund.*` 等）
**When** STAFF 无/有对应权限
**Then** 既有 `@PreAuthorize`/`sec:authorize` 行为不变（无权 403/导航不显，有权可达）
> 验证层：**L1**（既有 `AdminAccountAccessControlTest` 等回归全绿）。
