# Story 1.5: 充值失败/取消与暂停态

Status: review

> V1.1 Epic 1（资金地基）第 5 story，接已 done 的 1.1-1.4。**全栈**（后端补两个只读端点 + 前端充值链路）。承接 1.4 余额页「Isi Saldo」按钮（已接线 `/me/pawcoin/recharge`）。**brownfield**：**无新 Flyway**（复用 1.1/1.2 表）。
> 源：`epics-v1.1.md` Story 1.5 · UX `EXPERIENCE.delta.md`（`p-pawcoin-recharge` / `p-recharge-fail` / `p-pawcoin-pause`）+ `DESIGN.delta` · 架构 §2/§3.1/§8。
> **范围边界**：充值档位页 + 支付载荷渲染 + **轮询检测** + 成功/失败/取消/暂停四态。**复用** 1.3 已建的 `POST /me/pawcoin/topups`。**不做**兽医限时支付的倒计时暂停顺延（那是 Story 3.4）、后台档位配置（Story 9.2）。

## Story

As a 用户,
I want 选档位充值 PawCoin，并在支付失败/取消/服务暂停时得到清楚不吓人的反馈,
so that 我能安心充值，失败也知道钱没变、暂停也知道不是出 bug（FR-50A / FR-43F）。

## Acceptance Criteria

1. **后端：充值选项端点（含暂停 flag）（L0/L1）**
   **Given** `GET /api/v1/me/pawcoin/topup-options`（JWT `role=user`）
   **Then** 返回 `{ tiers: [{id, amount, coins}...], paused }`；`tiers` 来自 `TopupTierProvider`（10k/25k/50k/100k，1:1）；`paused` 来自 env `PAWCOIN_TOPUP_PAUSED`（默认 false，`PayProperties.topupPaused`）——**AB-6C：后端不做一键硬暂停，仅暴露运营/工程手动设的 env flag 供前端渲染**（**L0/L1**）

2. **后端：支付状态轮询端点（L0/L1）**
   **Given** `GET /api/v1/me/pawcoin/topups/{token}/status`（JWT）
   **Then** 按 `payment_intents.public_token` 查回 `{ status }`（`PENDING`/`PAID`/`FAILED`/`EXPIRED`）；**仅本人意图**（`intent.userId == 当前 sub`，否则 404，防越权枚举他人 token）（**L1**）

3. **前端：充值档位页（L0）**
   **Given** `/me/pawcoin/recharge`（`p-pawcoin-recharge`，shell 外顶层隐 Tab）
   **When** 进入（GET topup-options）
   **Then** 渲染 4 档位网格 + 渠道选择（QRIS/DANA）+ 主按钮；**若 `paused=true` → 渲染暂停态（AC6），不渲染档位**（**L0**）
   **And** 选档位+渠道 → `POST /me/pawcoin/topups`（**前端生成 `Idempotency-Key` UUID**，1.3 强制该头）→ 拿 `TopupResponse`（**L0/L1**）

4. **前端：支付载荷渲染 + 轮询到账（L0）**
   **Given** `TopupResponse.payload`（QRIS=二维码图 URL / DANA=deeplink）
   **When** 展示支付
   **Then** QRIS → `Image.network(payload)` 渲染二维码（stub 载荷显占位）；DANA → 「Buka DANA」按钮 `url_launcher`（**L0**）
   **And** **轮询** `GET .../topups/{token}/status` 每 ~3s；`PAID` → 成功 toast + 返回余额页并**刷新**（invalidate `pawCoinProvider`）；**轮询 Timer 在 dispose/终态必取消**（防 "Timer still pending"）（**L0**）

5. **前端：充值失败/取消态（L0，FR-43F）**
   **Given** 轮询返回 `FAILED`/`EXPIRED`，或用户主动返回（取消）
   **Then** 渲染失败态「Saldo tidak berubah, tidak ada catatan」（**余额不变、不落流水**——后端 1.3 保证失败回调不 credit）；提供「重试」返回档位页（**L0**）

6. **前端：浮存门槛暂停态（L0）**
   **Given** `paused=true`（运营/工程手动触发）
   **Then** 渲染 `p-pawcoin-pause`：「Isi Saldo Sementara Tidak Tersedia … Saldo yang sudah ada tetap aman」——**措辞须区别于支付失败**（不能读成 bug，齿轮/暂停语气非 error）（**L0**）

7. **双语 l10n（L0）**
   **Then** 档位/渠道/支付/成功/失败/暂停文案 id+en 双套 + `flutter gen-l10n`；**源码零硬编码用户可见串**（**L0**）

8. **端到端视觉（L2）**
   **Given** Android 模拟器 + 本地后端（stub 网关）
   **Then** 充值页→选档位→stub 二维码→（手工触发 stub 回调 settlement）→轮询到 PAID→返回余额页余额增加；`PAWCOIN_TOPUP_PAUSED=true` 时渲染暂停态（**L2**，留本地）

### 后端要点
- **暂停 flag（AB-6C）**：`PayProperties.topupPaused`（env `PAWCOIN_TOPUP_PAUSED`，默认 false）。**后端不实现一键硬暂停**——仅暴露此 env flag，运营/工程改 env 重启触发；前端渲染暂停态。浮存门槛监控/告警是 Story 9.x，本 story 只渲染 flag。
- **状态轮询**：`GET /me/pawcoin/topups/{token}/status` → `findByPublicToken` → **校验 `intent.getUserId() == currentUserId`**（不符/不存在 → `AppException.notFound`，别用 forbidden 以免泄漏 token 存在性）→ 返回 `{status: PENDING|PAID|FAILED|EXPIRED}`。只读，无副作用（不主动 refreshFromGateway；到账靠 1.3 回调）。
- **档位端点**：`GET /me/pawcoin/topup-options` → `{tiers, paused}`；tiers 遍历 `TopupTierProvider`。`currentUserId(jwt)` 门控（登录即可，档位对所有用户一致）。照 1.4 `PawCoinController` 的 `@RequestMapping("/api/v1/me")` + `currentUserId`。
- **DTO 护栏**：延续——对外不暴露自增 id；status 端点只回枚举名。
- **无 Flyway**、**不动 SecurityConfig**（`/api/v1/me/**` 已需 JWT）。

### 前端要点（复用 1.4 `features/pawcoin/`）
- **档位/渠道选择**：GET topup-options → 4 档位卡 + QRIS/DANA chip（照 DESIGN.delta 克制风格，禁 emoji）。选中态用 `AppColors.mint` 边框 + `mintTint` 底。
- **下单**：`POST /me/pawcoin/topups`，前端**每次尝试生成一个 `Idempotency-Key`（UUID）**（1.3 强制该头，缺失 422）——用现有 uuid 依赖或 `DateTime`+随机；查 pubspec 是否有 uuid，无则用 `Object.hashCode`/时间戳组合（避免真随机不确定性影响测试；测试可注入固定 key）。
- **支付载荷**：`TopupResponse.payload` —— QRIS 为二维码图 URL → `Image.network(payload)`（照 `shared/widgets/app_image.dart` / consult 的 `Image.network` 用法，带 loadingBuilder/errorBuilder）；stub 载荷（`stub-qr://...`）非真图 → 显占位 + 提示「测试环境」。DANA 为 deeplink → 「Buka DANA」按钮 `launchUrl`（`url_launcher: ^6.3.1` 已有，照 `settings_page.dart`/`login_page.dart` 用法）。
- **轮询**：`Timer.periodic(Duration(seconds: 3), ...)` 调 `pollStatus(token)`；`PAID` → 成功（toast + `context.pop()` 回余额页 + `ref.invalidate(pawCoinProvider)` 刷新余额）；`FAILED`/`EXPIRED` → 失败态。**⚠️ Timer 必须在 `dispose()` 与到达终态时 `cancel()`**——记忆库 `story_1_5_gating` 的 "Timer still pending" 即常驻 timer 未清；本 story 若泄漏 timer，测试会红。用 `StatefulWidget` + `dispose` 清 timer，或 Riverpod `ref.onDispose`。
- **成功刷新**：回余额页后 `ref.invalidate(pawCoinProvider)` 让 1.4 余额页重拉（余额+流水更新）。
- **失败/取消态（FR-43F）**：文案「Saldo tidak berubah, tidak ada catatan」，warm 非 error 红；「重试」返回档位页。用户中途返回（返回键）= 取消，同样余额不变（后端未收到支付回调即不 credit）。
- **暂停态**：warm 齿轮空态「Isi Saldo Sementara Tidak Tersedia … Saldo yang sudah ada tetap aman」——**措辞刻意区别于支付失败**（EXPERIENCE.delta：避免读成 bug）。
- **路由**：`/me/pawcoin/recharge`（shell 外顶层，`_controlledLocations` 已含 `/me`，自动门控——**无需改门控集合**）；照 1.4 的 `/me/pawcoin` GoRoute。
