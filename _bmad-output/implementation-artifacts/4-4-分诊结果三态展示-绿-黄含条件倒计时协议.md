# Story 4.4: 分诊结果三态展示（绿/黄含条件倒计时协议）

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a **用户**,
I want **清晰看到分诊危险等级、观察建议与用药参考**,
so that **我能据此决定是否就医并执行居家处理**。

> 本 Story 交付 Epic 4 的**绿/黄结果展示**（红色态走 Story 4.5 半屏 overlay）——白底 + 左 3px 区域色边框 + triage badge；黄色「条件倒计时协议块」（accent-consult 浅底 #EEF4F7）含**具体观察指标 + 时间窗口 + 升级触发条件**、视觉可区分、**不得终结性表述**；同屏 ≤15s 返回三项（危险等级 + 观察建议 + 居家用药参考）+ 前置免责声明；非红结果可「存入档案」（触发 FR-16，承接 Story 2.5）。
>
> 前置依赖：Story 4.1（`GET /triage/{id}` 返回 `TriageResultResponse` 三态结构 + `triageResultProvider`）、Story 4.3（DONE 后按 dangerLevel 交棒）、Story 2.5（FR-16 存档弹窗承接）、Story 1.2（设计 token / triage 三色）。本 Story **前端为主**；后端仅确认 4.1 结果结构满足展示字段。

## Acceptance Criteria

> **验证层标注**：每条 AC 末尾标注验证层级与所需本地环境——
> **L0 静态**（widget/lint/文案规则纯逻辑） · **L1 集成/运行时**（需 Docker daemon + postgres + redis，跑通 4.1 契约 + 打桩 Gemini 控制绿/黄返回） · **L2 端到端/外部凭证**（需真实 Gemini + 真机，验真实结果同屏 ≤15s + 真机视觉/无障碍）。
> 三态视觉、黄色协议块结构、终结性表述禁用、免责前置可在 **L0/L1**（打桩 Gemini 喂构造的绿/黄结果）；真实结果同屏 ≤15s + 真机渲染/对比度落 **L2**。

### AC1 — 绿色结果页（暂无紧急风险 + 居家观察 + 软引导 + 存档 + 免责）

**Given** 分诊返回绿色
**When** 展示结果页
**Then** 绿色图标 + 「暂无紧急风险」+ 居家观察建议 + 软性引导「想更放心？可以咨询兽医确认」+ 底部「将本次结果存入档案」引导（FR-1 绿色定义）
**And** 结果页底部展示小号次要色免责声明，不干扰主内容（NFR-9）
> 验证层：**L0/L1**（打桩 Gemini 返回 GREEN → widget 验绿色 badge + 「暂无紧急风险」+ 观察建议 + 软引导兽医 + 存档入口 + 底部小号免责）；真机对比度（免责 ≥3:1）= **L2**。

### AC2 — 黄色结果页（三项同屏 + 条件倒计时协议块 + 禁终结性表述）

**Given** 分诊返回黄色
**When** 展示结果页
**Then** 同屏呈现危险等级 + 观察建议 + 居家用药参考；观察建议必须含具体观察指标、时间窗口、升级触发条件，以独立卡片/加粗视觉区分（accent-consult 浅底协议块）（FR-2、UX-DR6）
**And** 黄色页不得出现「不严重/可以放心」等终结性表述
> 验证层：**L0**（widget：黄色 badge + 三项同屏 + 协议块 accent-consult 浅底 #EEF4F7 + 协议块含"指标/时间窗口/触发条件"三要素且视觉可区分；文案守卫：渲染文案不含终结性词表）+ **L1**（打桩 Gemini 返回 YELLOW + 含三要素的 advice → 端到端落黄页）；真实黄色结果同屏 ≤15s = **L2**。

### AC3 — 非红结果「存入档案」触发 FR-16

**Given** 任意非红结果页
**When** 用户点击「存入档案」
**Then** 触发 FR-16 存档弹窗（承接逻辑见 Story 2.5）
> 验证层：**L0**（点击 → 调起存档弹窗回调）+ **L1**（端到端：绿/黄结果点存档 → 2.5 存档流写健康事件入时间线，含日期/症状/AI 评级/摘要；需 profile/2.5 就位）。

## Dev Notes

### 关键架构约定（本 Story 必须落实）

**结果三态视觉（UX-DR6，权威）**：绿/黄 = 白底 `rounded.md` + 左 3px 区域色边框 + triage badge；黄色观察协议块 = **accent-consult 浅底 #EEF4F7** `rounded.sm`；红色走半屏 overlay（**Story 4.5**，本 Story 不做）。

**黄色条件倒计时协议（FR-2）**：观察建议必须含**具体指标 + 时间窗口 + 升级触发条件**；不得终结性表述；时间窗口/触发条件视觉可区分。三要素优先由后端结构化产出（B1），前端按结构分区呈现。

**前端约定**：Riverpod `AsyncValue`；引用 `core/theme` token（triage 三色 + accent-consult #EEF4F7）无硬编码；i18n .arb（id/en）；ProblemDetail→本地化（403/404 多态，UX-DR18）。

**存档（FR-16 / 架构媒体桥接）**：存入时把私密图复制到私密桶留存，档案只引用应用自有 URL（不引用会过期的 IM/临时 URL）；健康事件含日期/症状/AI 评级/摘要。本 Story 触发，2.5 承接。

### 🔒 安全攸关约束（不可协商）

> 绿/黄页是用户对"是否就医"做决策的依据，措辞与免责直接关乎安全。

- **不得终结性表述（FR-2 / 信任边界）**：黄色页（含绿色观察文案）**禁止**「不严重/可以放心/没事了/无需就医」等终结措辞——无论来自固定文案还是模型产出。F4 的词表守卫为强制护栏，模型若吐出终结词必须被拦截降级，绝不原样渲染。
- **免责声明前置可留证（NFR-9）**：所有 AI 结果页附**前置**免责声明（小号字、次要色、不干扰主流程），可留同意/告知记录；绿/黄页均不可省略免责。
- **红色绝不在本页"软化"展示**：danger_level 由后端落库（经 4.2 只升不降）；本页**只处理绿/黄**，`RED` 一律交棒 4.5 半屏强提醒——**严禁**在结果卡里把红色降格为普通黄/绿卡或"温和提示"，绕过红色态强提醒。
- **存档不泄私密健康图**：存档走 2.5 桥接（私密桶 + 签名 URL），结果页不缓存/不日志签名 URL 与健康内容。
- **软引导兽医仅限非红**：绿/黄页的"咨询兽医确认"是软性引导；红色态零兽医/变现引流是 4.5 约束，二者不混用（本页不出现红色，不冲突）。

### 强制护栏（架构 §Enforcement —— 违反即返工）

- 引用主题 token / .arb，无硬编码色值/文案。
- 不在前端判定 danger_level，一律以后端落库值路由；红色不在本页渲染（交 4.5）。
- 终结性词表守卫不可省略；免责前置不可省略。
- 三态/失效/无权限/已删除多态完整（UX-DR18），不白屏不崩溃。

### 范围边界（防 scope creep —— 本 Story 明确不做）

- ❌ 不做红色半屏 overlay / 导航 / 无障碍 assertive 播报（**Story 4.5**）——红色结果交棒。
- ❌ 不做分诊后端/Gemini/安全规则层（**4.1/4.2**）。
- ❌ 不做问诊入口/上传/等待 spinner（**4.3**）——本 Story 从 DONE 绿/黄结果接手。
- ❌ 不实现存档后端写入（**2.5**）——只触发弹窗 + 传 triageId。
- ❌ 不接真实兽医咨询（**Epic 5**）——软引导为占位导向。
- ✅ 只做：绿/黄结果三态卡（白底+左3px边框+badge）+ 黄色条件倒计时协议块（#EEF4F7 三要素视觉可区分）+ 禁终结性表述守卫 + 前置免责 + 三项同屏 ≤15s + 非红「存入档案」触发 FR-16。
