# Story 8.4: 里程碑用户打卡（已打卡 / 去发布两路径）

Status: review

> 后端打卡 API（关联成长日历内容，一内容一里程碑不可重复）+ 前端内容关联选择器（置灰已关联）+
> 「去发布」跳转预选成长日历 + 发布成功自动完成回填。两路径收口同一 check-in API。

## Story
As a **用户**, I want **对「第一次洗澡」这类需手动记录的里程碑打卡**, so that **把真实发生过的成长时刻补记或新发布并点亮里程碑**。

## Acceptance Criteria
### AC1 — 「已打卡」内容关联选择器（FR-42）
点「已打卡」→ 仅本人已发布**成长日历**内容；选一条 → 标记完成、记录时间；关闭未选 → 无变更。**L0**。
### AC2 — 一内容一里程碑（置灰）
每条成长日历内容至多关联一个里程碑；已关联置灰不可选（后端 partial-unique + 前端置灰）。**L0**。
### AC3 — 「去发布」路径
点「去发布」→ 跳发布入口预选成长日历（仅跳转不预锁定）；放弃 → 未完成；发布成功（仍成长日历）→ 自动完成；
编辑切其它类型并发布 → 不自动完成。**L0**（导航 + 回填路径）。
