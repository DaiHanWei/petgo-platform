# Story 2.3: 统一「+」发布入口与内容发布

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a **用户**,
I want **通过单一「+」入口发布日常分享/专业科普/成长日历快乐时刻**,
so that **我能便捷地记录与分享，无需在多个入口间纠结**。

> 本 Story **首次建立 `content_posts` 表**（含 type/danger 等弹性字段），是 Epic 3 Feed/详情/互动的数据根，也是 2.4/2.6 时间线与名片读「成长日历快乐时刻」的来源。本 Story 只做"统一发布入口 + 三类内容发布 + 成长日历绑定宠物 + B/C 灰置 + 仅重传失败件 + Idempotency-Key"，**不做 Feed 流、不做详情、不做评论/点赞**（Epic 3）。
>
> **依赖前序**：1.1（脚手架/content 模块）、1.3（JWT）、1.6（用户状态 A/B/C，决定成长日历可选性）、2.1（图片压缩/直传公开桶/权限工具）、2.2（pet_profiles，成长日历绑定宠物）。

## Acceptance Criteria

> **验证层标注**：**L0 静态** · **L1 集成/运行时**（Docker+postgres+redis）· **L2 端到端/外部凭证**（真实 OSS/STS、真机相册）。
> 核心落 **L0/L1**；图片直传真实 OSS 已在 2.1 的 L2 覆盖，本 Story 用其产出 URL 落库（L1）。

### AC1 — 统一 Publish Compose 单页发布

**Given** 登录用户点击「＋」
**When** 打开 Publish Compose 全屏 bottom sheet（UX-DR16）
**Then** 同页内含内容类型 Segment（全部/日常分享/成长日历/科普）→ 图片上传区 → 文字输入区 → 发布按钮，类型选择不超过一步、无独立页面跳转（FR-12）
**And** 文字上限 1000 字符实时计数（如「剩余 800 字符」），超出禁止发布；图片每条 ≤9 张（单张 ≤10MB），V1 不支持视频
> 验证层：**L0**（bottom sheet 布局/Segment 单步切换/字数实时计数/≤9 张/≤10MB 前端校验 widget 测试；服务端 Bean Validation 1000 字/≤9 张 MockMvc）。「全部」为浏览态语义，发布时须落具体类型（见 Dev Notes）。

### AC2 — 成长日历绑定宠物 + B/C 灰置

**Given** 选择「成长日历快乐时刻」
**When** 状态 A 用户发布
**Then** 需绑定宠物，发布后该条进入宠物档案时间线（写 `content_posts` 表，本故事创建该表，含 type/danger 等弹性字段）
**And** 状态 B/C 用户的「成长日历」选项灰置不可选，悬浮提示「需要先创建宠物档案」+ 跳转创建入口
> 验证层：**L1**（成长日历 post 写库带 `pet_id` 关联 + type=GROWTH_MOMENT，2.4 时间线据此读取，需 postgres）+ **L0**（B/C 用户 Segment 灰置 + 悬浮提示 + 跳创建入口 widget 测试；服务端校验「成长日历必须有档案否则 422/409」）。

### AC3 — 仅重传失败件 + 内存草稿

**Given** 发布上传过程中网络中断/服务器错误
**When** 用户点击「重试」
**Then** 仅重传失败文件，已填文字在当前 session 内存保留；用户退出编辑页则内容清空（V1 无持久草稿）（NFR-10）
> 验证层：**L0**（上传状态机：每图独立 状态(待传/传中/成功/失败)，重试仅作用失败件；退出 sheet 清空内存草稿 — widget/provider 测试，用 mock uploader）。

### AC4 — 公开提示 + Idempotency-Key

**Given** 任意发布
**When** 提交成功
**Then** 内容默认全平台公开，发布确认页提示「发布后内容将对所有用户公开展示」，请求带 Idempotency-Key 去重
> 验证层：**L0**（确认提示文案 i18n + 客户端生成 `Idempotency-Key`）+ **L1**（同 Idempotency-Key 重复提交只落一条 — `IdempotencyService` + Redis）。

### AC5 — 成长日历事件日期字段（F9 日期口径分离） `[R2]`

> 🆕 **PRD V1.0.0 第二轮断档补齐（F9 · 2026-06-08）：** 成长日历「事件日期」与发布时间分离——`content_posts` 加列 `event_date date null`（仅 type=GROWTH_MOMENT 有值，日常/科普为 null）。事件日期决定档案侧显示位置，发布时间（`created_at`）决定 Feed / 「我的发布」排序。

**Given** 用户编辑 type=GROWTH_MOMENT 成长日历内容
**When** 在编辑页设置「事件日期」
**Then** Flyway 给 `content_posts` **加列 `event_date date null`**；成长日历类型编辑页显示「事件日期」字段，默认值按入口分流——从「＋」直接进入取**今天**、从日历格子进入取**该格子日期**（2.4 联动）
**And** 事件日期**不可选未来**、可选任意过去；选未来一律前端禁选 + 服务端 422
**And** 落库语义：仅 GROWTH_MOMENT 写 `event_date`，日常/科普 `event_date` 保持 null；`event_date` 决定档案侧时间线/日历显示位置，`created_at` 决定 Feed/「我的发布」排序（两者解耦）
> 验证层：**L1**（Flyway 加 `event_date` 列成功 + 成长日历 post 落 `event_date`、日常/科普为 null + 服务端拒未来日期 422，需 postgres）+ **L0**（事件日期选择器禁未来 widget + 入口默认值今天/格子日期分流 widget 测试）。

## Dev Notes

### 架构约定

- **命名**：表 `content_posts`(复数 snake_case，软删 `deleted_at`)、枚举 `type`/`danger_level` UPPER_SNAKE 落 varchar、`image_urls` JSONB（架构「分诊结果/档案弹性字段用 JSONB」）；实体 `ContentPost`、DTO record、provider。JSON camelCase / UTC ISO / null 省略。
- **模块边界**：成长日历绑宠物校验**经 profile service 接口或事件，禁 content 直接读 pet_profiles 表**（架构「禁跨模块访问对方 repository」）。
- **幂等**：发布带 `Idempotency-Key` → `shared/ratelimit/IdempotencyService`(Redis 防重锁)。
- **公开桶**：内容图进①公开桶 + CDN（复用 2.1）。
- **错误**：校验 422 / 成长日历无档案 422 或 409 / 限流 429，ProblemDetail；前端 dio→本地化（上传 toast、网络 banner 5s）。

### 强制护栏

- **禁 MQ/缓存层/新中间件**；Redis 仅幂等/限流。
- `ddl-auto=validate`，schema 只由本 Story Flyway。
- **无持久草稿**（NFR-10）：仅内存保留文字、仅重传失败件，**勿引入本地草稿持久化**。
- author_id/pet 归属取自 JWT + service 校验，不信任客户端。
- 日志记业务事件 id 可，**不落用户正文 PII 进结构化字段做检索**；本表无健康数据，但 danger_level 关联问诊时不外泄症状细节到日志。
- 软删 `deleted_at` 为级联删除/Epic3 删除做准备。

### 范围边界（不做）

- ❌ 不做 Feed 流/硬过滤/无限滚动（3.2）、详情（3.3）、点赞（3.4）、评论（3.5）、删除（3.6）、举报（3.7）。
- ❌ 不做时间线渲染（2.4 读本表）、名片读取（2.6 读本表）。
- ❌ 不做视频/收藏/@提及/搜索（V1 范围外）。
- ✅ 只做：`content_posts` 建表(含弹性字段) + 发布端点(幂等/限流) + Publish Compose 单页 + 三类型 + 成长日历绑宠物 + B/C 灰置 + 仅重传失败件 + 内存草稿 + 公开提示。
