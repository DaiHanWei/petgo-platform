# Story 6.3: 高清图付费下载

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

> **所属**：V1.1 Epic 6 第三个 Story（**全栈 · 支付集成**，安全攸关）。承接 6-1（`serial_id`/身份证数据接口）+ 6-2（KTP 渲染 + 1600×900 客户端导出）。交付：身份证高清图**一次性永久付费解锁**（Rp5,000，QRIS/PawCoin）+ 解锁后无限复下载 + 幂等不重复扣费。**HD 图本体由客户端渲染导出（6-2 RepaintBoundary→1600×900 PNG）**，后端只管**收费 + 记录一次性永久购买 + 解锁标志**。
> ⚠️ **资金四不变式（总账/幂等/非负/审计）等同安全规则层不可绕过**——绝不重复扣费、绝不重复解锁、失败态不解锁。范式**照抄 `AiUnlockService` + `AiUnlockPaidHandler`**（AI 一次性解锁，与 ID_HD 同构）。

## Story

As a **用户**,
I want **付费下载身份证高清图**,
so that **我能永久保存（FR-49D）**。

## Acceptance Criteria

> **验证层**：**L0**（编译/单测，无 DB）· **L1**（Docker pg+redis：真扣费/建购买/幂等/解锁）· **L2**（Midtrans sandbox 真购一次 QRIS）。

### AC1 — `id_card_hd_purchases` 一次性永久解锁（Flyway 迁移）

**Given** `id_card_hd_purchases`（一次性永久，**Flyway V75**——见 Dev Notes「Flyway 号」，非架构旧稿 V55）
**When** 付费 **Rp5,000**（`id-hd.download-price`，env 默认 5000；pricing_config 留 9-2）
**Then** 解锁后**可无限复下载**、**幂等**（同用户重复购买不二次扣费、不二次建行）
> 验证层：**L1**（真 pg：PawCoin 扣费建购买行、重复购买短路不再扣、`UNIQUE(user_id)` 兜底）+ **L0**（迁移 DDL + service 编译）。

### AC2 — 双渠道支付（QRIS / PawCoin），照抄 AI 解锁范式

**Given** 支付渠道 QRIS（Midtrans）/ PawCoin（钱包 1:1）
**When** 发起 HD 购买
**Then**
- **PawCoin**：`wallet.debit` 同事务原子扣费 → 立即建购买行 → 解锁（余额不足回滚，不解锁）
- **QRIS**：`PaymentIntentService.createIntent(ID_HD, QRIS, 5000, "id-hd:{petProfileId}")` → 返回支付信息；到账经 `PaymentIntentPaidEvent` → **新 `IdHdPaidHandler`（同步 `@EventListener` `MANDATORY`，同事务原子记购买，禁 AFTER_COMMIT）** → 解锁
**And** 已购买 → 入口短路直接返回「已解锁」，**不扣费不建行**（AC1 幂等）
> 验证层：**L1**（PawCoin 扣费+建购买；QRIS 建意图+回调到账建购买+双通道回调重放不双建）+ **L0**（service 分支 + handler purpose 过滤单测）。

### AC3 — 前端 HD paywall + 解锁后下载

**Given** 身份证详情页（6-2）HD 下载入口
**When** 点「下载高清图」
**Then**
- 未解锁 → 弹 paywall（Rp5,000 + 渠道选择 QRIS/PawCoin）→ 支付成功 → 解锁 → 导出 1600×900 PNG 存相册
- 已解锁（`hdUnlocked=true`）→ 直接导出下载（无限次）
**And** 目标资源/档案失效等边界不白屏（沿用多态兜底）
> 验证层：**L0**（paywall 与已解锁分支 widget 测 + 导出触发；`hdUnlocked` 标志驱动）+ **L2**（Midtrans sandbox 真购一次 QRIS + 真机存图）。

## Dev Notes

### 支付集成范式（照抄，安全攸关）

- **模板 = `triage/service/AiUnlockService` + `AiUnlockPaidHandler`**（AI 一次性解锁，与 ID_HD 同构：一次性、双渠道、幂等不重复扣费）。ID_HD 更简单（无免费额度、无红色态短路）。
- **`PaymentPurpose.ID_HD` 已存在**（枚举已含，无需改）。`payment_intents.purpose` CHECK 已含 ID_HD（V47/init，无需改迁移）。
- **PawCoin 即时扣费**：`PawCoinWalletService.debit(userId, coins, SPEND, refType, refId, 幂等键)`——同事务原子改钱包+平衡分录+流水；幂等键 `id-hd:{petProfileId}` 稳定可重放；余额不足抛冲突→回滚。
- **QRIS 到账**：`createIntent` → Midtrans → `PayCallbackController`/轮询 → `PaymentIntentService.applyCallback`（同事务发 `PaymentIntentPaidEvent`）→ `IdHdPaidHandler`（`MANDATORY` 加入同事务）→ 建购买行。**双通道（回调+轮询）幂等去重已由 1.1 applyCallback + Redis 前置 + gateway_ref 唯一兜住**；本 Story 再加 `existsByUserId` 短路。
- **绝不 AFTER_COMMIT 记购买**（血泪：notify AFTER_COMMIT+默认 REQUIRED 曾致 INSERT 静默不提交）——用同步 `@EventListener`+`MANDATORY`。

### 一次性永久语义

- `UNIQUE(user_id)` = 一次购买、永久解锁（单账号）。`pet_profile_id` FK **ON DELETE SET NULL** → 删档/注销后购买行仍在（permanent），删档不被 FK 阻断（勿设 CASCADE 否则删档丢解锁）。⚠️ 对照 memory：新表 FK 处置要想清 ON DELETE，别阻断既有删除测试。
- 已购买短路在**任何扣费分支之前**（照 AiUnlockService 入口短路）——重复点「下载」绝不二次扣费。

### HD 图产出 = 客户端渲染导出

- **后端不生成/不存储 HD 图**（身份证是 Flutter 客户端渲染，6-2）。后端只：收费 + 记一次性永久购买 + 返 `hdUnlocked`。客户端在 `hdUnlocked=true` 时 `RepaintBoundary.toImage` 导出 1600×900 PNG 存相册（无限次）。
- 分享落地页（6-4）另做；本 Story 只做「本机下载存图」。

### 强制护栏（违反即返工）

- 资金四不变式：总账平衡（LedgerService 平衡分录）、幂等（多层：入口短路+debit 幂等键+UNIQUE+回调去重）、非负（debit 余额不足回滚）、审计。
- `ddl-auto=validate`；schema 归 Flyway；CHECK 用 varchar+UPPER_SNAKE。
- 凭证/PII 不入库不入日志；`payment_intent` 对外只用 token 不用顺序 id。

### Flyway 号

- 当前最大迁移 **V74**（6-1 serial）。架构旧稿「V55 init_id_hd」已作废（与 serial 合并的旧计划）。本 Story = **V75**（决策 E2 顺延）。`payment_intents` 表 + `ID_HD` purpose 早在 V47/init 建好，本迁移只建 `id_card_hd_purchases`。
