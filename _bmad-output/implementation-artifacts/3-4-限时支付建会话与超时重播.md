# Story 3.4: 限时支付、建会话与超时重播

Status: review

> V1.1 **Epic 3 第四个（计费流闭环枢纽）** story，**纯后端**。承接 3-1（`consult_orders`/`ConsultBillingService.createOrder` 原语已建、**零调用方**）+ 3-2（免费入队广播）+ 3-3（接单 CAS + 支付窗 + `revertExpiredAcceptances` 回队重播，**上限缺口显式留本 story**）+ Epic 1（`PaymentIntentService`/`PawCoinWalletService`/`PaymentIntentPaidEvent`）+ 2-3（三路径支付解锁**范式模板**）。本 story 把「接单后待支付」推到「支付成功建订单 + 建腾讯 IM 会话」，并补齐重播上限封顶、跳充值暂停顺延（A-4）、支付成功但建会话失败的系统故障落地。
> 源：`epics-v1.1.md` Story 3.4（`POST /consultations/{token}/pay` QRIS/PawCoin→建 `consult_orders`(IN_PROGRESS)+建 IM 会话+记 `paid_at`；超时/取消/跳充值→作废重播+`paused_at`顺延 A-4；支付成功但 IM 建会话失败→系统故障退款路径）· 架构 `architecture-v1.1-delta.md` §3.2（两表 + `consult_orders` 快照）/§5.1（全版本最复杂状态机、服务端权威计时、IM 会话支付后才建、已扣未交付触发退款、重播上限 5次/30min→`failed_consult_requests`）/§4（资金正确性地基 NFR-1~4：双分录/幂等/PawCoin 非负同事务/审计）· 决策 A-4（暂停顺延）/A-5（两表）/C-1（反套利：仅系统故障未交付才可转 PawCoin 附 bonus）/H-4（单列 CAS）。
> **⚠️ 承接现状（Explore 已核实，照此复用勿重写）**：`ConsultBillingService.createOrder(...)`/`ConsultOrder.inProgress(...)` 已建待调；支付范式 = 2-3 `AiUnlockService`(同步 PawCoin + 异步现金) + `AiUnlockPaidHandler`(`@EventListener @Transactional(MANDATORY)` filter purpose)，**照抄给 `VET_CONSULT`**；腾讯 IM = `TencentImClient.createConversation(userImId, vetImId)`；**V1.0 `ConsultAcceptService` 把 IM 失败当非阻塞——与本 story「失败即退款」相反，本 story 需新失败处理**；**退款引擎 Epic 4 未建**，仅 `PawCoinWalletService.credit(REFUND/BONUS)` 原语可用；**无 consult 价格配置**，需仿 `TriageProperties` 建 `ConsultProperties`。
> **范围边界（与下游切分）**：本 story = 支付端点(3 渠道) + 建单/建会话（PawCoin 同步 / 现金异步 handler）+ 重播上限封顶 + 暂停顺延 A-4 + 建会话失败系统故障落地。**不做**：用户端下单三屏 UI（3-5）、兽医端队列/进行中会话 UI（3-6）、兽医到手金额/月结（3-7）、Konsultasi 定稿 + 封禁挂起逃生（3-8）、**用户主动发起的两段退款审批工作流（Epic 4，A-1/A-2）**、后台改价 UI（9-2，价走 env）、PawCoin bonus 溢价配置（9-2 `pawcoin_config.premium_rate`）。

## Story

As a 用户,
I want 在限时窗内付费成功即建会话、超时/取消/跳充值不吃亏,
so that 付费即得服务、没付上也表单不丢（FR-43D）。

## Acceptance Criteria

1. **后端：咨询价格与分成配置（仿 TriageProperties，成交落快照）（L0）**
   **Given** `ConsultBillingService.createOrder` 的 `amount`/`vetPayout`/`vetShareRate`/`unitPrice` 由调用方从 config 取（3-1 Javadoc）
   **When** 建价格来源
   **Then** 新建 `shared/consult/ConsultProperties`（`@ConfigurationProperties("petgo.consult")` + `ConsultConfig` 注册，仿 `TriageProperties`/`TriageConfig`）：`unitPrice`（env `CONSULT_UNIT_PRICE` 默认 **50000** IDR）+ `vetShareRate`（env `CONSULT_VET_SHARE_RATE` 默认 **60**）；`vetPayout = unitPrice * vetShareRate / 100`（=30000，架构 §3.2 例）。`application.yml` + `.env.example` 接线（**L0**）
   **And** 成交价/分成/单价 **落 `consult_orders` 快照**（`amount`/`unit_price_snapshot`/`vet_share_rate_snapshot`/`vet_payout`）——后台 9-2 改价不影响历史订单/退款（**L0/L1**）；`ai_unlock_price` 渐进范式：env→9-2 `pricing_config` 落地后换 DB 读（同 2-1/2-3 注释标注）

2. **后端：限时支付端点 + 鉴权/寻址/守卫（L0/L1）**
   **Given** 3-3 接单成功的 `consult_requests`(ACCEPTED_AWAIT_PAY)
   **When** 用户 `POST /api/v1/consultations/{requestToken}/pay`（JWT USER，body `{channel: QRIS|PAWCOIN}`）
   **Then** 守卫链：`userId` 取自 JWT；`findByRequestToken`→无则 **409**（与「已被处理」同码防枚举）；**归属校验**（`request.userId != jwt` → 403/409 防枚举）；`state != ACCEPTED_AWAIT_PAY` → **409**（未接单/已支付/已回队）；**支付窗未过期**（`pay_deadline_at >= now`，服务端权威；已过期 → **409**「支付窗已过期」，交由扫描器回队）；**未暂停**（`paused_at` 非空时应先 `/resume` 或直接续算，见 AC7）（**L0** DTO/守卫 / **L1** 鉴权/落库）

3. **后端【同一事务 NFR-3】：PawCoin 同步支付 → 建单 + 建会话（L1）**
   **Given** channel=PAWCOIN（站内即时扣，1 koin=Rp1）
   **When** 支付
   **Then** **单一 `@Transactional`** 内原子完成（照 2-3 `AiUnlockService.unlockByPawCoin` 范式）：
   - `walletService.debit(userId, unitPrice, SPEND, "VET_CONSULT", requestId, "vet-consult:"+requestToken)`（幂等键**稳定**=requestToken；余额不足 → `debit` 抛冲突 **409/402** 整事务回滚，不建单不建会话）
   - `billingService.createOrder(userId, vetId, petProfileId, amount=unitPrice, PAWCOIN, paymentIntentId=null, vetPayout, vetShareRate, unitPrice, paidAt=now)`（建 `consult_orders`(IN_PROGRESS) + append `PAID` stage event）
   - **建腾讯 IM 会话**：`imClient.ensureAccount(userImId,...)` + `imClient.ensureAccount(vetImId,...)` + `conv=imClient.createConversation(userImId, vetImId)`；订单记 `session_started_at=now` + append `SESSION_STARTED` stage event（会话标识如何挂在 `consult_orders`——见 Dev Notes「IM 会话标识落点」）
   - `requests.deleteIfState(requestId, ACCEPTED_AWAIT_PAY)`（CAS 删 request，==1 才算转单成功；==0 说明已被扫描器回退/他路处理 → 回滚）
   **And** 三写（扣币 + 建单 + 删 request）**要么全提交要么全回滚**（NFR-3）；兽医保持 BUSY（进行中会话占用，不 goAvailable）（**L1**）

4. **后端【现金异步，复用双通道到账】：QRIS 支付 → handler 建单建会话（L1/L2）**
   **Given** channel=QRIS（Midtrans 收款，异步捕获）
   **When** 发起支付
   **Then** `createIntent(userId, VET_CONSULT, channel, unitPrice, "IDR", "vet-consult:"+requestToken)`（1.1 幂等建单+限流）+ 返回 `PaymentIntentResponse`（QR/deeplink）——**此刻不建单/不建会话/不删 request**（request 仍 ACCEPTED_AWAIT_PAY）（**L1**）
   **And** **到账处理器** `ConsultPaidHandler`（`pay` 或 `consult`，`@EventListener @Transactional(propagation=MANDATORY)`，`if(purpose != VET_CONSULT) return;`，照 `AiUnlockPaidHandler`/`TopupPaidHandler` 范式，**禁 AFTER_COMMIT 异步**——血泪护栏）→ 调 `ConsultPayService.completePaidConsult(event)`：按 `event.userId()` 锚定该用户唯一 ACCEPTED_AWAIT_PAY request（占用不变量：一人一 live request，见 Dev Notes「现金 intent↔request 锚定」）→ 建单（`payment_intent_id`=intent.id, pay_channel=event.channel）+ 建 IM 会话 + `deleteIfState`；**与 `applyCallback.markPaid` 同事务**（**L1** 处理器/L1 端到端 / **L2** sandbox 真付一次，**L2 阻塞于 Midtrans 商务合同**，标注待本地）
   **And** 幂等：回调/轮询双通道 → `applyCallback` 已 PAID 不再发事件；handler 若锚定不到 ACCEPTED_AWAIT_PAY request（已支付删/已回队）→ 走 AC5 系统故障落地或幂等短路（不重复建单/不双扣）（**L1**）

5. **后端【安全攸关：已扣未交付】：支付成功但 IM 建会话失败 → 系统故障落地（L1）**
   **Given** 支付已成功（PawCoin 已扣 / 现金已捕获）但 `createConversation` 抛异常
   **When** 建会话失败
   **Then** 分路（**本 story 触发系统故障路径、不建退款引擎**——退款执行属 Epic 4）：
   - **PawCoin 同步路径**：IM 建会话在**同一事务内** → 失败即**整事务回滚**（扣币撤销、不建单、request 仍 ACCEPTED_AWAIT_PAY）→ 返回 **502/503**「系统繁忙，请重试」；用户**未被扣费**（无需退款）；Tencent 侧孤儿会话无害（无资金损失）（**L1**）
   - **现金异步路径**（Midtrans 已捕获、不可由本地事务回滚 = 真「已扣未交付」）：**不建单**、`deleteIfState` 删 request（避免卡死）、publish `ConsultRequestFailedEvent(reason=SYSTEM_FAILURE, userId, sessionId=0, submittedAt=request.createdAt, onlineVetCount)` → `failed_consult_requests`(SYSTEM_FAILURE，运营跟进) + 通知用户「系统繁忙，我们将尽快为您处理」；**实际退款（转 PawCoin 附 bonus / 真钱）留 Epic 4 + 9-2 bonus 配置**——本 story 只做「触发路径 + 落地不丢单」（**L1**）
   **And** 该分支**禁自动重复建会话**（IM 失败一次即落系统故障，不循环重试拖死支付事务）（**L1**）

6. **后端：支付窗超时/取消 → 作废回队重播 + 重播上限封顶（补 3-3 缺口）（L1）**
   **Given** 3-3 `revertExpiredAcceptances`（回队 + rebroadcast_count++，**无上限**）
   **When** 支付窗到期仍 ACCEPTED_AWAIT_PAY（用户未付）
   **Then** 扩 `revertExpiredAcceptances`：回退前判 **上限**——`rebroadcast_count >= 5`（`CONSULT_MAX_REBROADCAST` 默认 5）**或** 请求存活 `>= 30min`（`created_at` 距今，`CONSULT_REQUEST_MAX_AGE_MIN` 默认 30）→ **不再回队**：`deleteIfState(id, ACCEPTED_AWAIT_PAY)` 删 request + `goAvailable(vetId)` + publish `ConsultRequestFailedEvent(TIMEOUT, ...)` → `failed_consult_requests`(TIMEOUT)；未达上限 → 照 3-3 回队重播（**L1**）
   **And**（可选）用户主动取消 `POST /api/v1/consultations/{requestToken}/cancel`（ACCEPTED_AWAIT_PAY 或 QUEUEING 均可）：`deleteIfState` 删 request + 若曾接单则 `goAvailable(vetId)` + `ConsultRequestFailedEvent(USER_CANCEL, ...)`；无痕不建单（A-5）（**L1**，若纳入——见 [OPEN]）
   **And** 上限触发彻底失败**不建订单**（未扣费，A-5）；重播/删除**单列 state CAS**（H-4），已支付/已他路处理行不受影响；扫描器 try/catch 不 crash（**L1**）

7. **后端：跳充值暂停顺延（A-4，服务端权威）（L1）**
   **Given** 支付窗内用户跳去充值 PawCoin（余额不足）
   **When** `POST /api/v1/consultations/{requestToken}/pause`（离开去充值）/ `POST .../resume`（返回）
   **Then** `pause`：CAS 记 `paused_at=now`（仅 ACCEPTED_AWAIT_PAY 且未暂停时；服务端权威）；`resume`：按**剩余时间顺延**（`remaining = pay_deadline_at - paused_at`；新 `pay_deadline_at = now + remaining`，清 `paused_at`）——**非重置**（A-4）（**L1**）
   **And** **暂停期间支付窗扫描器跳过**（`paused_at IS NOT NULL` 的 ACCEPTED_AWAIT_PAY 不判超时回退，防暂停中被扫走）；`resume` 后恢复计时（**L1**）
   **And** 暂停顺延**服务端权威**（不信客户端计时）；前端跳充值/返回触发在 3-5（本 story 只做后端状态与端点）（**L1**）

8. **后端：契约 + 幂等 + 归属/防枚举 + 无迁移（L0/L1）**
   **Given** 支付/暂停/取消端点
   **When** 调用
   **Then** 非 USER 角色 403（`/consultations/**` 由 SecurityConfig 门控 hasRole USER）、缺 JWT 401、非本人 request → 与「不存在」同 409/403 防枚举；pay 响应 DTO 契约（PawCoin 同步 → 订单视图 `{orderToken,status,sessionStarted}`；现金 → `PaymentIntentResponse`，或联合 DTO 区分同步/异步返回）；**幂等**（NFR-2）：同 requestToken 重复 pay——已转单删 request 后 → 409（幂等；不双扣）、debit 幂等键 `vet-consult:{requestToken}` + intent 幂等键兜底（**L0** DTO / **L1** 幂等/鉴权）
   **And** **无新迁移**（`consult_orders`/`consult_order_stage_events` V66、`payment_intents` V60、`failed_consult_requests` V37、`pawcoin_*`/`ledger_entries` V61 均已在；现金锚定用占用不变量不加列，见 Dev Notes）；`mvn -B package` 绿（**L0**）
