# Story 6.6: 全局通知中心（铃铛）

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a **用户**,
I want **一个铃铛入口汇聚我的所有通知**,
so that **我能集中查看与管理未读消息**。

> 本 Story 实现**首页顶部铃铛入口 + 未读红色角标（Redis 计数）+ 通知中心列表（倒序 + 点击跳目标 + 标记已读）**（FR-34）。
> **依赖**：通知数据来自 **Story 6.1** 已建的 `notifications` 表（6.2/6.3 写入的四类通知：兽医已回复 / 问诊已结束 / 被点赞 / 被评论）；角标读 **6.1** 已建的 Redis 计数 `notify:unread:{userId}`；点击跳目标复用 **6.1** 深链路由表（FR-38）。本 Story 是 Epic 6 的**消费端/收口 Story**。

## Acceptance Criteria

> **验证层标注**：**L0** 静态/单测 · **L1** 集成/运行时（Docker + postgres + redis：列表查询、角标计数、标记已读）· **L2** 端到端/外部（深链点击直达三态、真机）。
> 列表/角标/标记已读为 L1（pg+redis）；点击条目深链直达复用 6.1 三态（L2 真机）。

### AC1 — 首页铃铛 + 未读红色角标（FR-34、与问诊 Tab 红点并存）

**Given** 首页顶部导航栏
**When** 渲染
**Then** 右侧显示铃铛图标，有未读时显示红色数字角标（与问诊 Tab 红点并存不互斥）（FR-34、FR-19 角标规则）
> 验证层：**L1**（角标数字 = Redis `notify:unread:{userId}` 计数，需 redis 在跑；有未读→显示数字、零→隐藏）+ **L0**（角标渲染：0 隐藏 / >0 显示 / 与问诊 Tab 红点共存不互斥的 widget test）。

### AC2 — 点击铃铛进通知列表，倒序展示六（~七）类，点击跳目标并标记已读

**Given** 用户点击铃铛
**When** 进入通知列表
**Then** 按时间倒序展示所有通知（兽医已回复/问诊已结束/被点赞/被评论/**宠物生日提醒（→「+发布」预选成长日历）/陪伴纪念日（→成长档案 Tab）**，及 **L级里程碑节点**——待澄清是否入中心，建议凡系统推送均入），点击条目跳对应目标并标记为已读（FR-34、FR-38）
> 验证层：**L1**（列表按 `created_at desc` 拉取**六/七类**、点击条目 `is_read=true`+`read_at` 写入、Redis 角标递减/重算，需 pg+redis）+ **L2**（点击条目经 6.1 深链路由直达目标页，三态真机）。**L0**：列表项 → 深链 location 映射（复用 6.1 `pushPayloadToLocation`，含三类新增）+ 空列表态渲染单测。
> 🔄 **PRD V1.0.0 修订（Fx · 2026-06-08）：** 铃铛通知列表展示类型由 4 类扩到 6（~7）类——新增「宠物生日提醒（→+发布预选成长日历）」「陪伴纪念日（→成长档案 Tab）」（及 L级里程碑节点，标注「待澄清是否入中心，建议凡系统推送均入」）的图标/文案/深链跳转。后端 `notifications` 查询按 type 通用返回（已含 6.7 写入的新 type），本 Story 主要是前端展示分类（决策 F2）。

### AC3 — L级里程碑条目走零态 + 系统推送直跳的已读同步 [R2·回改]

> **[R2·回改]** 代码已实现，本轮回改补齐。明确铃铛列表「L级里程碑达成」条目的**零态降级**（里程碑本体在 Epic8 延后）+ 系统推送直跳目标页时通知中心条目的**已读同步**。

**Given** 铃铛通知列表
**When** 渲染「L 级里程碑达成」条目类型
**Then** ① 列表/枚举/深链**预留**该条目（点击跳成长档案 Tab → 里程碑列表页 FR-38）；**里程碑本体在 Epic8 延后，本轮该条目走零态**——无真实里程碑数据时**不显示**该条目（仅保留类型枚举/图标/深链映射，不渲染空壳，不报错）
**And** ② 用户通过**系统推送直接跳转**目标页（不经通知中心列表点击）时，通知中心内**对应条目同步标记为已读**（`is_read=true`，角标递减/重算与列表点击口径一致）——避免「推送已读但中心仍显未读」的不一致
> 验证层：**L0**（widget/provider 测试：① 无里程碑数据时该条目不渲染、有数据时按预留枚举/图标/深链渲染的零态降级断言；② 模拟系统推送 deepLinkToken 直跳 → 对应通知条目 `is_read` 同步翻已读 + 角标 invalidate 的 provider 测试）。**L2**（真机：点系统推送直达目标 → 回通知中心该条目已读）。

## Dev Notes

### 关键架构约定（沿用全局）

- 后端 `com.petgo.notify.{web,service,domain,repository,dto,event}`；读自己 6.1 建的 `notifications` 表与 Redis 角标键，**不跨模块 join**。
- 前端 `lib/features/notify/{data,domain,presentation}`（铃铛/角标/列表）+ 复用 `lib/core/router/deep_link_routes`（6.1）。
- API：游标分页 `{items,nextCursor,hasMore}`、camelCase、ISO-8601 UTC、null 省略；对外只暴露 token。

### 推送/通知中心专项约定

- **未读角标 = Redis 计数** `notify:unread:{userId}`（6.1 建，send INCR / 标记已读 DECR / 读取容错回算）——[architecture.md §缓存 Redis 收窄：未读角标计数]。
- 点击条目跳目标复用 **6.1 深链路由表（FR-38）**，目标失效复用 6.1 多态兜底。
- 角标与问诊 Tab 红点**并存不互斥**（FR-19 角标规则）。

### 强制护栏（违反即返工）

- **禁** Redis 当通用缓存（角标键仅未读计数用途）。
- **禁** 跨模块直访 consult/content repository——通知内容已在 `notifications` 表持久化（6.1/6.2/6.3 写入），列表直接读自己的表。
- **禁** 对外返回顺序 id / `target_ref` 内部字段——只暴露 token + deepLinkType。
- 越权：只能读/标记自己的通知（403 防护）。
- 日志严禁记录通知正文中的 PII/健康相关文本。

### 范围边界（本 Story 明确不做）

- ❌ 不建 `notifications` 表 / 不建角标计数键（Story 6.1 已建，本 Story 消费）。
- ❌ 不实现推送下发（6.2/6.3 写入通知，本 Story 只读展示）。
- ❌ 不建深链路由表本体（6.1，本 Story 复用）。
- ❌ V1 不做通知分类筛选/删除单条/免打扰设置（除非 PRD 另有要求）；不做服务端推送合并。
- ✅ 只做：首页铃铛 + 未读角标 + 通知中心列表（倒序四类 + 空态）+ 点击跳目标 + 标记已读 + 对应后端读取/已读/计数端点。
