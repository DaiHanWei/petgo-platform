# Story 2.6: 宠物名片 H5 对外分享页

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a **炫娃型宠物主**,
I want **一个精美的对外宠物名片 H5**,
so that **不用 App 的朋友也能看到我的宠物并被引导下载**。

> 本 Story 实现**对外 H5 名片**：Java 服务端 Thymeleaf 直出 `GET /p/{cardToken}` + OG/Twitter meta + noindex + 预渲染 OG 静态图，闭合病毒拉新飞轮。**全部在后端**（架构「H5 名片 = Java 服务端模板直出 + OG 预渲染静态图」，无独立 SSR、无 Flutter 页面）。分享按钮调起在 2.7。
>
> **依赖前序**：1.1（Thymeleaf 已在脚手架、`profile/web` 空壳）、2.1（公开桶 + CDN，存 OG 预渲染图）、2.2（pet_profiles + card_token）、2.3（成长日历快乐时刻数据源）。多态失效页覆盖 token 无效/宠物删除/账号注销。

## Acceptance Criteria

> **验证层标注**：**L0 静态**（Thymeleaf 渲染/MockMvc/OG meta/noindex/404）· **L1 集成/运行时**（Docker+postgres+redis，真实读 token→渲染）· **L2 端到端/外部凭证**（真实公开桶 + CDN 存取 OG 图、≤3s 真实加载、社交平台预览）。

### AC1 — Thymeleaf 直出名片（头像/名字/品种/介绍 + 最近 5 条快乐时刻）

**Given** 已创建档案的宠物
**When** 访问 `GET /p/{cardToken}`（Thymeleaf 服务端直出）
**Then** 顶部展示宠物头像/名字/品种/一句话介绍，下方为最近 5 条「成长日历快乐时刻」照片流（倒序），不含日常/科普内容（FR-14）
**And** 页面填充 Open Graph/Twitter meta（支持 WhatsApp/Line/Instagram 链接预览），含 noindex，底部展示 App 下载引导
> 验证层：**L1**（cardToken→查档案+最近 5 条 type=GROWTH_MOMENT 倒序，Thymeleaf 直出 HTML，需 postgres）+ **L0**（MockMvc 验返回 HTML 含头像/名字/品种/介绍、OG/Twitter meta、`<meta name="robots" content="noindex">`、下载引导；**不含**日常/科普）。

### AC2 — ≤3s 加载（预渲染 OG 静态图）

**Given** 名片页加载
**When** 用户/访客打开链接
**Then** 加载时间 ≤3 秒（预览大图为生成卡片时预渲染的 OG 静态图，缓存于公开桶 + CDN）（NFR-2）
> 验证层：**L1**（OG meta `og:image` 指向公开桶/CDN 预渲染静态图 URL；首屏不阻塞于多张原图——服务端直出 + CDN 图）+ **L2**（真实网络下首屏 ≤3s、OG 静态图经 CDN 命中）。

### AC3 — 编辑联动同步最新

**Given** 档案信息或快乐时刻更新
**When** 名片被再次访问
**Then** 内容同步最新（FR-39 编辑联动）
> 验证层：**L1**（每次请求实时读库 → 编辑后再访问即最新；OG 预渲染图在档案/头像变更时**重渲染**——见 Dev Notes 重渲染触发）。

### AC4 — 多态失效页（防枚举 + noindex + 404）

**Given** 名片 token 无效、对应宠物已删除或关联账号已注销
**When** 访问 `/p/{cardToken}`
**Then** 返回统一友好失效页（不暴露 token 是否曾存在以防枚举）+ App 下载引导，HTTP 404 + noindex（UX-DR18 ④⑥）
> 验证层：**L0**（token 不存在/宠物 deleted/账号注销三种 → **同一**友好失效页 + HTTP 404 + noindex + 下载引导，**文案/状态码不区分三种情况**防枚举 — MockMvc）。

## Tasks / Subtasks

> 三段组织。**几乎全后端**（Thymeleaf 直出 + OG 预渲染 + 多态失效）。前端仅提供分享入口（2.7 主做，此处确保链接格式）。建议：后端为主。

### 🟦 后端子任务（petgo-backend / `com.petgo.profile.web` + `templates/`）

- [ ] **B1. `CardPageController`（Thymeleaf 直出）** (AC: 1, 4)
  - [ ] `profile/web/CardPageController`：`GET /p/{cardToken}`（**公开、无需鉴权**，架构边界明确）。按 cardToken 查档案 → 取最近 5 条 type=GROWTH_MOMENT 倒序（经 content service 接口）→ 填 model 渲染 `templates/card.html`。
  - [ ] **不含日常/科普**：只查成长日历快乐时刻。
- [ ] **B2. `card.html` 模板 + OG/Twitter meta + noindex** (AC: 1, 2)
  - [ ] `resources/templates/card.html`：顶部头像/名字/品种/一句话介绍；下方最近 5 条快乐时刻照片流（公开桶/CDN URL）；底部 App 下载引导。
  - [ ] **（决策 E4 隐私必做）H5 对外展示的全部图片（头像 + 5 张快乐时刻 + OG 图）一律走 2.1 的「公开桶服务端去 EXIF」分发**（`x-oss-process` 去元数据 或 无 EXIF 副本 URL）——客户端剥离可被绕过，对外 H5 是 GPS 泄漏的最终暴露面（G-4），此处是兜底落点，不可用原始客户端直传 URL 直接对外。
  - [ ] `<head>` 填 OG（`og:title/og:description/og:image/og:type/og:url`）+ Twitter Card meta + `<meta name="robots" content="noindex,nofollow">`。`og:image` = **预渲染 OG 静态图 CDN URL**（非多张原图）。
- [ ] **B3. OG 预渲染静态图** (AC: 2, 3)
  - [ ] 生成/更新名片时**预渲染一张 OG 静态图**（含头像+名字+品种，社交预览大图）存①公开桶 + CDN，落 `pet_profiles.og_image_url`（或专表）。
  - [ ] **重渲染触发**：档案编辑（2.8）/头像变更/快乐时刻变化时重渲染或失效（与 AC3 编辑联动）。本 Story 提供重渲染 service，2.8 编辑成功后调用。
  - [ ] 实现方式记录（服务端图像合成库 / 模板截图等，取与 Java25/SB4 兼容方案，避免重依赖）。
- [ ] **B4. 多态失效页（防枚举）** (AC: 4)
  - [ ] token 不存在 / 宠物 `deleted_at` 非空 / owner 账号注销 → **统一** `templates/card_gone.html`（友好失效 + 下载引导）+ HTTP **404** + noindex。**三种情况文案与状态码完全一致**，不泄漏 token 是否曾存在。
  - [ ] 失效判定经 profile/auth service 接口（账号注销态），不直接 join。
- [ ] **B5. 性能与边界** (AC: 2)
  - [ ] 服务端直出 + CDN 图，首屏不串行拉多原图；确认 `/p/**` 经 Cloudflare（CF CDN for H5）；本端点不套用户鉴权/限流到拒绝正常访客（防滥用可加宽松限流）。

## Dev Notes

### 架构约定

- **H5 = Java 服务端 Thymeleaf 直出**（架构砍独立 SSR）：`GET /p/{cardToken}` 直出 HTML + 服务端填 OG/Twitter meta；预览大图生成卡片时**预渲染静态图缓存到公开桶**。
- **对外路径用不可枚举 token**：`/p/{cardToken}`（2.2 生成），**绝不暴露顺序 id**；noindex 防爬。
- **公开边界**：`/p/{cardToken}` 无需鉴权，仅展示快乐时刻最近 5 条（架构 §Architectural Boundaries 明确）。
- **模块边界**：取快乐时刻经 content service 接口、注销态经 auth service，**禁直接 join**。
- **媒体**：名片图均①公开桶 + CDN（2.1）；**名片绝不含私密健康图/健康事件**（隐私边界）。

### 强制护栏

- **禁 MQ/缓存层/新中间件**；OG 静态图缓存=对象存储+CDN（非 Redis 通用缓存）。
- `ddl-auto=validate`；如加 `og_image_url` 列需 Flyway 迁移（或复用 2.2 表，本 Story 加列迁移）。
- **防枚举**：失效页对「不存在/删除/注销」**统一响应**（同文案同 404），不泄漏 token 曾否存在（FR-14 安全 + UX-DR18）。
- noindex 强制。
- **日志不落** cardToken 关联的 PII；名片仅公开数据，但 access 日志勿堆 token+owner 关联可定位个人。
- 数据出境/隐私：名片只读公开内容，无健康数据。

### 范围边界（不做）

- ❌ 不做分享按钮/系统分享调起（2.7）。
- ❌ 不做 app 内名片预览页（名片是对外 H5，app 内档案是 2.4）。
- ❌ 不含日常/科普/健康事件（仅成长日历快乐时刻最近 5 条）。
- ✅ 只做：`CardPageController` + `card.html`(OG/Twitter/noindex/下载引导) + OG 预渲染静态图 + ≤3s + 编辑联动重渲染 service + 统一防枚举 404 失效页。

### Project Structure Notes

- 后端 `com.petgo.profile.web.CardPageController`（Thymeleaf，与架构 `profile/web/{ProfileApiController, CardPageController(Thymeleaf)}` 对齐）；`resources/templates/{card.html, card_gone.html}`（架构 `templates/card.html`）；OG 预渲染 service 在 `profile/service`；公开桶/CDN 复用 2.1。`/p/**` 经 Cloudflare CDN（架构 §Infrastructure「CF CDN for API+H5」）。
- 前端无新增页面，仅 cardToken→URL 约定。
