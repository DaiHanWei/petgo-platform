Status: review

> **决策（本轮已与用户对齐）**：① **Iris 出款扩 `PaymentGateway.disburse()`** + `StubPaymentGateway` 桩（L1 全绿）+ `MidtransGateway` 接 Iris sandbox（L2 真出款一笔）；财务打款调 `gateway.disburse` 得 ref 存 `payment_proof`。② **兽医"反向冲销"靠订单态自动排除**——订单 `→REFUNDED` 后 3-7 `aggregateCompletedByVet`(status=COMPLETED) 自动排除→月结随动；**不造幻影 `VET_PAYABLE` 冲销分录**（VET_PAYABLE/VET_PAID 从未真正入 ledger，兽医收入=订单聚合制，冲销无物可冲会致借贷不平）；`REFUND_OUT` **只记真钱退款流出**（DEBIT REFUND_OUT / CREDIT CASH_IN，`net_amount`）。
> **范围边界**：本 story = 主管审批（通过/驳回）+ 财务 Iris 打款 + 订单终态 + REFUND_OUT ledger + admin 退款列表/详情 UI + Iris disburse 抽象。**不做**：PawCoin 退款（4-5 已即时退完 DONE，绝不进 4-6）；工单结案通知 + CSAT（4-7）；前端（用户端已在 4-5）。**处理对象仅 QRIS 真钱单**（`approval_status=PENDING_APPROVAL`）。

## Story

As a 主管/财务,
I want 审批退款申请、财务 Iris 打款到账留凭证、订单终态与账目随动,
so that 退款闭环、账目平、职责分离可审计（AB-5E/A-1）。

   **When** 扩审批留痕字段

2. **后端：Iris 出款抽象 `PaymentGateway.disburse`（L0/L1 桩，L2 sandbox）**
   **Given** `PaymentGateway` 仅收款 `createCharge`
   **When** 加出款能力
   **Then** `PaymentGateway` 加 `disburse(DisburseRequest)`→`DisburseResult`（disbursementRef/status/rawMeta）；`StubPaymentGateway.disburse` 返确定性桩 ref（无凭证，L0/L1）；`MidtransGateway.disburse` 接 Midtrans **Iris/Disbursement** API（mode=live，需凭证，L2 sandbox 真出款一笔）；`DisburseRequest` 携 `payoutAccount`/`accountHolderName`（**PII，绝不 log**）+ `amount`(net)/`channel`/`refundRef`（**L0/L1 桩 + L2 live**）

3. **后端：主管审批通过/驳回（`approval_status` 流转 + 职责分离）（L1）**
   **Given** `approval_status=PENDING_APPROVAL`（4-5 用户已填收款）、订单 `REFUNDING`
   **When** 主管操作（`refund.approve` 权限）
   **Then**
   - **通过**：职责分离守卫（`approver≠submitter`，含 SUPER_ADMIN 不豁免）+ `approval_status=APPROVED` + `approver_admin_id` + `approval_note`(必填) + `approved_at` + 审计 `REFUND_APPROVED`（**订单仍 REFUNDING，等财务打款**）（**L1**）
   - **驳回**：`approval_status=REJECTED` + `reject_reason`(必填) + `rejected_at` + 订单 **CAS `REFUNDING→COMPLETED` 置 `refund_rejected=true`**（`markRefundRejectedFromRefunding`，A-2 UX 不撒谎）+ 发 `REFUND_REJECTED` 通知给用户（走 `NotificationService.send` REQUIRES_NEW，无 PII）+ 审计 `REFUND_REJECTED_APPROVAL`（**L1**）
   **And** 非 `PENDING_APPROVAL` 状态→409 幂等拒；`approver=submitter`→职责分离拒 + 违规审计（**L1**）

4. **后端：财务 Iris 打款（订单 REFUNDED + 凭证 + REFUND_OUT 分录 + 职责分离）（L1/L2）**
   **Given** `approval_status=APPROVED`、订单 `REFUNDING`、订单渠道 `QRIS`
   **When** 财务打款（`refund.payout` 权限）
   **Then** 职责分离守卫（`payer≠submitter` 且 `payer≠approver`，含 SUPER_ADMIN 不豁免）+ `approval_status=PROCESSING` → **解密 payout PII 调 `gateway.disburse(net, account, holder, refundRef)`**（幂等键 `refund-payout-{token}`）→ 成功：`approval_status=DONE` + `payer_admin_id` + `payment_proof`(disbursementRef) + `paid_at` + 订单 **CAS `markRefunded`（`REFUNDING→REFUNDED`）** + **`LedgerService.post` `REFUND_OUT` 分录**（DEBIT `REFUND_OUT` / CREDIT `CASH_IN`，`net_amount`，幂等键 `refund-out-{token}`）+ 审计 `REFUND_PAYOUT_RECORDED`（**L1 桩 / L2 sandbox 真出款**）
   **And** **幂等**：重复打款（订单已 REFUNDED / disburse 幂等键 / ledger 幂等键 三闸）不重复出款/不重复记账；非 APPROVED→409；`payer∈{submitter,approver}`→拒 + 违规审计（**L1**）

5. **后端：兽医收入随动 + 账目平（L1）**
   **Given** 订单 `→REFUNDED`
   **When** 月结/收入聚合
   **Then** 3-7 `aggregateCompletedByVet`/`aggregateCompletedForVet`（`status=COMPLETED`）**自动排除已 REFUNDED 订单** → 兽医不再得该单分成、月结视图随动（**无需幻影 VET_PAYABLE 冲销分录**——从未计提，决策 D-2）；`REFUND_OUT` 分录借贷平（**L1**）

6. **后端：admin 退款管理 UI（列表 + 详情 + 角色门控）（L1）**

   **Then** Thymeleaf SSR：列表（退款 + 订单 + 状态，**PII 脱敏**——账号仅末 4 位）+ 详情（各段操作按钮按权限显隐：客服 `refund.submit`→need 判定[4-4 端点]；主管 `refund.approve`→通过/驳回；财务 `refund.payout`→打款）；`AdminRefundController` 扩 `POST /{token}/approval`(通过)/`/approval-reject`(驳回)/`/payout`（`@PreAuthorize` 分权限点，`AdminUserDetails.getAdminAccountId()`，redirect+flash，照 `AdminContentManageController`）；layout 侧栏加「退款管理」入口（**L1**）

## Dev Notes

> 纯后端，退款两段审批的**第二段闭环**（主管 + 财务）+ 真钱出款抽象 + 账目平。在 4-3 原语 + 4-4/4-5 已建流转上加**审批后果**（订单终态/回落 + ledger + 通知 + disburse）+ admin UI。

### 承接现状（已核实，照此复用/勿改）

- **退款态流（本 story 收尾）**：`COMPLETED --(4-4 approveNeed)--> REFUNDING --(4-5 用户填收款 QRIS)--> approval PENDING_APPROVAL --(4-6 主管 approve)--> APPROVED --(4-6 财务 disburse)--> DONE + 订单 REFUNDED`。驳回旁路：主管 reject → REJECTED + 订单 `REFUNDING→COMPLETED+refund_rejected`。
- **职责分离守卫（4-3，A-1，最高危，直接复用勿改）**：`guardDutySeparation(actorAdminId, otherAdminId, role, token)` = admin_id 相等即拒 + 独立事务留审计（含 SUPER_ADMIN 不豁免）。approve 已守 approver≠submitter；payout 已守 payer≠submitter 且 payer≠approver。本 story 扩后果时**保留这两段守卫在最前**。
- **PawCoin 单绝不进 4-6**：4-5 PawCoin 即时退已置 `approval_status=DONE` + 订单 REFUNDED。4-6 只处理 QRIS 真钱单（`PENDING_APPROVAL`→…）。payout 加 `order.payChannel=QRIS` 防御性校验（PawCoin 单不该走到这，若走到即拒）。
- **兽医收入=订单聚合制（非 ledger）**：3-7 `aggregateCompletedByVet`/`aggregateCompletedForVet` 按 `status=COMPLETED` 聚合（REFUNDING/REFUNDED 排除）。订单 `→REFUNDED` 后自动排除→月结随动。**VET_PAYABLE/VET_PAID 从未 post 进 ledger**（仅枚举）→**不做幻影冲销分录**（无物可冲，post 会借贷不平）。
- **ledger 双分录（1.2）**：`LedgerService.post(entryGroup, lines, idempotencyKey)` 强制借贷平（不平不落库）+ 幂等（Redis + DB uq）。退款真钱流出=`DEBIT REFUND_OUT net / CREDIT CASH_IN net`（平台现金减少）。`net_amount`（用户实收，= order − channel_fee；渠道费平台不退，留存）。`entryGroup=UUID`——脚本无 `Math.random`/`UUID`？service 层可用 `java.util.UUID`（非 workflow 脚本，正常 Java 可用）。
- **PaymentGateway 抽象（A-3）**：`PayConfig` 按 `petgo.pay.mode` 装配 Stub(默认/L1)/Midtrans(live/L2，缺凭证拒启动；prod 禁 stub)。加 `disburse` 对称既有 `createCharge`：Stub 返确定性 ref，Midtrans 接 Iris。**桩实现绝不含 PII，live 只 log 异常类名不 log body/凭证/PII**（既有护栏）。
- **PII 出款**：`payout_account`/`account_holder_name` 字段级 AES-GCM 加密（4-3 `EncryptedStringConverter`）。payout 时实体读出即解密（converter 透明），传给 `gateway.disburse`——**绝不入日志/审计 summary**。`payment_proof` 存网关 disbursementRef（非 PII，可留痕）。
- **admin UI（Thymeleaf slice）**：`@Controller` + redirect + `RedirectAttributes` flash + `@PreAuthorize`；`AdminUserDetails.getAdminAccountId()` 作 actor。列表/详情照 `templates/admin/anomalies.html`+`anomaly-detail.html`；侧栏 `layout.html`。权限点 `refund.approve`/`refund.payout`（4-3 `AdminPermissions` 已加）。

### 关键设计决策（dev 照此）

- **D-1 Iris 出款经 `PaymentGateway.disburse`（用户拍板）**：抽象加 disburse，Stub(L1 桩 ref)/Midtrans(L2 Iris sandbox 真出款一笔)。财务打款 = `disburse` 成功后置 DONE + 存 disbursementRef。**disburse 在 DB 事务外**（照收款侧 `PawCoinTopupService` charge 在事务外范式，避免网络长持事务）——或若编排简单可短事务内调用，dev 择；建议：先短事务置 PROCESSING，再事务外 disburse，成功后短事务置 DONE + 订单 + ledger（避免 disburse 失败回滚已扣状态）。**幂等键**保证重试不双出。
- **D-2 兽医冲销靠订单态（用户拍板）**：订单 REFUNDED → 3-7 聚合自动排除，月结随动。**不 post VET_PAYABLE 冲销**（从未计提）。`REFUND_OUT` 只记真钱退款流出（借贷平）。
