# Story 9.4: AI 问诊收入统计与订单查询

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

> **所属**：V1.1 Epic 9 第四个 Story（**后端 + admin 模板**，**纯查询、无迁移**）。承接 9-1 权限码 `order.view`/`order.export`。交付：AI 问诊收入汇总统计（AB-8C）+ `ai_consult_orders` 订单级只读查询/详情/CSV 导出（AB-8G），与兽医咨询订单（`consult_orders`）**命名空间隔离**（独立表、独立页面、独立展示语义）。
> **关键事实**：`ai_consult_orders`（V65，独立命名空间）已有成交价快照 + `status`(PENDING_PAYMENT/COMPLETED/ABNORMAL) + pay_channel(QRIS/PAWCOIN，DANA 已废) + paid_at。**收入 = COMPLETED 订单金额之和**（PENDING 不计收入、ABNORMAL 供对账不计收入）。**无退款/无分成/无待核查**（AI 一次性解锁，比兽医订单简单）。
> **无 Flyway 迁移**：只读聚合 + 查询，不改 schema。

## Story

As a **运营**,
I want **AI 问诊收入汇总与订单级查询**,
so that **我掌握 AI 收入（AB-8C/8G）**。

## Acceptance Criteria

> **验证层**：**L0**（汇总计算/CSV 组装单测、`mvn compile`）· **L1**（真 pg：汇总正确、订单查询、导出、命名空间隔离）。无 L2。

### AC1 — AI 收入汇总正确

**Given** `order.view` 门控 + 汇总（COMPLETED 收入合计 + 各态计数 + 按渠道收入）
**When** 查看
**Then** 收入 = `sum(amount) where status=COMPLETED`；PENDING/ABNORMAL **不计收入**；按渠道（QRIS/PAWCOIN）拆分正确
> 验证层：**L1**（造 COMPLETED + PENDING + ABNORMAL → 汇总只含 COMPLETED，渠道拆分对）+ **L0**（汇总组装单测）。

### AC2 — 订单级只读查询 + 详情

**Given** 订单列表（倒序）+ 详情
**When** 查看
**Then** 只读呈现（order_token/user/amount/channel/status/paid_at）；**与兽医订单页隔离**（独立路由 `/admin/ai-orders`、独立表 `ai_consult_orders`）；无退款入口
> 验证层：**L1**（列表返回 AI 订单、不含兽医订单；详情正确）+ **L0**（行映射单测）。

### AC3 — CSV 导出

**Given** `order.export` 门控
**When** 导出
**Then** CSV（订单号/用户/分诊任务/金额/渠道/状态/成交时间），text/csv + 下载头
> 验证层：**L1**（导出 200 + text/csv + 表头 + 行）+ **L0**（CSV 转义单测）。
