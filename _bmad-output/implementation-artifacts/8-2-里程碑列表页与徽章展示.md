## Story
As a **用户**,
I want **一个分级清晰、徽章直观的里程碑列表页**,
so that **我能一眼看到已解锁与待完成的成长里程碑**。

## Acceptance Criteria

### AC1 — 列表页：顶部宠物信息 + L/M/S 三级分区 + 各级进度（FR-42）
进度条点击进入真页（`/profile/milestones`，已是路由）：顶部宠物头像+名字+总进度；内容区按 L/M/S 独立分区，每区标题显示该级完成进度。**L0**（widget test：分区/进度/header 渲染）。

### AC2 — 徽章彩色/灰锁 + 完成日期
已完成→彩色徽章+完成日期；未完成→灰色锁定轮廓（V1 emoji/图标 + mint 描边占位）。**L0**。

### AC3 — 点击徽章弹层分流
系统自动 / 系统推送+发布 → 说明文案（触发条件），无「已完成」按钮；用户打卡未完成 → 「已打卡 / 去发布」两入口卡片（已打卡 picker + 打卡 API 在 8.4；去发布跳发布在 8.4 细化）。**L0**。

### AC4 — 加载失败 F13 统一失败态 + 重试。**L0**。

### AC5 — 连带真供数：成长档案统计栏「已完成 X/N」+ 2-6 名片里程碑区块改真数据（C5 四处同步）。**L0** 契约金标（App fromJson 字段集）+ **L1** 真后端一致。
