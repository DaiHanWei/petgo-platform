## Story

As a **宠物处于紧急状况的用户**,
I want **在红色评级时收到不可忽视的强提醒**,
so that **我能立刻意识到需立即带宠物就医、不被任何干扰延误**。

> 本 Story 交付 Epic 4 的**红色半屏强提醒**（FR-3 / UX-DR7）——这是产品生命安全支柱最直接的用户面。① 0–5s 锁定：半屏自底滑起、红底 #C97A7A + ⚠️ 大白图标 + display 文案「请立即带 [宠物名] 去宠物医院就诊」+ 副文、**单按钮**禁用并显倒计时；② 解锁后：**单一**「我已知晓」按钮，点击关闭遮罩、返回结果页（**不展示地图导航、不推荐任何医院**）；③ 关闭后结果页保留、**仍不展示兽医 CTA**；无障碍 alertdialog / ASSERTIVE 播报、不依赖颜色单一。**红色态零兽医/变现引流（NFR-9）**。
>
> 🔄 **PRD V1.0.0 修订（F3 · 2026-06-08，安全攸关）：** 删除「去导航 + 系统地图深链」整条（含主按钮「去导航」、系统地图深链调用、`triage_navigation.dart`、地图搜索词 i18n、J2 真机地图验收）；解锁后由原「双按钮（去导航/稍后处理）」改为**单一「我已知晓」按钮**关闭遮罩、返回结果页，不展示地图、不推荐医院。红色态零兽医入口 / 零变现护栏不变（本就有，未削弱）。
>
> 前置依赖：Story 4.1（结果含 dangerLevel=RED）、Story 4.2（安全规则层强制升红 → RED 含模型红 + 兜底红两源）、Story 4.3（DONE 后红色交棒触发位）、Story 1.2（设计 token）。本 Story **前端为主**（半屏 overlay + 倒计时 + 单确认 + 无障碍）；后端无新增。

### 🔒 安全攸关约束（不可协商）

> 本 Story 是生命安全支柱**最直接的用户面**，AC 严谨度要求最高。以下每条均为红线，违反即返工且需安全复审。

- **5s 锁定不可绕过**：前 5 秒双按钮禁用 + 遮罩不可关闭——**背景点击、拖拽下滑、系统返回键、切前后台**均不得提前关闭或跳过倒计时。这是强制用户"看见"的硬护栏。
- **红色态零兽医 / 零变现引流 / 零导航（NFR-9，绝对）**：红色 overlay 内、以及关闭后保留的红色结果页，**永不**出现兽医咨询入口、广告、付费、任何变现/引流元素、**地图导航/医院推荐**。与绿/黄页的"软引导兽医"**严格区分**——红色态唯一出口是单一「我已知晓」按钮关闭遮罩、返回结果页。
  > 🔄 **PRD V1.0.0 修订（F3 · 2026-06-08，安全攸关）：** 删除原「去就医（系统地图）/ 稍后处理（二次确认）」双出口，改为单一「我已知晓」；不展示地图导航、不推荐任何医院。
- **红色不可被降格展示**：danger_level=RED（经 4.2 只升不降）**必须**走半屏强提醒——**严禁**在前端把红色降格为普通卡/温和提示/与绿黄同款弱提醒，绕过强提醒护栏。前端不自行判定等级，一律以后端落库 RED 为准。
