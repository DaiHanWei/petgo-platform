# Story 4.5: 红色半屏强提醒

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a **宠物处于紧急状况的用户**,
I want **在红色评级时收到不可忽视的强提醒并能一键导航就医**,
so that **我能立刻带宠物就医、不被任何干扰延误**。

> 本 Story 交付 Epic 4 的**红色半屏强提醒**（FR-3 / UX-DR7）——这是产品生命安全支柱最直接的用户面。① 0–5s 锁定：半屏自底滑起、红底 #C97A7A + ⚠️ 大白图标 + display 文案「请立即带 [宠物名] 就医」+ 副文、双按钮禁用并显倒计时；② 解锁后：主「去导航」触发系统确认 dialog → 调系统地图传搜索词「宠物医院/Klinik Hewan」，次「稍后处理」触发应用内二次确认「确认已了解风险？」；③ 关闭后结果页保留、**仍不展示兽医 CTA**；无障碍 alertdialog / ASSERTIVE 播报、不依赖颜色单一。**红色态零兽医/变现引流（NFR-9）**。
>
> 前置依赖：Story 4.1（结果含 dangerLevel=RED）、Story 4.2（安全规则层强制升红 → RED 含模型红 + 兜底红两源）、Story 4.3（DONE 后红色交棒触发位）、Story 1.2（设计 token）。本 Story **前端为主**（半屏 overlay + 倒计时 + 双确认 + 系统地图深链 + 无障碍）；后端无新增。

## Acceptance Criteria

> **验证层标注**：每条 AC 末尾标注验证层级与所需本地环境——
> **L0 静态**（widget/lint/倒计时纯逻辑） · **L1 集成/运行时**（需 Docker daemon + postgres + redis，跑通 4.1 契约 + 打桩/4.2 升红制造 RED） · **L2 端到端/外部凭证**（需真机：系统地图深链跳转、屏幕阅读器 VoiceOver/TalkBack ASSERTIVE 播报、真实 RED 链路）。
> 半屏滑起/倒计时锁定/双确认/零兽医 CTA 可在 **L0/L1**；系统地图真实深链 + 真机屏幕阅读器 assertive 播报 + 真实升红链路落 **L2**（生命安全面，L2 必做）。

### AC1 — 0–5s 锁定半屏（自底滑起 + 红底 + ⚠️ + display 文案 + 双按钮禁用倒计时 + 零变现）

**Given** 分诊结果为红色（含安全规则层强制升红）
**When** 结果呈现
**Then** 半屏红色遮罩自底滑起（红底 #C97A7A + 大白 ⚠️ + display 文案「请立即带 [宠物名] 就医」+ 副文），双按钮在前 5 秒禁用并显示倒计时，遮罩不可在 5s 内关闭（FR-3、UX-DR7）
**And** 红色态不展示兽医咨询入口、零变现引流（NFR-9）
> 验证层：**L0/L1**（widget：自底滑起半屏 + #C97A7A + ⚠️ + display 文案带宠物名 + 双按钮禁用 + 倒计时递减 + 5s 内无法关闭/无背景点击关闭 + 全屏内无任何兽医/变现入口）；4.2 升红制造的 RED 也走此页 = **L1**；真实 RED 链路 = **L2**。

### AC2 — 解锁后双按钮（去导航系统确认+系统地图 / 稍后处理二次确认）+ 关闭后零兽医 CTA

**Given** 5 秒锁定结束
**When** 按钮激活
**Then** 主按钮「去导航」触发系统确认「是否打开地图前往附近宠物医院？」，确认后调用系统地图传入搜索词「宠物医院/Klinik Hewan」；次按钮「稍后处理」触发应用内二次确认「确认已了解风险？」
**And** 关闭遮罩后结果页保留、仍不展示兽医 CTA
> 验证层：**L0/L1**（5s 后按钮启用；「去导航」→ 系统确认 dialog；「稍后处理」→ 应用内二次确认；关闭后结果页保留且**无兽医 CTA**）；真实调起系统地图（iOS Apple Maps / Android Maps 深链）传搜索词 = **L2**（真机）。

### AC3 — 无障碍（alertdialog / ASSERTIVE 播报、不依赖颜色单一）

**Given** 无障碍场景
**When** 红色半屏出现
**Then** 以 alertdialog / AccessibilityLiveRegion.ASSERTIVE 播报，红底配 ⚠️ 图标 + 大字不依赖颜色单一（NFR-13）
> 验证层：**L0**（语义树断言：role=alertdialog、liveRegion=assertive、含 ⚠️+大字文本不靠颜色传达等级）+ **L2**（真机 VoiceOver/TalkBack 实测打断式 assertive 播报「请立即带 X 就医」）。

## Dev Notes

### 关键架构约定（本 Story 必须落实）

**红色半屏 Alert（UX-DR7，权威，逐条落实）**：
- ① 锁定期 0–5s：半屏自底滑起、红底 #C97A7A、⚠️ 大白图标 + display 文案 + 副文、双按钮禁用并显倒计时。
- ② 解锁后：主「去导航」触发系统确认 dialog（确认后调系统地图）、次「稍后处理」触发应用内「确认已了解风险？」二次确认。
- ③ 关闭后结果页保留、不展示兽医 CTA；屏幕阅读器以 alertdialog/ASSERTIVE 播报，红底配 ⚠️ + 大字不依赖颜色。

**红色态联动（架构 §Decision Impact）**：安全规则层挂 triage 结果后置，红色态联动 UX 半屏告警 + **零变现入口**。RED 含模型红 + 4.2 强制升红两源，本页无差别处理。

**前端约定**：引用 `core/theme` token（红色 #C97A7A、display 字阶）无硬编码；i18n .arb（id/en，地图搜索词随语言）；系统地图深链经 `core/router`/平台 channel；UX-DR11 自底上滑转场。

### 🔒 安全攸关约束（不可协商）

> 本 Story 是生命安全支柱**最直接的用户面**，AC 严谨度要求最高。以下每条均为红线，违反即返工且需安全复审。

- **5s 锁定不可绕过**：前 5 秒双按钮禁用 + 遮罩不可关闭——**背景点击、拖拽下滑、系统返回键、切前后台**均不得提前关闭或跳过倒计时。这是强制用户"看见"的硬护栏。
- **红色态零兽医 / 零变现引流（NFR-9，绝对）**：红色 overlay 内、以及关闭后保留的红色结果页，**永不**出现兽医咨询入口、广告、付费、任何变现/引流元素。与绿/黄页的"软引导兽医"**严格区分**——红色态唯一出口是"去就医（系统地图）"与"稍后处理（二次确认）"。
- **红色不可被降格展示**：danger_level=RED（经 4.2 只升不降）**必须**走半屏强提醒——**严禁**在前端把红色降格为普通卡/温和提示/与绿黄同款弱提醒，绕过强提醒护栏。前端不自行判定等级，一律以后端落库 RED 为准。
- **无障碍打断式播报（NFR-13）**：alertdialog + ASSERTIVE 确保屏幕阅读器用户**立即被打断告知**；等级不依赖颜色单一（⚠️ + 大字 + 文本三重），保障色盲/低视力用户同等收到生命安全警示。
- **导航出口可靠**：「去导航」必须真实可达系统地图（含无地图 app 的友好降级），不让紧急用户卡在死按钮。
- **免责声明前置可留证（NFR-9）**：红色页延续免责承载（前置、不干扰强提醒主体）；二次确认「确认已了解风险？」本身即风险告知留证。

### 强制护栏（架构 §Enforcement —— 违反即返工）

- 红色态**零变现入口**（架构 Enforcement 明列）；本 Story 全程不得埋任何兽医/引流/付费节点。
- 不在前端判定 danger_level；RED 一律以后端落库值触发，不绕过 4.2。
- 引用 token / .arb，无硬编码色值/文案；地图搜索词随 id/en 切换。
- 多态完整（UX-DR18）：无绑定宠物 → display 降级通用文案；结果失效/无权限 → 走 4.1 403/404 友好态，不白屏。

### 范围边界（防 scope creep —— 本 Story 明确不做）

- ❌ 不做绿/黄结果三态卡（**Story 4.4**）——本 Story 只处理 RED 半屏。
- ❌ 不做分诊后端/Gemini/安全规则层（**4.1/4.2**）——消费其 RED 输出。
- ❌ 不做问诊入口/上传/等待 spinner（**4.3**）——从 DONE+RED 交棒位接手。
- ❌ 不接真实兽医咨询（**Epic 5**）——且红色态本就零兽医引流。
- ❌ 不做红色结果存档（红色态以就医为唯一动作；存档为绿/黄非红能力，见 4.4/FR-16）。
- ✅ 只做：红色半屏 overlay（自底滑起 + #C97A7A + ⚠️ + display 带宠物名 + 副文）+ 5s 锁定倒计时（不可绕过）+ 去导航系统确认+系统地图深链 + 稍后处理二次确认 + 关闭后结果页保留零兽医 CTA + alertdialog/ASSERTIVE 无障碍 + 零变现引流。
