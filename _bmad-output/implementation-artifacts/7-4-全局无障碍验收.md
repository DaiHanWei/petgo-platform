# Story 7.4: 全局无障碍验收

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a **有无障碍需求的用户**,
I want **App 满足基础无障碍标准**,
so that **我能无障碍地使用关键功能，尤其是紧急的红色预警**。

> 本 Story 是 Epic 7 的**横切验收门槛** story（NFR-13 / UX-DR15 / NFR-14），**前端重**。它不新增业务功能，而是把无障碍作为**可验收的设计/实现门槛**收口——对前面各 Epic（auth/triage/consult/content/profile/notify/me）**已实现的全部界面**做无障碍达标审计与修复：AA 对比度、≥44pt 触摸目标、三色态非颜色单一依赖、disclaimer ≥3:1、动态字体 ≤3 级、红色 Alert assertive 播报。这是 PetGo 设计 token（`core/theme`）与各页面实现的合规收口，其中红色预警可访问性属**安全攸关**——紧急医疗预警必须被屏幕阅读器用户清晰、断言式获取。

## Acceptance Criteria

> **验证层标注**：每条 AC 末尾标注验证层级与所需本地环境——
> **L0 静态**（对比度可由 token/截图算法静态校验、触摸目标/语义可 widget test） · **L1 集成/运行时**（需 Docker daemon + postgres + redis 拉真实页面数据） · **L2 端到端/外部凭证**（需真机 + 真实 TalkBack/VoiceOver 屏幕阅读器、系统字体放大）。
> 对比度/触摸目标/三色态可大量在 **L0** 验；屏幕阅读器 assertive 播报与动态字体真实缩放为 **L2**。

### AC1 — AA 对比度（disclaimer ≥3:1）+ ≥44pt 触摸目标 + 三色态非颜色单一

**Given** 全 App 文本与图标
**When** 无障碍审计
**Then** 在 #FAF8F5 底色上文本/图标对比度满足 WCAG AA（≥4.5:1），disclaimer 即使 10px 也维持 ≥3:1（NFR-13、UX-DR15）
**And** 所有可交互元素触摸目标 ≥44×44pt，三色态用 icon+text+color 不依赖颜色单一
> 验证层：**L0**（`core/theme` 颜色 token 对 #FAF8F5 计算对比度——正文/图标 ≥4.5:1、disclaimer ≥3:1；可交互元素 hitTestBehavior/最小尺寸 ≥44×44pt widget test；绿/黄/红三色态断言含 icon+text，移除颜色仍可区分）+ **L1**（拉真实页面数据后逐屏复核动态内容对比度，需起栈）。

### AC2 — 红色半屏 Alert assertive 播报（安全攸关）

**Given** 红色半屏 Alert（FR-3）
**When** 屏幕阅读器场景
**Then** 以 alertdialog / AccessibilityLiveRegion.ASSERTIVE 播报
> 验证层：**L0**（红色 Alert 组件 `Semantics(liveRegion/assertive)` / `alertdialog` role 标注存在、配 ⚠️ 图标 + 大字不依赖颜色单一——widget/语义树测试）+ **L2**（真机 TalkBack/VoiceOver 实测：红色预警弹出**立即断言式播报**、打断当前朗读、用户无需聚焦即获知；红色页零变现入口语义无歧义）。

### AC3 — 动态字体 ≤3 级 + 标题封顶 + 浅色/竖屏

**Given** 系统字体放大
**When** 用户调整动态字体
**Then** body 及以下随系统字号缩放（≤3 级），标题封顶防布局破坏；V1 仅浅色模式、portrait-only（NFR-14）
> 验证层：**L0**（`MediaQuery.textScaler` clamp 上限 ≤3 级、标题样式封顶、`SystemChrome` 锁竖屏 + 仅浅色主题——可单元/widget 验）+ **L2**（真机系统字体放到最大：body 放大、标题不破布局、不溢出/不截断关键信息；横屏被锁、深色不触发）。

## Dev Notes

### 关键架构约定（本 Story 必须落实）

- **目录**：前端 `lib/core/theme/{colors,typography,spacing,elevation}`（对比度/字号 token 单一事实源）+ `lib/shared/widgets/red_alert_overlay`（assertive 播报）+ 各 `features/*/presentation`（逐屏审计修复）。后端无新增逻辑。
- **底色基准**：#FAF8F5（UX 设计 token）；所有对比度按此底色计算。
- **平台**：portrait-only + V1 仅浅色模式（NFR-14，架构 §Starter A），本 Story 收口确认。
- **横切性**：这是验收 story——覆盖**前面各 Epic 已实现界面**，不局限单页；产出是各页面达标 + token 守门测试 + 真机播报验证。

### ⚖️ 合规/安全攸关约束

- **红色预警可访问性 = 安全攸关（NFR-13 / 架构安全语义）**：红色等级（FR-3）是紧急医疗预警，屏幕阅读器用户**必须以 ASSERTIVE/alertdialog 断言式、打断式获取**——延迟或被动播报可能延误紧急处置。配 ⚠️ 图标 + 大字 + 文本，**不依赖颜色单一**（色盲用户必须能识别红色等级）。与"安全规则层只升不降""红色页零变现入口"语义一致：assertive 播报内容不得含变现引导。
- **不依赖颜色单一传达关键信息**：三色态（绿/黄/红）承载分诊危险等级——必须 icon+text+color 三重编码，灰度/色盲可用。这是把医疗安全信息对无障碍用户**无歧义传达**的合规底线。
- **disclaimer 可读性（PDP/信任）**：免责声明（前置可留证）即使 10px 也须 ≥3:1 对比度——确保 PDP 告知/免责对所有用户**实际可读**，非形式化存在。

### 无障碍门槛清单（NFR-13 / UX-DR15，逐屏达标）

- AA 对比度：正文/图标 ≥4.5:1（对 #FAF8F5）；disclaimer ≥3:1。
- 触摸目标：所有可交互元素 ≥44×44pt。
- 三色态：icon+text+color，不依赖颜色单一。
- 红色 alert：alertdialog / AccessibilityLiveRegion.ASSERTIVE 播报。
- 动态字体：body 及以下 ≤3 级缩放，标题封顶防破坏。
- 平台：V1 仅浅色、portrait-only。

### 强制护栏

- 对比度/触摸目标/三色态以**可执行测试**固化（token 矩阵 + widget），CI 守门防后续 Epic 回退——不靠人工自觉。
- 红色 Alert 的 assertive 语义**绝不可降级为 polite/被动**；零变现入口在无障碍语义树中亦不得出现。
- `textScaler` 上限 clamp 不可移除（防超大字号破布局）；portrait/浅色锁定不可绕过。
- 不因无障碍修复引入横屏/深色分支（违反 NFR-14）。

### 范围边界（本 Story 明确不做）

- ❌ 不新增任何业务功能/界面——只做已实现界面的无障碍达标与修复。
- ❌ 不实现深色模式（V2）、不解锁横屏。
- ❌ 不做双语文案内容（7.2）——但配合 7.2 验长文案（印尼语）放大下不破布局。
- ❌ 不改业务逻辑/后端（仅核对错误/红色态文案语义自足）。
- ✅ 只做：对比度收口 + 触摸目标 + 三色态非颜色单一 + 红色 assertive 播报 + 动态字体 ≤3 级封顶 + 平台约束确认 + 逐屏横切审计 + 守门测试。

### Project Structure Notes

- 前端：`lib/core/theme/`（对比度/字号 token，单一事实源 + 守门测试）+ `lib/shared/widgets/red_alert_overlay`（assertive）+ `lib/shared/widgets/triage_result_card`（三色态）+ 各 `features/*/presentation`（逐屏审计修复）+ `app.dart`（textScaler clamp / portrait / 浅色主题）。
- 后端：无新增；仅核对 `shared/error`（ProblemDetail 文案）与 triage `dangerLevel=RED` 字段语义供前端无障碍呈现。
