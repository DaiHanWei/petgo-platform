# Story 4.5: 用户选退款方式与填收款

Status: review

## Story

As a 用户,
I want 客服批准退款后按我的支付方式选退款方式（PawCoin 即时退币 / QRIS 填真钱收款账户）,
so that 我按原支付方式拿回钱、真钱退款净额透明可预期（UX-DR4/8，反套利 C-1）。

## Acceptance Criteria

1. **后端：用户端退款列表 + 详情（解锁「选方式」入口）（L1）**
   **Given** 当前用户有退款请求（`refund_requests.user_id = JWT sub`）
   **When** `GET /api/v1/me/refund-requests`（JWT `role=user`，仅本人——照 `PawCoinController` 取 `jwt.sub`，绝不接受任意 userId，决策 C1）
   **Then** 返回本人退款请求视图列表：`refundToken`/`orderToken`/订单支付渠道（`PAWCOIN`/`QRIS`）/**退款方式**（`PAWCOIN` 付→`INSTANT_PAWCOIN`；`QRIS` 付→`REAL_MONEY`）/`needDecision`/`approvalStatus`/`orderAmount`/**可选出款渠道费预览**（QRIS：BCA0/OVO2500/GOPAY2500，供前端联动展示净额）；**仅 `need_decision=APPROVED` 且未提交（approval_status 空 / 未 REFUNDED）项标记「可选方式」actionable**（**L1**）
   **And** 视图**不含**任何 PII 密文（`payout_account`/`account_holder_name` 绝不外泄，即使本人也不回显明文——只回是否已填）（**L1**）

2. **后端：PawCoin 订单即时退币（L1）**
   **Given** 退款请求 `need_decision=APPROVED`、订单支付渠道 `PAWCOIN`、订单 `REFUNDING`（4-4 批准置）
   **When** 用户确认 `POST /api/v1/refund-requests/{refundToken}/refund-pawcoin`（JWT `role=user`，仅 owner）
   **Then** `wallet.credit(userId, order.amount, REFUND, "refund_request", refundId, idempotencyKey)`（全额、**无手续费**、幂等）+ 订单 **CAS `markRefunded`（`REFUNDING→REFUNDED`）** + 退款请求置 `approval_status=DONE`（系统执行、`payer_admin_id` 空，**不经 4-6**）（**L1**）
   **And** 幂等：重复确认（订单已 REFUNDED / CAS 返 0）→ 不重复入账、不报错（按已处理，credit 幂等键 + CAS 双闸）；非 owner→403；非 APPROVED / 非 PAWCOIN 渠道→409 拒绝（**L1**）

3. **后端：QRIS 订单填真钱收款账户（不可逆 + 净额权威 + 不加 bonus）（L1）**
   **Given** 退款请求 `need_decision=APPROVED`、订单支付渠道 `QRIS`、`approval_status` 空（未提交）
   **When** 用户提交 `POST /api/v1/refund-requests/{refundToken}/payout-info`（JWT `role=user`，仅 owner；body `channel`(BCA/OVO/GOPAY)/`payoutAccount`/`accountHolderName`）
   **Then** 复用 4-3 `RefundService.fillPayout` → **后端权威算净额 `net=order−fee`**（前端传的费/净额一律忽略，FR-NFR-5）+ PII 加密落库 + `approval_status=PENDING_APPROVAL`（**订单留 `REFUNDING` 等 4-6，本 story 不打款、不记真钱 ledger、绝不加 bonus**）（**L1**）
   **And** **不可逆边界**：已提交（`approval_status` 非空）再次提交→409 拒绝（提交后不可改，UX-DR14）；非 owner→403；非 APPROVED / 非 QRIS→409；PawCoin 渠道值（非 BCA/OVO/GOPAY）→400 校验拒（**L1**）

4. **前端：退款方式选择/填收款/账户核实 3 屏（`features/refund/`）（L0/L2）**
   **Given** 用户从「我的退款」列表进入某 `approved` 退款请求
   **When** 走 3 屏流程
   **Then**
   - **屏1 选方式 rs-choose**：按 `refundMethod` 分流——`INSTANT_PAWCOIN`→展示「原路退回 PawCoin·全额·无手续费」+ 单一「确认退币」CTA（调 AC2，成功 Toast + 回列表标 REFUNDED）；`REAL_MONEY`→「退真钱到银行/e-wallet」+「填写收款账户」CTA 进屏2（**L0/L2**）
   - **屏2 填收款 p-refund-account（QRIS 专属）**：选出款渠道（BCA/OVO/GoPay，显示各渠道费）+ 账号 + 户名；**实时联动展示净额 `net=order−fee`（与后端一致，DESIGN.delta 组件④核心正确性）**；**返回键=提交前可安全退回屏1**（UX-DR14）（**L0/L2**）
   - **屏3 账户核实 p-refund-review**：只读复核渠道/账号/户名/净额 + 显式「确认提交（不可撤销）」；提交调 AC3，成功进「审核中」态（`PENDING_APPROVAL`，等主管）+ 回列表；**提交后不可逆**（无返回改）（**L0/L2**）
   **And** 净额展示与后端权威一致（无前端自算费率误差）；无死链/无死控件；渠道费 map 与后端 `PayoutChannel` 同源（前端常量，dev 保持一致，加校验注释）（**L0/L2**）

5. **后端+前端：契约 + 静态/集成/端到端验收（L0/L1/L2）**
   **Given** 全部实现
   **When** 验证
   **Then** `DB_NAME=petgo_l1 mvn -B test` 绿：L0（DTO 契约、路由）+ L1（列表仅本人无 PII；PawCoin 即时退→钱包 +amount + 订单 REFUNDED + approval DONE + 幂等；QRIS 填账户→净额权威 + PENDING_APPROVAL + 不可逆 409 + PII 密文；owner 403；渠道/状态门控 409）；`flutter analyze` 零警告 + `flutter test` 绿（**L0/L1**）
   **And** 不破坏 4-3 `fillPayout`/4-4 批准回落/3-8 挂起退款/PawCoin 钱包既有路径（回归绿）；模拟器端到端：PawCoin 即时退 + QRIS 3 屏填账户 + 返回键语义（**L2 留本地**）

### 承接现状（已核实，照此复用/勿改）

- **订单状态流（本 story 处于中段）**：`COMPLETED --(4-4 approveNeed: markRefundingFromCompleted)--> REFUNDING --(4-5 PawCoin 即时: markRefunded / 或 4-6 财务打款)--> REFUNDED`。`markRefunded`(CAS `REFUNDING→REFUNDED`) 已在 `ConsultOrderRepository`（3-8 建）。**PawCoin 即时退直接 markRefunded；QRIS 留 REFUNDING 等 4-6**。
- **PawCoin 退币范式（3-8 `ConsultSuspensionService.forceEndSuspended`，照抄语义）**：`wallet.credit(userId, order.amount, PawCoinTxnType.REFUND, refType, refId, idempotencyKey)` → 全额、内部记 `FLOAT_LIABILITY` ledger、**幂等**（Redis 前置 + DB 兜底）；随后 `markRefunded`。本 story `refType="refund_request"`、`refId=refundRequest.id`、`idempotencyKey="refund-pawcoin-"+refundToken`（稳定键，重复确认幂等短路）。**双闸幂等**：credit 幂等键 + 订单 CAS（markRefunded 返 0 表示已退，跳过）。
- **QRIS 填账户（4-3 `RefundService.fillPayout` 已建，直接复用）**：`fillPayout(refundToken, PayoutChannel, payoutAccount, accountHolderName)` → `net=max(0, order−channel.fee())` 后端权威 + `EncryptedStringConverter` 加密 PII + `approval_status=PENDING_APPROVAL`。本 story 只在其**外层加 owner/状态/渠道门控 + 不可逆守卫**，净额/加密逻辑不重写。
- **渠道费权威**：`PayoutChannel` BCA(0)/OVO(2500)/GOPAY(2500)。前端渠道费 map 必须与此同源（dev 保持一致 + 注释；9-2 后台可调则外部化，本 story 常量）。
- **用户端点范式**：`/api/v1/me/**` 默认需 JWT（不改 SecurityConfig）；`@AuthenticationPrincipal Jwt jwt` → `Long.parseLong(jwt.getSubject())`（照 `PawCoinController.currentUserId`，绝不接受 body/path 里的 userId，防越权 C1）。写端点 Idempotency-Key 照 `PawCoinTopupController`。
- **通知**：本 story **不发通知**（AB-5B：批准解锁选方式经 need=APPROVED；退款完成/审核中不在本 story 发新通知类型；REFUND_REJECTED 是 4-4 驳回路径）。

### 关键设计决策（dev 照此）

- **D-1 退款方式由原支付渠道决定，非用户自由选**：`PAWCOIN` 付→`INSTANT_PAWCOIN`（原路退币铁律，line 208）；`QRIS` 付→`REAL_MONEY`（退真钱）。屏1「选方式」实为**按渠道展示对应单一路径**，非二选一。**绝不提供 QRIS→PawCoin+bonus 转换**（C-1 反套利，那是系统故障未交付分支，非本 story）。
- **D-2 PawCoin 即时退（本轮用户拍板，不经 4-6）**：PawCoin 站内币无真钱打款/套现风险，两段审批专为 QRIS 真钱设。即时 `credit(REFUND)`+`markRefunded`+`approval_status=DONE`（系统执行 payer 空）。与 3-8 一致、幂等安全（credit 键 + CAS 双闸绝不双退）。
- **D-3 QRIS 不可逆边界**：`approval_status` 空=未提交（可退回屏1）；一旦 `fillPayout` 置 `PENDING_APPROVAL` 即不可逆（再提交 409）。前端屏2 返回键=退回屏1（提交前）、屏3 提交后无返回改（UX-DR14）。
- **D-4 净额权威在后端（FR-NFR-5）**：前端**只展示**联动净额（`order−fee`），**提交只传 channel/account/holder，绝不传 fee/net**；后端按 `PayoutChannel` 权威算，前端传值一律忽略。前端 map 仅为即时展示，与后端同源加校验注释。
- **D-5 列表零 PII 回显**：`GET /me/refund-requests` 视图**不回** `payout_account`/`account_holder_name` 明文（即使本人），只回 `payoutFilled` bool + 渠道。避免解密 PII 进 API 响应（日志/PII 红线）。
- **D-6 无迁移**：`approval_status=DONE`（V71 CHECK 已含）、`markRefunded`（已建）、`credit`（已建）、`fillPayout`（已建）全部复用。**不加 `source`/`pawcoin_premium_rate_snapshot` 列**（4-6/bonus 需要时另起 ALTER，本 story 不预埋）。
