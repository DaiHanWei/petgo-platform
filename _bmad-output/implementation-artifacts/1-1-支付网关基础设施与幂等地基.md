# Story 1.1: 支付网关基础设施与幂等地基

Status: ready-for-dev

> V1.1 增量首个 story。Epic 1（PawCoin 钱包与充值 / 资金地基）。**brownfield**：V1.0 已上线，Flyway 停在 V46，本 story 用 **V47**。
> 源：`_bmad-output/planning-artifacts/epics-v1.1.md` Story 1.1 · 架构：`architecture-v1.1-delta.md` §2/§3.1/§4 · 排期：`sprint-status-v1.1.yaml`（V47=init_payment_intents 已固化）。
> **范围边界**：本 story 只建**支付网关抽象 + 幂等 + payment_intents 意图状态机 + 回调去重**。**不做**充值下单闭环（Story 1.3）、**不做**双分录总账/PawCoin 钱包（Story 1.2）。本 story 用 `mode=stub` 即可跑通 L0/L1，`mode=live` 才打真实 Midtrans（L2）。

## Story

As a 平台,
I want 一套抽象的支付网关（Midtrans 收款）+ 幂等锚 + 回调/轮询双通道去重,
so that 后续一切收费场景（AI 解锁、兽医咨询、PawCoin 充值、身份证高清）都建在同一个可靠、可替换、防重复入账的支付基座上（FR-43 / FR-NFR-2）。

## Acceptance Criteria

1. **模块与网关抽象就位（L0）**
   **Given** 新增 `shared/pay`（基础设施）与 `pay`（业务模块）
   **When** 定义 `PaymentGateway` 接口 + `StubPaymentGateway`（默认）+ `MidtransGateway`（`mode=live`）+ `PayConfig`/`PayProperties` + `PayException`
   **Then** `mvn -B compile` 通过；`PayConfig` 按 `petgo.pay.mode` 选 bean（stub/live），无凭证也能装配启动

2. **payment_intents 表与实体（L0/L1）**
   **Given** Flyway `V47__init_payment_intents.sql` + `PaymentIntent` 实体 + `PaymentIntentRepository`
   **Then** schema `snake_case`、枚举 `varchar`+CHECK、`timestamptz` UTC、`public_token` 唯一、`gateway_ref` 唯一（回调去重库级兜底）、`@Version` 乐观锁（**L0** 编译 + 单测）
   **And** `mvn -B package` 绿灯，`ddl-auto=validate` 下实体与 V47 schema 契约一致（**L1**）

3. **意图状态机幂等（L0/L1）**
   **Given** `PaymentIntentService.createIntent(purpose, amount, channel, idempotencyKey)`
   **When** 同一 `Idempotency-Key` 并发/重放创建
   **Then** 复用既有 `shared/ratelimit/IdempotencyService` 命中即返回既有 intent、不重复建单（**L1**）
   **And** 状态迁移 `PENDING → PAID/FAILED/EXPIRED` 由实体方法守卫、非法迁移抛 `AppException.conflict`（**L0**）

4. **回调验签 + 双通道去重（L1）**
   **Given** `PayCallbackController`（`POST /pay/callback`，内网/白名单，不带 `/api/v1`）
   **When** 同一笔支付经回调 + 轮询两通道各到一次
   **Then** 仅推进一次意图状态、二次入账被幂等拒绝（Redis 前置 + `gateway_ref` 唯一约束权威）（**L1**）
   **And** Midtrans `signature_key`（SHA-512）验签失败即拒（`AppException.forbidden`）、密钥 env 注入不入库、**日志绝不打印 body/凭证/签名**（**L1**）

5. **配置与凭证接线（L0/L1 + L2 前置）**
   **Given** `application.yml` 加 `petgo.pay` 块 + `.env.example` 加占位块
   **Then** `mode=stub` 默认免凭证、`MIDTRANS_SERVER_KEY`/`MIDTRANS_CALLBACK_TOKEN` 留空占位（**L0/L1**）
   **And** `mode=live` 用真实 Midtrans **sandbox** 凭证跑通一笔 QRIS 收款回调（**L2**，留本地/带凭证环境，Completion Notes 标注）

## Dev Notes

> 以下范式来自对现有 V1.0 代码的实测勘探，**照抄不另起炉灶**（brownfield 铁律）。

### 技术栈实测确认
- Spring Boot **4.0.6** / Java **21**；HTTP 客户端 = Spring **`RestClient`**（同步，Boot 4 内置，**不用 WebClient**）。
- **Jackson 3**：生产代码内部自建 `ObjectMapper` 时注意；**测试基类注入的是 `tools.jackson.databind.ObjectMapper`（Jackson 3），别用 `com.fasterxml`**。
- **无 Testcontainers**：L1 打**本地 docker-compose 的真 postgres+redis**（dev profile）。

### 外部 HTTP 客户端范式（照 `shared/ai/GeminiDeveloperApiClient.java` + `shared/im/LiveTencentImClient.java`）
```java
public MidtransGateway(PayProperties props) {
    Duration t = Duration.ofSeconds(props.getTimeoutSeconds());
    SimpleClientHttpRequestFactory rf = new SimpleClientHttpRequestFactory();
    rf.setConnectTimeout(t); rf.setReadTimeout(t);
    this.client = RestClient.builder().baseUrl(props.getBaseUrl()).requestFactory(rf).build();
}
// 密钥走 header（Authorization: Basic base64(serverKey:)），绝不入 URL query（避免落日志）
```
异常转译护栏：`catch (RuntimeException e){ log.warn("支付网关调用失败: {}", e.getClass().getSimpleName()); throw new PayException("支付网关调用失败"); }` —— **只 log 异常类名，绝不打印 body/凭证/签名 URL/上游堆栈**。验签用纯 JDK（照 `LiveTencentImClient.hmacSha256`/`verifyCallback`，常量时间比对）。

### 幂等（复用，勿另造）——`shared/ratelimit/IdempotencyService.java`
```java
Optional<Long> findResourceId(String idempotencyKey)   // 命中即取回既有资源 id
void store(String idempotencyKey, long resourceId)      // TTL 24h, PREFIX="idem:"
```
用法照 `ContentService.publish`（入口先 `findResourceId` 命中返回、落库后 `store`）。**回调去重必须库级唯一约束兜底**（`gateway_ref` UNIQUE，照 `consult_anomalies.session_id` 去重范式）——因回调可能超 24h Redis TTL，Redis 仅作前置、DB 唯一约束作权威。写端点加 `RedisRateLimiter.check(...)`（超限 `AppException.rateLimited`→429）。

### config 范式（照 `shared/ai/AiConfig` 的 mode 选 bean）
```java
@Configuration @EnableConfigurationProperties(PayProperties.class)
public class PayConfig {
  @Bean PaymentGateway paymentGateway(PayProperties p){
    return "live".equalsIgnoreCase(p.getMode()) ? new MidtransGateway(p) : new StubPaymentGateway(p);
  }
}
```


### 实体范式（照 `consult/domain/ConsultSession.java`）
- 主键 `Long id` IDENTITY；对外只暴露 `public_token`（`profile/service/CardTokenGenerator` = `SecureRandom` base62，绝不由顺序 id 派生）。
- 枚举 `@Enumerated(STRING)` + `length=16`（**禁 length=1**，Hibernate6 映 CHAR(1)→validate 全红）。
- `Instant createdAt/updatedAt` + `@PrePersist/@PreUpdate` 赋值（实体层管，迁移层也给 `DEFAULT now()`，并存）。
- JSONB：`@JdbcTypeCode(SqlTypes.JSON) Map<String,Object> gatewayMeta`。
- `@Version long version` 乐观锁（回调并发/状态迁移 CAS）。
- 状态迁移封装实体方法内、`requireStatus()` 守卫抛 `AppException.conflict`。

### 回调 Controller（照 `shared/im/web/ImCallbackController.java`）
`POST /pay/callback`（**不带 `/api/v1`**，内网/白名单语义同 `/im/callback`）；`gateway.verifyCallback(body)` 失败→`AppException.forbidden`；面向 App 的**创建**端点才走 `/api/v1`（JWT + 写限流 + `Idempotency-Key` 头）。错误统一 `AppException.*` → 全局 RFC 9457 ProblemDetail（`shared/error/AppException.java`）。

### V47 迁移完整 SQL（照 `V44__init_consult_anomalies.sql` 风格）
```sql
-- Story 1.1（V1.1 资金地基）：payment_intents 支付意图表。实测最大号 V46 + 1 = V47（决策 E2 单调分配）。
-- 凭证/网关正文绝不入库；金额用最小币种单位整型(IDR 无小数)；status 枚举 varchar + CHECK；
-- gateway_ref 唯一 = 回调/轮询双通道去重的库级权威兜底（Redis 仅前置）。
CREATE TABLE payment_intents (
    id            BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    public_token  VARCHAR(32)  NOT NULL,
    user_id       BIGINT       NOT NULL,
    purpose       VARCHAR(16)  NOT NULL,   -- VET_CONSULT | PAWCOIN_TOPUP | AI_UNLOCK | ID_HD
    channel       VARCHAR(16)  NOT NULL,   -- QRIS | DANA | PAWCOIN
    amount        BIGINT       NOT NULL,
    currency      VARCHAR(8)   NOT NULL DEFAULT 'IDR',
    status        VARCHAR(16)  NOT NULL,   -- PENDING | PAID | FAILED | EXPIRED
    gateway_ref   VARCHAR(128),            -- 网关订单号，回调去重
    gateway_meta  JSONB,                   -- 原始快照（脱敏）
    version       BIGINT       NOT NULL DEFAULT 0,
    created_at    TIMESTAMPTZ  NOT NULL DEFAULT now(),
    updated_at    TIMESTAMPTZ  NOT NULL DEFAULT now(),
    CONSTRAINT ck_payment_intents_status  CHECK (status  IN ('PENDING','PAID','FAILED','EXPIRED')),
    CONSTRAINT ck_payment_intents_purpose CHECK (purpose IN ('VET_CONSULT','PAWCOIN_TOPUP','AI_UNLOCK','ID_HD')),
    CONSTRAINT ck_payment_intents_channel CHECK (channel IN ('QRIS','DANA','PAWCOIN'))
);
CREATE UNIQUE INDEX uq_payment_intents_token       ON payment_intents (public_token);
CREATE UNIQUE INDEX uq_payment_intents_gateway_ref ON payment_intents (gateway_ref);
CREATE INDEX        idx_payment_intents_user       ON payment_intents (user_id);
CREATE INDEX        idx_payment_intents_status     ON payment_intents (status);
```
命名约定：`uq_<表>_<列>` / `idx_<表>_<列>` / `ck_<表>_<语义>`。**决策 E2：已提交迁移冻结，加列一律新起 ALTER**。
