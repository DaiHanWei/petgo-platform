## Story

As a **用户**,

## Acceptance Criteria

> **验证层标注**：**L0 静态**（`flutter analyze` / `flutter test`，widget/单测，无需真机）· **L2 端到端**（Android 模拟器视觉验收——三风格观感、水印、导出图、编辑不落库）。本 Story **无 L1**（前端渲染 + 会话级状态，数据接口 6-1 已 L1 覆盖；生成/读取接线复用 6-1 端点，其后端行为不在本 Story 重验）。

### AC1 — 三风格 + 1600×900 + 三级水印

**Given** 三风格（KTP / Paspor / Pelajar）+ 1600×900 输出画布 + 三级水印
**When** 渲染 / 切换风格
**Then** 三种风格均可渲染，取 6-1 `GET /me/id-card` 真实数据（宠物名/类型/品种/生日/头像 + 展示编号 serialId）；导出图固定 **1600×900**；**三级水印**齐备（① 斜向重复文字水印 ② 「HIBURAN / 娱乐仿制」戳记 ③ 品牌 logo 水印）
> 验证层：**L0**（三风格 widget 各自可构建 + 导出尺寸 1600×900 断言 + 三级水印元素存在断言）+ **L2**（模拟器三风格切换观感 + 导出图视觉）。

### AC2 — KTP 正反面 + 背面娱乐仿制免责

**Given** KTP 风格
**When** 渲染
**Then** KTP 有**正面 + 背面**（可翻面/切换），**背面含娱乐仿制免责声明**（「非官方证件 · 无法律效力 · Hanya untuk hiburan」等，字体可读但不喧宾夺主，UX High-185）
> 验证层：**L0**（KTP 正反两面 widget + 背面免责文案存在断言）+ **L2**（模拟器翻面观感）。

### AC3 — 编辑仅当前预览会话，不写档案、后端不接收

**Given** 证件卡预览可编辑（如展示名/副标题等预览字段）
**When** 用户在预览里编辑
**Then** 编辑**仅作用当前预览会话**（本地不可变状态），**不写宠物档案**、**后端不接收**（无 PATCH /pet-profiles、无任何写档案调用）；退出预览即丢弃，档案数据不变
> 验证层：**L0**（编辑改预览状态但绝不调 `ProfileRepository.update` / 任何档案写端点的 widget 测 + 重开预览恢复档案原值断言）+ **L2**（模拟器改预览→退出→档案未变）。

### AC4 — 不渲染成可冒充真实政府证件 + 老用户「尚未生成」引导态

**Given** 合规红线（FR-49B：娱乐仿制，不得冒充真实政府证件）+ 老用户未生成态（6-1 `generated=false`）
**When** 渲染 / 首次进入
**Then** 视觉**明确区分于真实政府证件**（TailTopia 品牌化 + 三级水印 + 免责，绝不 1:1 复刻真实 KTP 版式/国徽/官方字段）；`generated=false`（老用户/未生成）→ 渲染「尚未生成」引导态 + 「生成身份证」CTA → 调 6-1 `POST /me/id-card` 分配号后渲染
> 验证层：**L0**（未生成态 widget + 生成 CTA 调 `POST /me/id-card` 接线断言；合规元素[水印/免责/品牌]恒在的断言）+ **L2**（模拟器合规观感人审 + 生成流）。

## Dev Notes

### 关键约定（沿用全局）

- **前端结构**：`features/profile/{data,domain,presentation}`；Riverpod + `AsyncValue` + 不可变；dio 拦截器 + ProblemDetail 映射；go_router 深链——继承 V1.0（architecture-v1.1-delta §8）。
- **i18n**：App **绝不**渲染后端显示串；所有证件文案本地化（en+id，印尼语主）。里程碑/证件标题源在前端（memory `petgo-i18n-model-and-debt`）。
- **主题**：当前 App 主题是**淡紫色 violet**（主色 #845EC9，源真相 colors.dart；`mint*` 是遗留别名，memory `petgo-mint-brand-switch`）。Poppins 字体。

### 设计约束（本 Story 核心，⚠️ 视觉方向待用户确认）

- **三风格**：KTP（印尼身份证风格，**正+反两面**）/ Paspor（护照风格）/ Pelajar（学生证风格）。三者共用 1600×900 画布 + 三级水印 + 品牌框架，风格差异在版式/配色/字段编排。
- **三级水印**（FR-49B 娱乐仿制强化）：① 斜向重复文字水印 ② 「娱乐仿制 / 非官方」戳记 ③ 品牌 logo 水印。三层叠加 → 强化「这是趣味卡，非真证件」。
- **合规红线（AC4，安全攸关，architecture-v1.1-delta §9）**：KTP 背面**必含免责**（非官方 · 无法律效力 · Hanya untuk hiburan）；**绝不** 1:1 复刻真实印尼 KTP 的国徽/官方版式/官方字段名 → 用 TailTopia 品牌化 + 卡通宠物元素明确区分；分享落地页（6-4）亦不得冒充真证件。
- **编辑会话态（AC3）**：预览可编辑字段仅存本地会话，**后端不接收、不写档案**（6-1 的接口只读/生成号，无字段写入；档案字段改走既有 2.8 PATCH，与身份证预览编辑无关）。
- **导出 1600×900**：`RepaintBoundary.toImage` 按比例导出；本 Story 只保证画布尺寸与截图能力，付费高清下载（Rp5,000 一次性永久）留 6-3、分享落地留 6-4。

### KTP 参考设计（本次实现基准，源 `/Users/dai/Downloads/KTP Tailtopia.png`）

横向卡（导出 1600×900，浅蓝底），版式：
- **底纹水印**：斜向重复文字「KARTU TANDA PENDUDUK TAILTOPIA」/「PENDUDUK TAILTOPIA」浅蓝，铺满 → 兼「品牌水印 + 安全底纹」（三级水印之一/二）。
- **左上**：TailTopia logo（圆角方 + 渐变描边 + 紫色 T/爪 标志）。
- **上中标题**：`KARTU TANDA PENDUDUK TAILTOPIA`（黑体加粗，两行）。
- **右上**：`pet profile batch`（斜体灰，趣味批次标 —— 真 KTP 此处是省/市，改品牌趣味标 = 区分真证件之一）。
- **NIK 行**：`NIK : <16位数字>`（黑体大号）。由 serialId 构造 KTP 风格号（区域码 3276 + 生日 DDMMYY + serial 补零），趣味且唯一。
- **字段行**（label : VALUE，值大写黑体）：`Nama`(name) / `Tempat/Tgl Lahir`(可编辑地点+生日) / `Spesies`(petType 本地化) / `Ras`(breed) / `Jenis Kelamin`(可编辑,默认 JANTAN) / `Alamat`(可编辑,趣味默认) / `Status Perkawinan`(可编辑,默认 LAJANG) / `Pekerjaan`(可编辑,默认 CHIEF HAPPINESS OFFICER) / `Kewarganegaraan`(可编辑,默认 INDONESIA) / `Berlaku Hingga`(可编辑,默认 SEUMUR HIDUP)。
  - `*customable` 字段 = 预览会话可编辑（AC3），非档案真值；来自档案的 Nama/Spesies/Ras/Tgl Lahir 默认取 6-1 数据、亦可预览覆盖。
- **右侧**：照片框（放宠物 avatarUrl，无则占位）；框下 place + date（如 `BANDUNG / 01-01-2022`）；再下 **爪印戳**（黑色 paw print，替代真证件指纹 = 区分真证件之一，第三级水印/戳记）。
- **KTP 背面**（AC2 合规必做）：品牌化背面 + **娱乐仿制免责**（`Kartu ini hanya untuk hiburan · Bukan dokumen resmi · Tanpa kekuatan hukum` / 中英对照可选）+ serial + logo。翻面切换。
- 字段 label（Nama/Tempat 等印尼语）是 **KTP 设计常量**（非 app-locale UI，硬编码印尼语，照 milestone_titles 范式）；页面 chrome/编辑面板/风格 Tab/免责走 i18n（en+id）。
- **合规**：整卡 TailTopia 品牌化 + 三级水印/戳记 + 宠物字段(Spesies/Ras) + 爪印 + 背面免责 → 明确「趣味仿制非真证件」；用户已确认此设计方向。

### 数据接线（6-1 已就绪）

- `GET /api/v1/pet-profiles/me/id-card` → `{generated, serialId, name, petType, breed, birthday, avatarUrl, intro}`；`generated=false`（老用户）→ 引导态。无档案 → 404（归一为「无档案」，前端落空态）。
- `POST /api/v1/pet-profiles/me/id-card` → 生成（分配号，幂等）→ 返回 `generated=true`+serialId。生成 CTA 调此。
- `serialId` **仅展示编号**（如「No. 000123」），**绝不**作分享/深链定位键（6-1 AC3 红线，前端亦守）。
