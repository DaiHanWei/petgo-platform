# Story 3.5: 用户端下单链路 UI

Status: review

> V1.1 **Epic 3 第五个** story，**后端小补 + 前端为主**。承接 3-2（`POST /consultations` 入队）+ 3-3（接单开支付窗）+ 3-4（`/pay`·`/pause`·`/resume`·`/cancel` 支付/暂停/取消已实现）。本 story 建用户侧下单三屏（确认→等待→限时支付）+ 明确中断语义（UX-DR11/14），并补一个**用户侧请求状态轮询端点**（3-2~3-4 只建了写端点、无读端点，前端无法轮询 待接单→待支付 跃迁）。
> 源：`epics-v1.1.md` Story 3.5（`p-vet-request-confirm`/`p-vet-waiting`(1min)/`p-vet-timed-pay`(1.5min)；待接单=可丢弃无痕 / 待支付=服务端权威中断即重播 / 支付按钮全程可用；`flutter analyze` 绿 L0 + 模拟器视觉 L2）· UX-DR11（下单三屏）· UX-DR14（全屏扣钱流返回键/中断语义）· 架构 §5.1（服务端权威计时、A-4 暂停顺延）。
> **⚠️ 现状（Explore 已核实）**：V1.1 计费下单三屏**完全无骨架**（全仓无 requestToken/queueDeadlineAt/PAYMENT_REQUIRED/consult 侧 /pay 痕迹）。但有三条成熟范式**直接复用**：① V1.0 免费问诊等待页 `consult_waiting_page.dart`（1min 倒计时 + 轮询 + `PopScope` + detached-cancel「无痕丢弃」——正是「待接单可丢弃」的现成实现）；② PawCoin 充值页 `recharge_page.dart`（渠道选择 + 轮询 + QR/deeplink）；③ 分诊解锁 `unlock_method_sheet.dart`（渠道卡 `_MethodTile` + 余额判可用 `balance >= price`）。付款成功导航复用既有 `GET /consult-sessions/active`（付费问诊建的 `consult_sessions` 行会被它命中→跳 `/consult/conversation/:id`）。
> **范围边界**：本 story = 请求状态 GET 端点 + 三屏 UI + 中断语义 + 倒计时 + 渠道选择 + 跳充值暂停顺延前端。**不做**：兽医端队列/进行中会话 UI（3-6）、兽医收入（3-7）、Konsultasi 入口 Tab 定稿 + 封禁挂起逃生（3-8，本 story 三屏可经路由/临时入口触达即可，正式入口整合在 3-8）、订单中心（5-x）、后端支付/暂停逻辑（3-4 已实现，本 story 只调）。

## Story

As a 用户,
I want 确认→等待→限时支付三屏与明确的中断语义,
so that 全流程清楚、返回不误伤（UX-DR11/14）。

## Acceptance Criteria

1. **后端：用户侧请求状态轮询端点（补 3-2~3-4 读端点缺口）（L0/L1）**
   **Given** 三屏需轮询「我的请求现在 QUEUEING（等）/ ACCEPTED_AWAIT_PAY（去支付）/ 已消失（超时或已转单）」驱动跃迁 + 服务端权威倒计时
   **When** 前端轮询
   **Then** 新增 `GET /api/v1/consultations/{requestToken}`（JWT USER，owner 校验）→ `ConsultRequestStatusResponse{state, queueDeadlineAt, payDeadlineAt, pausedAt}`（服务端权威 timestamptz）；**非本人/不存在（含已超时删/已转单删）→ 404**（前端据 404 + `GET /consult-sessions/active` 区分「已转单进会话」vs「超时无兽医」）（**L0** DTO / **L1** 端点/鉴权）
   **And** `ConsultRequestService` 加只读 `statusOf(userId, requestToken)`；`ConsultRequestRepository.findByRequestToken` 已有复用；无迁移（**L1**）

2. **前端：确认发起屏 `p-vet-request-confirm`（L0）**
   **Given** 用户从问诊入口选「找兽医（付费）」
   **When** 进确认屏
   **Then** 展示付费问诊说明 + 单价（`Rp50.000`，手工千分位；价可先前端常量/或从后端下发——见 Dev Notes）+ 主 CTA「发起问诊」；点击 → `POST /api/v1/consultations`（无 body，userId 自 JWT）→ 拿 `requestToken`/`queueDeadlineAt` → `pushReplacement` 等待屏；`alreadyActive=true`（已有 live 请求）→ 直接跳对应态屏（不重复建）（**L0**）
   **And** 无宠物档案 → 后端 409 → 前端 l10n 文案引导先建档（复用既有 ProblemDetail→l10n 映射范式）（**L0**）

3. **前端：等待接单屏 `p-vet-waiting`（1min，可丢弃无痕）（L0/L2）**
   **Given** QUEUEING 请求
   **When** 等待
   **Then** 照 `consult_waiting_page.dart` 范式：1min 倒计时（显示 `queueDeadlineAt - now`，MM:SS）+ 3s 轮询 `GET /consultations/{token}`；轮询到 `ACCEPTED_AWAIT_PAY` → `pushReplacement` 限时支付屏；404（超时无兽医）→ 整屏切「暂无兽医接单」态（非弹层）（**L0**）
   **And** **中断=可丢弃无痕（UX-DR14）**：`PopScope(canPop:false)` 拦返回 → 二次确认「取消问诊？」→ 确认则 `POST /{token}/cancel` 无痕丢弃；`didChangeAppLifecycleState` detached → `cancel`（照 waiting 页 `_cancelOnExit`）——待接单态退出即取消、不留痕（A-5）（**L0** / 模拟器 **L2**）

4. **前端：限时支付屏 `p-vet-timed-pay`（1.5min，支付按钮全程可用）（L0/L2）**
   **Given** ACCEPTED_AWAIT_PAY 请求
   **When** 支付
   **Then** 1.5min 倒计时（显示 `payDeadlineAt - now`，MM:SS，服务端权威）+ 渠道选择（QRIS/PawCoin，照 `unlock_method_sheet` 渠道卡）+ **支付按钮全程可用**（倒计时中任意时刻可点，不置灰）；点支付 → `POST /{token}/pay {channel}`：
   - `mode=DONE`（PawCoin 即时）→ `GET /consult-sessions/active` 拿会话 → `pushReplacement` `/consult/conversation/:id`
   - `mode=PAYMENT_REQUIRED`（现金）→ 展示 QR（QRIS 二维码图）+ 轮询 `GET /payments/{intent}/status`（1.5 已有）→ PAID → `GET /consult-sessions/active` → 跳会话（照 `recharge_page.dart` 轮询范式）（**L0** / 模拟器真付 **L2**）
   **And** **PawCoin 余额不足**：选 PawCoin 时读 `pawCoinProvider.value?.balance` 判 `balance >= price`，不足 → 禁选或提示 + 「去充值」按钮（照 `unlock_method_sheet` `enabled` 范式）（**L0**）
   **And** **中断=服务端权威即重播（UX-DR14）**：返回/退出**不本地取消**——`PopScope` 提示「离开将由系统处理」，退出后服务端支付窗超时扫描自动作废重播（3-4 已实现，前端不做删除）；**支付表单不丢**（重进同 token 仍可支付）（**L0**）

5. **前端：跳充值暂停顺延（A-4）（L0/L2）**
   **Given** 支付屏选 PawCoin 余额不足点「去充值」
   **When** 离开去充值 / 返回
   **Then** 离开前 `POST /{token}/pause`（服务端记 `paused_at`）→ `push` `/me/pawcoin/recharge`（既有页）；充值页返回 → `POST /{token}/resume`（服务端按剩余顺延 `pay_deadline_at`）→ 重新 `GET /consultations/{token}` 拿新 `payDeadlineAt` 续倒计时（非重置）（**L0** 时序 / 模拟器 **L2**）
   **And** 暂停期间倒计时前端可暂停显示；`resume` 失败（已超时等）→ 回退到合适态（重新轮询 status，404 则超时态）（**L0**）

6. **前端：路由 + 模型 + repository + i18n（L0）**
   **Given** 三屏
   **When** 接线
   **Then** `app_router.dart` 加三条 shell 外顶层路由（隐 Tab Bar，token 走 path param）；`features/consult/domain/` 加 `ConsultRequest`(requestToken/state/queueDeadlineAt/payDeadlineAt/pausedAt/alreadyActive) + `ConsultPayResult`(mode/order/payment，手写 fromJson)；`ConsultRepository` 加 `createRequest`/`requestStatus`/`payRequest`/`pauseRequest`/`resumeRequest`/`cancelRequest`；`ApiPaths` 加 `consultations`/`consultationStatus(token)`/`consultationPay(token)`/`consultationPause|Resume|Cancel(token)`；`app_id.arb`+`app_en.arb` 加所有新文案（确认/等待/支付/渠道/倒计时/余额不足/暂停，印尼语主）→ `flutter gen-l10n`（**L0**）

7. **前端：静态验收（L0）**
   **Given** 全部实现
   **When** 验证
   **Then** `flutter analyze` **零警告**、`flutter test` 全绿（含新 widget/controller 测试 + `microcopy_rules_test` 通过）；三屏 controller 单测覆盖 `DONE|PAYMENT_REQUIRED` 分支、余额不足、404 跃迁、中断语义（照 `triage_paywall_test` 范式，fake repo + `DioException(409/404)`）（**L0**）
   **And** 模拟器全链路视觉验收（确认→等待→接单→支付→会话）留本地（**L2**，云端 headless 只跑 L0）

## Dev Notes

> 后端小补（一个只读 GET）+ 前端为主（三屏）。**不重复造轮子**：等待/倒计时/中断照 V1.0 `consult_waiting_page.dart`、支付渠道/轮询照 `recharge_page.dart`+`unlock_method_sheet.dart`、付款后导航复用 `GET /consult-sessions/active`。三屏全新但每块都有现成范式可抄。

### 后端承接（3-2~3-4，已实现，前端直接调）
- **`POST /api/v1/consultations`**（3-2 `ConsultRequestController`，JWT USER）→ `ConsultationResponse{requestToken, state, queueDeadlineAt, alreadyActive}`（占用命中 alreadyActive=true 返现有）。**无宠物档案 → 409**。
- **`POST /api/v1/consultations/{token}/pay`**（3-4 `ConsultPayController`，body `{channel: QRIS|PAWCOIN}`）→ `ConsultPayResponse{mode: DONE|PAYMENT_REQUIRED, order:{orderToken,status,paidAt,sessionStartedAt}, payment:{token,purpose,channel,amount,currency,status}}`。守卫不符 409（防枚举）；支付窗过期 409；PawCoin 余额不足 409；IM 失败 503。
- **`POST /{token}/pause`·`/resume`·`/cancel`**（3-4，void 200）。pause 记 paused_at；resume 顺延；cancel 物理删。
- **`GET /consult-sessions/active`**（V1.0 `ConsultSessionController:71`，`findActiveForUser`）→ 用户当前活跃会话（含 `id`/`imConversationId`）。**付费问诊建的 consult_sessions(IN_PROGRESS, source=DIRECT) 应被其命中**——付款成功后前端调此拿会话 id 跳 `/consult/conversation/:id`。⚠️ **dev 须验证** `findActiveForUser` 不按 source 过滤掉付费会话（若过滤则微调 service，本 story 附带）。
- **`GET /payments/{intent}/status`**（1.5，轮询付款态 PAID/PENDING/FAILED/EXPIRED）——现金路径轮询到账用。
- **本 story 新增**：`GET /api/v1/consultations/{token}`（AC1，请求状态轮询）——3-2~3-4 只建写端点，前端轮询 待接单→待支付 跃迁必需此读端点。SecurityConfig `/api/v1/consultations/**` 已门控 hasRole USER（3-4 已补），新 GET 自动覆盖。

### 关键设计决策（dev 照此）
- **服务端权威计时（UX-DR14）**：倒计时纯**显示**（`deadline - now(client)`）；**过期/跃迁由轮询服务端 state 驱动**，客户端倒计时归零不自行删流程——照 waiting 页「超时靠轮询非本地 timer 单独判」。待支付到 0 → 继续轮询 → 服务端扫描回退后 GET 404 → 前端据 404+active 区分。
- **待接单 vs 待支付中断（UX-DR14，核心分野）**：待接单 = 前端 detached/返回**主动 `POST /cancel`** 无痕（照 waiting `_cancelOnExit`）；待支付 = 前端**不删**，退出后服务端支付窗超时自动作废重播（3-4），前端 `PopScope` 只提示不调删除（表单不丢，重进可付）。
- **付款后导航**：PawCoin `DONE` 立即 `GET /consult-sessions/active`→跳会话；现金 `PAYMENT_REQUIRED` 轮询 `/payments/status`→PAID→`/active`→跳会话。均复用既有 active + `/consult/conversation/:id`（无需新导航端点）。
- **单价来源**：确认屏/支付屏展示单价。后端 `ConsultProperties.unitPrice`(3-4，默认 50000) 未下发前端。**默认**：前端先常量 `kVetConsultPriceIdr=50000`（与后端 env 对齐，注释标 9-2 后台落地后改后端下发）；或 pay 现金响应 `payment.amount` 即权威值可回填。dev 择简（确认屏用常量、支付屏可用 payment.amount 校准）。**[OPEN]**
- **入口**：本 story 三屏经路由 `/consult/vet-request` 可达即可（测试/临时入口）；正式 Konsultasi Tab 入口整合在 3-8。不在本 story 动既有 Tab 结构。
- **无迁移**（后端仅加只读 GET + DTO）。
