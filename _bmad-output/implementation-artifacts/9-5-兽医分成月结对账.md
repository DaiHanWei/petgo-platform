# Story 9.5: 兽医分成月结对账

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

> **所属**：V1.1 Epic 9 第五个 Story（**后端 + admin 模板**，**触资金/财务**）。承接 3-7 月结生成（`vet_settlements`，V68，每月 1 日 WIB @Scheduled 聚合上月 COMPLETED 订单的 `vet_payout` 快照）。交付：财务对账流 **PENDING_FINANCE → PAID → ARCHIVED** + 打款凭证 + 每步审计（AB-8D）。
> **关键落差**：3-7 建表用状态 `PENDING`/`SETTLED`（2 态，无凭证/paid_at），与架构 §3.4 `PENDING_FINANCE`/`PAID`/`ARCHIVED`（3 态 + payment_proof + paid_at）不符。本 story **迁移对齐架构**（V80）：`PENDING`→`PENDING_FINANCE`、`SETTLED`→`PAID`，加 `ARCHIVED` + 凭证/时间/操作人列。
> **vet 端零改**（关键）：兽医 App 按 `status==SETTLED` 判已结算（`vet_income.dart`）。故 **`VetIncomeResponse.ofSettlement` 映射** finance 3 态 → vet 友好 2 态（`PENDING_FINANCE`→`PENDING`、`PAID`/`ARCHIVED`→`SETTLED`），兽医只关心「已否打款给我」，App 无需改。
> **权限**：9-1 未含月结权限。本 story 加 `settlement.view`（对账查看）+ `settlement.payout`（确认打款/归档），常量+i18n**无迁移**（同 9-1 范式）。

## Story

As a **财务**,
I want **月结对账→确认→已付款→归档**,
so that **兽医分成可对账打款（AB-8D）**。

## Acceptance Criteria

> **验证层**：**L0**（状态机转换/映射/校验单测、`mvn compile`）· **L1**（V80 迁移值迁移 + schema validate、对账流转 + 凭证 + 审计、vet 端映射不变）。无 L2。

### AC1 — V80 迁移对齐架构（3 态 + 凭证）+ 值迁移

**Given** `vet_settlements` 状态迁移 `PENDING→PENDING_FINANCE`/`SETTLED→PAID` + 加 `ARCHIVED` + `payment_proof`/`paid_at`/`archived_at`/`settled_by`（**Flyway V80**）
**When** 上下文启动 + 月结生成
**Then** 存量行值已迁移；CHECK=('PENDING_FINANCE','PAID','ARCHIVED')；生成新月结为 `PENDING_FINANCE`；`ddl-auto=validate` 一致
> 验证层：**L1**（迁移后存量值正确 + validate 过 + 新生成 PENDING_FINANCE）+ **L0**（`of()` 建 PENDING_FINANCE）。

### AC2 — 财务对账流转 PENDING_FINANCE→PAID→ARCHIVED + 凭证 + 审计

**Given** `settlement.view`（看）/`settlement.payout`（转）门控
**When** 财务确认打款（填凭证）/ 归档
**Then** `PENDING_FINANCE`--确认(填 payment_proof)-->`PAID`(记 paid_at/settled_by)--归档-->`ARCHIVED`(记 archived_at)；**非法跃迁拒**（如 PENDING_FINANCE 直接归档、已 PAID 重复确认）；每步 `AdminAuditService` 哈希链
> 验证层：**L1**（确认→PAID+凭证+审计；归档→ARCHIVED+审计；越级/重复 422）+ **L0**（状态机 guard 单测）。

### AC3 — 兽医端状态映射不变（App 零改）

**Given** 兽医收入页读 `VetIncomeResponse`
**When** 财务把某月结 PAID/ARCHIVED
**Then** 兽医侧该月显 `SETTLED`（映射：PENDING_FINANCE→PENDING、PAID/ARCHIVED→SETTLED）；当月实时聚合恒 `PENDING`
> 验证层：**L0**（`ofSettlement` 映射 3→2 态）+ **L1**（income 端点 history status 仍 PENDING/SETTLED）。

## Dev Notes

- **Flyway 号权威**：V80（当前 max V79=9-3）。决策 E2 顺延。
- **值迁移顺序**：先 DROP 旧 CHECK 再 UPDATE 值再 ADD 新 CHECK（否则 UPDATE 撞旧约束）。
- **vet 端零改是硬约束**：App 按 SETTLED 判已结算；映射收口在 `ofSettlement` 单点。
- **打款凭证**：`payment_proof` 存参考号/说明/URL（VARCHAR），本 story 不做文件直传。
- **审计**：确认/归档接 `AdminAuditService` 哈希链（§7，资金写操作 NFR-4）。
- **实际 disburse**：本 story 是**对账台账 + 状态流转 + 凭证登记**（财务线下/Iris 打款后回填凭证）；不在此发起真实转账（与退款 payout 的 Iris 集成解耦，AB-8D 只要台账可对）。
