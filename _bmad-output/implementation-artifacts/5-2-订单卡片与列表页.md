## Story

As a 用户,
I want 在订单中心一眼看清每类订单的类型与状态,
so that 我一眼看懂每笔支付（UX-DR1/2）。

## Acceptance Criteria

1. **前端：订单列表页 `p-orders`（消费 5-1）（L0/L2）**
   **Given** 用户进入订单中心
   **When** 加载 `GET /orders`
   **Then** 顶部 **PawCoin 余额块**（koin，只读，跳充值/余额页可选）+ **类型筛选**（全部/兽医/AI/充值）+ 订单卡列表（按后端返回顺序，倒序）+ **游标加载更多**（`hasMore` 时「加载更多」，照 `pawcoin_controller` StateNotifier loadMore）（**L0/L2**）
   **And** 空态友好文案（无订单）；加载态；错误/离线态可重试（**离线不留白不转圈**，UX-DR6 承接）（**L0/L2**）

2. **前端：订单卡片组件（4 类型 + 3 状态色 + 退款 chip）（L0）**
   **Given** 一条 `OrderSummaryView`
   **When** 渲染卡
   **Then** **类型图标**（4 类：兽医/AI/充值/HD 预留）+ **本地化 title/subtitle**（按 orderType+statusCode，**不渲染后端串**）+ **金额**（`amount` 非 null 显 `Rp...`，null 显 "Belum ada pembayaran"）+ **状态徽章**（statusColor→色：SUCCESS 绿/WARN 黄/**INFO 紫柔底非红**）（**L0**）
   **And** 兽医 `REFUNDING`（statusColor=INFO）→ 退款处理中 chip **info 蓝非 error 红**（DESIGN.delta 组件②，纯展示不可点）；`COMPLETED_REFUND_REJECTED` → 提示「退款申请未通过」样式（**L0**）

3. **前端：类型筛选 + 游标翻页正确性（L0）**
   **Given** 筛选 chip
   **When** 切类型
   **Then** 重新拉 `?type=VET_CONSULT|AI_UNLOCK|PAWCOIN_TOPUP`（全部=无 type）→ 列表重置 + 游标重置；加载更多带 `cursor=nextCursor` 追加无重复（**L0**）
   **And** 卡片点击 → 跳详情 `/me/orders/{orderToken}`（5-3 建；本 story 预留导航，详情页 5-3）（**L0**）

4. **前端：契约 + 静态/端到端验收（L0/L2）**
   **Given** 全部实现
   **When** 验证
   **Then** `flutter analyze` 零警告 + `flutter test` 绿（模型 fromJson + statusColor→色映射 + 卡片渲染 widget 测试）（**L0**）
   **And** microcopy 契约（EN/ID key 对齐 + ≤1 emoji）通过；模拟器：列表渲染 + 3 类卡 + 退款中 info 蓝 + 类型筛选 + 加载更多 + 空态（**L2 留本地**）

## Dev Notes

> 纯前端，消费 5-1 `GET /orders`。功能优先（无 pages/ 原型）；紫色主题 + 既有徽章 token；i18n 按 code 本地化不渲染后端串。

### 承接现状（5-1 契约，照此消费，勿改后端）

- **接口**：`GET /api/v1/orders?type=&cursor=&limit=`（JWT USER，AuthInterceptor 自动带 token/续期）。响应 `OrderPage`{`items`,`nextCursor`,`hasMore`,`pawcoinBalance`}。`items[i]`=`OrderSummaryView`{`orderType`,`orderToken`,`statusCode`,`statusColor`,`amount`(可空),`payChannel`,`createdAt`}。
- **statusColor 后端权威**：WARN/INFO/SUCCESS。前端只**映射颜色 + 图标**，不重算状态语义。**兽医 REFUNDING→INFO**（退款中蓝非红）。
- **statusCode（前端本地化 + 图标/文案分支）**：兽医 `IN_PROGRESS`/`COMPLETED`/`COMPLETED_REFUND_REJECTED`/`REFUNDING`/`REFUNDED`；AI/充值 `COMPLETED`/`PAID`（5-1 仅纳终态）。App 按 `orderType`+`statusCode` 本地化 title/subtitle（[[petgo-i18n-model-and-debt]]，**绝不渲染后端 title/subtitle**——5-1 契约也不下发）。
- **游标翻页**：首页 `cursor` 不传；`loadMore` 带 `cursor=nextCursor`；`hasMore=false` 收尾。照 `pawcoin_controller` StateNotifier（items 追加、loadingMore/loadMoreFailed 态）。
- **PawCoin 余额**：`pawcoinBalance`（koin，1:1）顶部只读块。可复用 `refund_format.rpFmt` 千分位或既有 pawcoin 展示。

### 关键设计决策（dev 照此）

- **D-1 statusColor→实际色（复用徽章 token）**：`SUCCESS`→`momenBadgeBg`/`momenBadgeText`（绿）；`WARN`→`goldTint`/`tipsBadgeText`（黄）；**`INFO`→`mintTint`/`mint600`（紫柔底，非红——退款中不制造焦虑，UX-DR2）**。无独立「蓝」token；紫柔底满足「info 非 error」核心诉求。未知色兜底中性 `line2`/`muted`。
- **D-2 4 类型图标 + 本地化**：`orderType`→图标（兽医 medical/pet、AI sparkle、充值 wallet/savings、HD id-card 预留）+ label（l10n）。ID_HD 类型本 story 无数据（5-1 不返），代码兜住但不显。
- **D-3 amount 占位**：`amount!=null`→`rpFmt`；`amount==null`→ l10n "Belum ada pembayaran"。本 story 3 类恒非 null，占位为泛化/HD 预留（勿删，防未来 null 卡崩）。
- **D-4 卡片点击导航**：`/me/orders/{orderToken}`（5-3 详情页建）。本 story 加导航 push，详情页 5-3 落地（本 story 若 5-3 未建，路由暂缺 → 可先建占位 route 或本 story 只 push 不建目标，dev 定；建议 push，5-3 补目标页）。
- **D-5 空/错/离线态**：空→友好文案 + 图标；错误/离线→重试按钮（**不留白不转圈**）。照 `refund_list_page` 空/错态范式。
- **D-6 类型筛选**：chips（全部/兽医/AI/充值，HD 预留隐藏）。切换重置列表 + 游标。
- **D-7 前端模块**：新 `features/order/`（与后端 order 包同名，PawCoin 独立不并）。Riverpod StateNotifier + 不可变 model + dio ProblemDetail 映射（继承 V1.0）。
