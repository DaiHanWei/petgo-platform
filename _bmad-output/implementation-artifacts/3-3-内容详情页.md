# Story 3.3: 内容详情页

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a **用户**,
I want **点击 Feed 卡片查看内容全文与评论**,
so that **我能完整阅读并参与互动**。

> 本 Story 实现内容详情页（FR-28）——Feed 卡片点击后的全屏阅读 + 互动容器。**前端重、后端中**：后端提供详情读取端点 + 评论分页读取（评论**写入**在 Story 3.5）；前端实现详情布局、多图左右滑、评论区分级加载、底部评论框、返回保持滚动位置，并显式覆盖**多态完整性**（不存在/已删除/无权限/失效）。
>
> **做**：① 详情页结构（顶部导航栏含返回 + 「···」举报入口位、作者信息区点击触发位、正文、多图左右滑角标 x/y、互动栏占位、评论区、底部固定评论框）；② 评论区读取分页（首载 10 条一级、超出「查看更多」每次 10 条；二级默认 3 条、超出「查看全部 X 条回复」）；③ 返回 Feed 保持滚动位置；④ 未登录点评论框触发 FR-0C；⑤ 多态：404 不存在/已删/下架/注销 → 「这条内容已不存在」占位页；403 越权 → 友好无权限页（UX-DR18 ④⑤）。
> **不做**：点赞**写入**逻辑（Story 3.4，本 Story 互动栏含按钮 UI 占位 + 显示计数读取）；评论/回复**写入与删除**（Story 3.5，本 Story 仅**读取**展示评论）；举报**提交**（Story 3.7，本 Story 仅放「···」举报入口位）；迷你主页卡（Story 3.8，本 Story 仅放作者点击触发位）；内容删除（Story 3.6，本 Story 「···」放作者删除入口位）。
> **依赖前序**：Story 3.2（Feed 卡片 + 跳详情导航 + 滚动位置保持基础）、Story 2.3（`content_posts` 表）、Story 1.5（游客只读 + 受控门控）、Story 1.4（FR-0C 强登录弹窗）。

## Acceptance Criteria

> **验证层标注**：**L0**（编译/lint/widget test/Service 单元 + MockMvc 切片） · **L1**（需 postgres+redis，验详情读取 + 评论分页 SQL + 404/403 真实链路） · **L2**（真机，多图滑动手势/返回滚动位置）。
> 详情与评论读取、多态 HTTP 语义为 **L1**；布局/多图角标/返回保持位置为前端 **L0 + L2**。无外部第三方依赖。

### AC1 — 详情页结构

**Given** 用户在 Feed 点击内容卡片
**When** 全屏跳转至详情页
**Then** 从上至下展示：顶部导航栏（返回 + 「···」菜单含举报入口）、作者信息（点击触发 FR-26）、正文、图片（多张左右滑动，角标 x/y）、互动栏（点赞按钮+数 / 评论数）、评论区、底部固定评论输入框（FR-28）
**And** V1 详情页不含社区视频展示
> 验证层：布局结构/多图左右滑/角标 x/y/底部固定评论框为 **L0 widget test** + **L2 真机滑动**；详情数据读取（正文/图片列表/互动计数/作者）为 **L1**（`GET /content-posts/{id}` 返回完整详情）。「···」举报入口、作者点击、点赞按钮在本 Story 为**占位 UI**（行为分别在 3.7/3.8/3.4）。

### AC2 — 返回 Feed 保持滚动位置

**Given** 用户在详情页点击返回
**When** 回到 Feed
**Then** 保持 Feed 原滚动位置（不回顶）
> 验证层：**L2 真机**（go_router push/pop 不重建 Feed、ScrollController/PageStorageKey 保位）+ **L0**（路由为 push 非 replace 的单元断言）。

### AC3 — 评论区分级加载

**Given** 评论区
**When** 加载评论
**Then** 首次加载 10 条一级评论，超出显示「查看更多评论」每次加载 10 条；二级评论默认展示前 3 条，超过显示「查看全部 X 条回复」
**And** 未登录用户点击评论输入框触发 FR-0C 登录引导
> 验证层：评论分页（一级 10/批游标、二级首 3 条 + count）为 **L1**（须真实评论数据断言分页边界）；「查看更多」「查看全部 X 条」交互为 **L0 widget test**；未登录点评论框触发 FR-0C 为 **L0**（门控分支）+ **L2**。本 Story 评论**只读**展示，写入在 Story 3.5。

### AC4 — 多态：不存在/已删/下架/越权

**Given** 用户从陈旧 Feed 卡、推送深链进入，或内容已被作者删除（FR-36）/ 运营下架（FR-25）
**When** 后端返回 404（不存在/已删）
**Then** 展示"这条内容已不存在"占位页 + 返回 Feed 按钮，不崩溃不白屏（UX-DR18 ④）
**And** 越权访问（403）展示友好无权限提示 + 返回（UX-DR18 ⑤）
> 验证层：404（含 `deleted_at` 非空软删、不存在 id、运营下架）与 403 的后端语义为 **L1**（统一文案不暴露资源是否曾存在以防枚举）；前端占位页/无权限页渲染为 **L0 widget test**。这是「规格页面多态完整性」要求的显式覆盖——**不只 happy path**。

### AC5 — 「···」菜单按内容归属分支 + 当天详情页入口承接 `[R2·回改]`

> **代码已实现，本轮回改补齐**（review→ready-for-dev）。明确「···」菜单项随内容归属（他人 vs 本人）分支显示，并承接成长档案「当天详情页」跳转来的条目。

**Given** 用户在内容详情页顶部「···」菜单
**When** 展开菜单
**Then** **他人内容**（`authorId != currentUserId`）：菜单含**举报**入口（FR-25，行为接 Story 3.7）
**And** **自己内容**（`authorId == currentUserId`）：举报项**替换为删除**入口（FR-36，行为接 Story 3.6）——同一菜单位按归属互斥显示，绝不同时出现举报与删除
**And** 未登录游客查看他人内容时「···」举报项点击触发 FR-0C
> 验证层：**L0**（widget test：构造 `authorId==currentUserId` → 菜单出现删除项、无举报项；`authorId!=currentUserId` → 出现举报项、无删除项；注销作者帖 → 他人分支。举报/删除的**提交行为**仍分别在 3.7/3.6，本 Story 仅菜单项归属分支可见性）。任务：前端。

**Given** 用户在成长档案「当天详情页」（Story 2.4，FR-37）看到当天发布的内容条目
**When** 点击该条目
**Then** 进入本内容详情页（复用 `/content/:id` push 路由 + 多态覆盖：被删/下架 → AC4 的 404 占位页）
> 验证层：**L0**（路由单元：当天详情页条目点击 → `context.push('/content/:id')`，返回回到当天详情页保位）+ **L2 真机**（从档案当天详情页跳详情再返回）。任务：前端。

## Dev Notes

### 关键架构约定（本 Story 必须落实）

**详情页结构**（FR-28）：返回栏（含举报入口）+ 作者（触发 FR-26）+ 正文 + 多图左右滑（角标 x/y）+ 互动栏 + 评论区（首载 10 条一级、二级默认 3）+ 底部固定评论框；返回 Feed 保持滚动位置。本 Story 搭**容器与读取**，互动/举报/迷你卡/删除的**行为**分散在 3.4/3.7/3.8/3.6——本 Story 放**入口位**，避免重复。

**评论读取契约**（FR-28）：一级时间**正序**（区别于 Feed 倒序）+ 游标分页 `{items,nextCursor,hasMore}` 首批 10；二级默认 3 + replyCount + 展开端点。二级**不再嵌套**（最多两级，Story 3.5 写入侧强约束）。

**多态完整性**（UX-DR18 + 项目记忆「规格页面多态完整性」）——本 Story **显式覆盖非 happy path**：
- ④ 不存在/已删/下架/注销内容清除 → **404**，统一文案不暴露资源是否曾存在（防枚举）。
- ⑤ 越权 → **403** 友好页（登录拦截走 FR-0C，二者不混用）。
- ⑥ 推送深链目标失效 → 失效页 + 返回（Epic 6 深链落地处复用本占位页）。
- 注销作者但内容匿名化保留 → **200 显「已注销用户」**，非 404（区分"内容被删"与"作者注销内容留存"）。

**命名映射链**：`content_posts`/`comments`(parent_id 自引用二级、author_id、created_at、deleted_at)；DB snake_case ↔ JSON camelCase；ProblemDetail 404/403。

**前端**：go_router push（二级页隐藏 Tab Bar，架构 §Frontend）；`AsyncValue` 三态；返回保位用 PageStorageKey；多态页复用 `shared/widgets` 占位。

### 强制护栏（架构 §Enforcement —— 违反即返工）

- **禁止擅自引入 MQ / 通用缓存层 / 新中间件**；详情/评论直读 PostgreSQL。
- `ddl-auto=validate`；评论表/索引变更走 Flyway（与 3.5 协调建表归属，避免重复迁移）。
- 模块间只经 service/事件，禁跨模块直接访问对方 repository；作者昵称/头像经 service 或写入冗余获取（数据边界）。
- 多态 404 **统一文案防枚举**，绝不暴露资源是否曾存在。
- 注销内容匿名化（NFR-8）：作者注销内容留存仍 200，投影「已注销用户」；点击其头像不触发迷你卡（Story 3.8/7.3）。
- 对外标识：详情 id 为已授权资源可暴露数字 id（架构 §Format：已授权资源可暴露数字 id）；推送深链用 token（Epic 6）。

### 范围边界（防 scope creep —— 本 Story 明确不做）

- ❌ 点赞**写入/切换**（Story 3.4）——本 Story 互动栏点赞按钮为 UI 占位 + 读取计数/liked 态。
- ❌ 评论/回复**写入与删除**（Story 3.5）——本 Story 评论**只读**展示 + 底部框门控。
- ❌ 举报**提交**（Story 3.7）——「···」仅放举报入口位。
- ❌ 迷你主页卡（Story 3.8）——作者点击仅放触发位。
- ❌ 内容删除（Story 3.6）——「···」作者删除仅放入口位。
- ✅ 只做：详情容器 + 详情/评论**只读** + 多图滑动 + 返回保位 + 评论框门控 + **多态完整覆盖**。
