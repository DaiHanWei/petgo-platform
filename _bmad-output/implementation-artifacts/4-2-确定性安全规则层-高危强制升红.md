# Story 4.2: 确定性安全规则层（高危强制升红）

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a **平台与用户**,
I want **一个独立于 AI 模型的安全兜底层**,
so that **即使模型假阴性，高危症状也必被升级为红色，避免延误就医**。

> 本 Story 是 PetGo 的**第一根生命安全支柱、NFR-6 不可协商地基**——`SafetyRuleLayer` 依据 `resources/safety/high_risk_symptoms.yml`（兽医顾问维护）对 Gemini 返回结果做**后置确定性校验**，命中高危清单则**强制升红、只升不降、独立于 Gemini、绝不被业务代码绕过**。本 Story 把 Story 4.1 预留的"Gemini 返回解析后、写库 DONE 前"挂载点填充为真正的强制升红逻辑。
>
> 前置依赖：**Story 4.1（分诊异步基建 + `TriageService` 后置校验挂载点 + `triage_tasks.danger_level` 落库）必须先就位**。本 Story 几乎全在后端；前端无独立 UI（红色态展示在 4.5，本 Story 只保证升红后的数据被下游一律按红处理）。

## Acceptance Criteria

> **验证层标注**：每条 AC 末尾标注验证层级与所需本地环境——
> **L0 静态**（清单解析 / 规则匹配纯函数单测，无需 DB） · **L1 集成/运行时**（需 Docker daemon + postgres + redis，验整条分诊链路升红落库） · **L2 端到端/外部凭证**（需真实 Gemini API key + 私密桶签名 URL + 真机）。
> 本 Story 的核心安全断言（命中清单 → 无论模型返回什么，最终必为 RED）**最强验证落在 L0/L1**——这正是确定性安全层的价值：它**不依赖** Gemini 真实返回即可证明兜底成立（用打桩 Gemini 喂"绿/黄"也必被升红）。L2 仅做一次真实链路冒烟确认挂载点未被旁路。

### AC1 — 后置确定性升红（命中清单强制 RED，只升不降，独立 Gemini）

**Given** 分诊任务的症状文字/解析结果
**When** Gemini 返回结果后进入后置校验
**Then** `SafetyRuleLayer` 依据 `resources/safety/high_risk_symptoms.yml` 清单（覆盖最高频致死 15–20 急症：误食巧克力/葡萄/木糖醇/百合、呼吸困难、呕吐带血、犬胃扭转、猫尿道阻塞等）命中则强制升红（NFR-6）
**And** 规则层只许往上兜、永不往下降，独立于 Gemini，绝不被业务代码绕过
> 验证层：清单加载 + 关键词/模式匹配 + "只升不降"纯函数（输入 danger_level + 命中标志 → 输出 danger_level）= **L0**（单测穷举：GREEN+命中→RED、YELLOW+命中→RED、RED+命中→RED、任意+未命中→原值不变；断言**永不出现 RED→YELLOW/GREEN 的下调**）。整条 `@Async` 链路调用到该层并落库 = **L1**。

### AC2 — 模型给绿/黄但命中高危 → 最终落库 RED，下游按红处理

**Given** 命中高危规则但模型给出绿/黄
**When** 后置校验执行
**Then** 最终 danger_level 被覆盖为 RED 并落库，下游一律按红色态处理
> 验证层：**L1**（打桩 Gemini **故意**返回 GREEN/YELLOW + 喂命中清单的症状文字 → 验 `triage_tasks.danger_level` 落库为 RED、`GET /triage/{id}` 返回 RED → 4.5 红色半屏据此触发）。**L0** 可单测覆盖层本身的覆盖逻辑；"下游一律按红"经 L1 验取结果端点只读最终落库值（无绕过快路径）。

## Tasks / Subtasks

> **按"后端子任务 / 前端子任务 / 联调验收"三段组织**。本 Story 是 **Epic 4 安全核心、后端重**——几乎全部在后端 `triage/service/SafetyRuleLayer` + 清单资产；前端无新 UI，仅确认升红数据驱动 4.5 红色态。建议执行顺序：清单与规则层 → 接入 4.1 挂载点 → 联调（重点验绕不过 + 只升不降）。

### 🟦 后端子任务（petgo-backend / triage/service/SafetyRuleLayer）

- [ ] **B1. 高危症状清单资产 `high_risk_symptoms.yml`** (AC: 1)
  - [ ] `src/main/resources/safety/high_risk_symptoms.yml`——结构化清单，覆盖**最高频致死 15–20 急症**，每条含：稳定 `id`（如 `chocolate_ingestion`）、`labels`(id/en 双语人类可读名)、匹配信号（关键词/同义词集合，支持 id+en，如「呼吸困难/sesak napas/labored breathing」「呕吐带血/muntah darah」「误食巧克力/葡萄/木糖醇/百合」「犬胃扭转/GDV」「猫尿道阻塞/urethral obstruction」等），可选 `notes`（兽医顾问维护备注）。
  - [ ] 文件头注释明确：**本清单由兽医顾问维护，是 NFR-6 安全地基，修改需兽医评审**；列出本批覆盖的 15–20 急症清单。
  - [ ] ⚠️ 清单为**可演进资产**：规则层从清单读取，**严禁把规则硬编码进 Java**（便于兽医顾问后续增删条目而不改代码）。
- [ ] **B2. 清单加载与校验** (AC: 1)
  - [ ] `triage/service/SafetyRuleLayer` 启动时加载并**校验清单结构**（缺字段/空清单 → 启动失败 fail-fast，而非静默放过——安全层缺清单等于失守，宁可不启动）。
  - [ ] 清单解析为内存匹配结构（关键词/同义词归一化：小写、去空白、id+en 统一比对）。
- [ ] **B3. 确定性匹配与「只升不降」强制升红核心** (AC: 1, 2)
  - [ ] `SafetyRuleLayer.enforce(symptomText, parsedResult, modelDangerLevel) → finalDangerLevel`：
    - 匹配信号源 = **症状原文（用户输入）+ Gemini 解析结果文本**（双源，防模型漏读用户原话）。
    - 命中任一高危信号 → `finalDangerLevel = RED`。
    - **（决策 E5）保守否定语境处理**：仅当高危词被**直接紧邻否定**（如「没有/未/无 + 呼吸困难」「没吃到 + 巧克力」，id/en 否定词表）**且该条全文无任何其他高危命中**时，才不计该次命中；**只要全文存在任一未被否定的高危命中，一律升红**。安全攸关默认「存疑即升红」——否定处理只为削减明显误报（「狗没有呼吸困难」），**绝不能因否定逻辑漏掉真实急症**（如「没有呼吸困难，但呕吐带血」必须因「呕吐带血」升红）。否定窗口取紧邻 N 词，跨句不否定。
    - **只升不降铁律**：`final = max(model, ruleResult)`（按 GREEN<YELLOW<RED 序），**永不下调**——即使模型返回 RED 而规则未命中，也保持 RED。该方法为**纯确定性、无副作用、不调 Gemini、不查 DB**。
  - [ ] 单测穷举所有 (model ∈ {G,Y,R}) × (命中 ∈ {true,false}) 组合，断言无任何路径出现下调。
  - [ ] 记录"是否因规则层升红"标志（如 `parsed_result.escalatedBySafetyRule=true`），供审计/可观测（**不落症状健康数据**，仅落布尔 + 命中规则 id）。
- [ ] **B4. 接入 Story 4.1 的后置挂载点（唯一、不可旁路）** (AC: 1, 2)
  - [ ] 在 `TriageService` 的「Gemini 返回解析后、写库 DONE 前」**唯一**挂载点调用 `SafetyRuleLayer.enforce(...)`，用其返回值作为**最终** `danger_level` 落库。
  - [ ] **删除/封堵任何"模型 danger_level 直接落库"的旁路**：确保不存在第二条写 `danger_level` 的代码路径绕过本层（4.1 已收敛为单点，本 Story 复核并加防回归测试）。
  - [ ] `GET /triage/{id}` 取结果**只读最终落库值**（不暴露模型原始裁决作为对外结果），确保下游一律按落库后的 RED 处理。
- [ ] **B5. 防回归护栏（可测的"绕不过"）** (AC: 1, 2)
  - [ ] 加一条 ArchUnit 风格或单测断言：写 `triage_tasks.danger_level` 的最终值必经 `SafetyRuleLayer.enforce`（或等价不变量测试——构造命中输入跑完整 service 必得 RED）。
  - [ ] 边界与多态（参 spec-page-state-completeness）：清单缺失/为空 → fail-fast；症状文字为空但解析结果命中 → 仍升红；模型返回非法/缺 danger_level → 视为最低级 GREEN 再过规则层（不因模型异常而漏兜）。

## Dev Notes

### 关键架构约定（本 Story 必须落实）

**分诊数据流（架构 §Integration Points 数据流-1）**：…→ `@Async` 调 Gemini → 解析绿/黄/红 → **SafetyRuleLayer 后置强制升红** → 写库 status=DONE → 客户端取结果。本 Story 实现倒数第二步，是整条链的安全闸门。

**结构（架构 §Project Structure / §Requirements Mapping）**：`triage/service/SafetyRuleLayer` + `resources/safety/high_risk_symptoms.yml`（兽医顾问维护）；跨切面映射明确标注此二者为 NFR-6 落点。

**命名映射链**：`danger_level` ∈ {GREEN,YELLOW,RED}（varchar+UPPER_SNAKE）；JSON `dangerLevel`；不变量"只升不降"以 GREEN<YELLOW<RED 全序定义。

### 🔒 安全攸关约束（不可协商）

> 本 Story **就是** NFR-6 不可协商地基本体。以下每条均为生命安全红线，违反即返工且需安全复审。

- **后置校验，不是前置/并行**：`SafetyRuleLayer` 置于 **Gemini 返回之后**做后置强制升红——它校验的是模型已产出的结果，而非替代模型。挂载点是 4.1 预留的唯一一处（Gemini 解析后、写库 DONE 前）。
- **只升不降，永不下调**：`final = max(model, rule)`（GREEN<YELLOW<RED）。**绝不允许**任何代码路径把已经是 RED 的结果调回 YELLOW/GREEN，也不允许规则层把模型的 RED 降级。这是"只许往上兜、不许往下降"的字面实现。
- **独立于 Gemini**：规则层为**确定性纯函数**——不调 Gemini、不依赖模型置信度、即使 Gemini 完全失灵/假阴性也独立成立。用打桩 Gemini 喂任意绿/黄都必被升红，正是其价值。
- **绝不被业务代码绕过**：写 `danger_level` 的最终值**必经** `enforce`；封堵一切旁路、加防回归不变量测试。`GET /triage/{id}` 只读经规则层落库后的最终值，不暴露模型原始裁决作为对外结果。
- **清单独立、兽医顾问维护、覆盖 15–20 最高频致死急症**：规则从 `resources/safety/high_risk_symptoms.yml` 读取，**严禁硬编码进 Java**；文件头注明维护责任与评审要求。清单缺失/为空 → **fail-fast 拒绝启动**（安全层失守宁可不上线）。
- **不误升以保红色信任（NFR-9 联动）**：红色态零变现、强提醒，故升红须基于清单命中而非泛化恐慌；未命中且模型非红则保持模型值，避免红色滥用稀释其严肃性。
- **日志不落健康数据**：审计仅落"是否升红 + 命中规则 id"布尔/标识，**不落症状原文/图片/解析健康内容**。

### 强制护栏（架构 §Enforcement —— 违反即返工）

- **绝不改动/绕过 AI 安全规则层的"只升不降"语义**（架构 Enforcement 明列）。
- 安全清单与规则解耦：规则层从 yml 读取，不硬编码；清单为可独立演进资产（兽医顾问增删条目不需改代码）。
- 日志/对象存储不落 PII/健康数据/令牌/签名 URL。
- 不引入 MQ/缓存/新中间件（本 Story 为纯同步后置校验，无异步新增）。

### 范围边界（防 scope creep —— 本 Story 明确不做）

- ❌ 不做分诊异步基建 / Gemini 接入 / `triage_tasks` 建表（**Story 4.1** 已完成，本 Story 仅接入其挂载点）。
- ❌ 不做红色半屏 UI / 导航 / 无障碍播报（**Story 4.5**）——本 Story 只保证 RED 数据正确落库并驱动下游。
- ❌ 不做绿/黄结果展示（**Story 4.4**）。
- ❌ 不做问诊入口/上传（**Story 4.3**）。
- ❌ 不扩展清单超出 15–20 急症范围（V1 起步范围，后续由兽医顾问演进——本 Story 只搭可演进结构 + 起步条目）。
- ✅ 只做：`high_risk_symptoms.yml` 起步清单 + `SafetyRuleLayer` 确定性后置强制升红（只升不降、独立、不可旁路）+ 接入 4.1 唯一挂载点 + 防回归不变量。
