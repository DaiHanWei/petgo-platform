# Story 2.5: 问诊记录存档承接与健康事件时间线

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a **用户**,
I want **把问诊结果存入宠物档案并在时间线看到健康事件**,
so that **宠物的健康历史得以沉淀为我不愿离开的资产**。

> 本 Story 在 profile 侧**承接**问诊存档：建健康事件数据落点、存档 UI（一次性弹窗）、IM 图复制到私密桶（`ImToOssArchiver`，2.1 占位本 Story 实现）。**触发点来自 Epic 4（AI 分诊结束）/ Epic 5（兽医咨询结束）**——本 Story 提供"被触发后存档"的 UI + 数据 + 桥接能力，**不实现分诊/咨询本身**。健康事件条目样式已在 2.4 渲染，本 Story 让其有真实数据。
>
> **依赖前序**：1.1、1.3、2.1（私密桶/签名 URL/`ImToOssArchiver` 占位）、2.2（pet_profiles，存档归属）、2.4（健康事件条目样式 + 时间线聚合接口）。触发上游 → Epic 4/5（本 Story 提供 service 接口供其调用）。

## Acceptance Criteria

> **验证层标注**：**L0 静态** · **L1 集成/运行时**（Docker+postgres+redis）· **L2 端到端/外部凭证**（真实 OSS 私密桶 + 腾讯 IM 凭证）。
> 健康事件写库/一次性弹窗逻辑落 **L0/L1**；IM 图复制到私密桶的真实跨云复制需 **L2**（真实腾讯 IM 取图 + 真实 OSS 私密桶落桶）。

### AC1 — 存档弹窗仅一次（按用户状态三态）

**Given** 一次问诊结束（触发点来自 Epic 4 AI 分诊 / Epic 5 兽医咨询）
**When** 系统按当前用户状态弹出对应存档弹窗
**Then** 弹窗仅出现一次，用户做出决策后不再重复询问（FR-16）；按状态分三态：
> 🔄 **PRD V1.0.0 修订（F6/三态 · 2026-06-08）：** 存档弹窗从「单一文案」扩为按用户状态三态，覆盖未建档与非 A 态用户的存档承接路径（与 2.2 单宠物建档 + FR-0G 状态切换串接）。
>
> - **① 状态 A + 已建档**：弹「是否将本次咨询存入 [宠物名] 的档案？」+「存入」/「跳过」（同旧逻辑）。
> - **② 状态 A + 未建档**：弹「创建档案后可保存本次记录」+「立即创建」/「跳过」。「立即创建」跳 FR-11 建档流程（2.2），建档成功后**自动回灌存档**（见 AC3）。
> - **③ 状态 B/C（无/未切有宠物）**：弹「切换为有宠物并创建档案后可保存」+「去创建」/「跳过」。「去创建」先走 FR-0G（系统切状态为 A）再跳建档，建档成功后同样**自动回灌存档**（见 AC3）。
>
> 验证层：**L1**（存档决策落库 `archive_decision ∈ {ARCHIVED, SKIPPED}` 持久化幂等：同一问诊已决策则不再弹，需 postgres）+ **L0**（按用户状态选三态文案 + 决策后置位 widget 测试；②③ 弹窗 CTA 路由分支 widget 测试）。

### AC2 — 存入写健康事件 + IM 图复制私密桶

**Given** 用户选择「存入」
**When** 存档执行
**Then** 以健康事件形式写入成长时间线，内容含日期、症状描述、AI 评级、处理建议摘要
**And** 若涉及兽医聊天图，将所需图从腾讯 IM 复制一份到 OSS 私密桶，档案只引用应用自有 URL（不引用会过期的 IM URL）
> 验证层：**L1**（健康事件写 `health_events` 表 + 2.4 时间线读得倒序条目，需 postgres）+ **L2**（`ImToOssArchiver` 从真实 IM 拉图 → 落真实 OSS 私密桶 → 健康事件 `image_keys` 存自有 key、展示走签名 URL，绝不存 IM URL）。

### AC3 — 先建档 → 建档完成自动回灌存档（延迟串接）

> 🔄 **PRD V1.0.0 修订（三态承接 · 2026-06-08）：** 新增「先建档→建档完成回灌存档」延迟串接，承接 AC1 的 ② A 未建档 / ③ B/C 路径。

**Given** AC1 的 ② A 未建档点「立即创建」或 ③ B/C 点「去创建」（B/C 先走 FR-0G 系统切状态为 A）
**When** 用户跳转 FR-11 建档（2.2）并成功创建宠物档案
**Then** 系统用**同一 `sourceRef`** 自动回调执行 `recordDecision(ARCHIVED)`，把挂起的存档意图回灌到新建档案的时间线
**And** 回灌经幂等键 `uq_health_events_source_ref` 保证：同一 sourceRef 仅落一条，重复回调/重复触发不重存；用户中途放弃建档则挂起意图不落库（无副作用）
> 验证层：**L1**（pending 存档意图 sourceRef 挂起 → 建档成功回调 → `health_events` 落一条 ARCHIVED + 2.4 时间线显示；同 sourceRef 重复回调幂等不重存，需 postgres）+ **L0**（建档成功回调串接 recordDecision 的 provider/widget 测试；放弃建档无回灌）。

### AC4 — 红色态结果页存档承接 + 跳过庆祝页回灌（FR-3 红色态 + F15） `[R2]`

> 🆕 **PRD V1.0.0 第二轮断档补齐（FR-3 红色态承接 + F15 · 2026-06-08）：** 承接 FR-3 红色态结果页「存入档案」入口，按用户状态分流；经 FR-16 触发的建档完成「**跳过庆祝页**」直接回灌本次问诊记录并返回问诊结果页（与 F15、AC1/AC3 三态承接一致）。

**Given** FR-3 红色态分诊结果页含「存入档案」入口
**When** 用户点击「存入档案」
**Then** 按用户状态分流：**状态 A 已建档** → 直接存入（无弹窗，调 `recordDecision(ARCHIVED)`）；**状态 A 未建档 / B/C** → 触发 FR-16 弹窗（按 AC1 ②③ 三态文案引导建档）
**And** 经 FR-16 触发的建档完成「**跳过庆祝页**」（F15）→ 用同一 `sourceRef` 自动回灌本次问诊记录（写 `health_events`）→ 返回问诊结果页（不进首页庆祝页流程）
**And** 回灌复用 AC3 的幂等键 `uq_health_events_source_ref`，同 sourceRef 仅落一条；放弃建档则挂起意图不落库
> 验证层：**L1**（存档落 `health_events` + 红色态先建档→跳过庆祝页→回灌→返回结果页串接，需 postgres）+ **L0**（「存入档案」入口按用户状态分流 widget 测试：A 已建档直存无弹窗 / A 未建档 / B/C 触发 FR-16；建档完成跳过庆祝页回灌的串接 widget 测试）。

## Dev Notes

### 架构约定

- **桥接规则（架构 §Data Architecture 明确）**：问诊存档时把所需图从 IM **复制一份到②私密桶**，档案只引用**应用自有 URL/key**（不引用会过期的 IM URL）。`ImToOssArchiver` 即此桥接，2.1 占位、本 Story 实现。
- **跨模块通信**：存档由 Epic4/5 触发——优先**进程内领域事件**（`ConsultClosedEvent` → profile `@Async @EventListener` 存档；分诊完成事件类似），与架构「ConsultClosedEvent → profile 订阅做存档桥接」「禁跨模块 repo」一致。
- **命名**：表 `health_events`、枚举 UPPER_SNAKE varchar、JSONB `image_keys`、`source_ref` 幂等键；DTO record；UTC ISO；camelCase。
- **私密健康图**：仅②私密桶 + 短 TTL 签名 URL（2.1 `SignedUrlService`），**绝不公开 URL**。

### 强制护栏

- **禁 MQ/缓存层/新中间件**；IM 复制用 `@Async`（异步）但**不引入 MQ**，可靠性靠 DB 状态/幂等键。
- `ddl-auto=validate`，schema 只由本 Story Flyway。
- **日志严禁落健康数据**：症状摘要/AI 评级/处理建议/签名 URL **不进结构化日志检索字段**（架构 §Enforcement 明令）。
- **私密图只走签名 URL**，库存 key 不存 IM/外部过期 URL。
- 存档幂等（source_ref），「只问一次」（FR-16）。
- 删除级联：注销/删档案连带 `health_events` + 私密桶图。

### 范围边界（不做）

- ❌ 不实现 AI 分诊（Epic 4）、兽医咨询/IM 会话（Epic 5）——本 Story 仅承接存档触发。
- ❌ 不做时间线 UI 骨架（2.4 已建，本 Story 只填健康事件数据 + 复用条目样式）。
- ❌ 不做名片 H5（2.6，名片只读快乐时刻，**不含健康事件**）。
- ✅ 只做：`health_events` 表 + 存档决策幂等 service + `ImToOssArchiver` 实现(IM→私密桶) + 一次性弹窗 + 健康事件接入 2.4 时间线 + 供 Epic4/5 调用的 API/事件。
