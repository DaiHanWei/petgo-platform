# TailTopia 运营试用测试流程（UAT）

> 给运营/试用人员的**手把手体验清单**。你**不需要懂技术**——拿着手机，按顺序操作，对照「应该看到」，正常打 √、不对打 ×，× 的填一句「实际看到了啥」并截图。

---

## 一、开始前（5 分钟准备）

1. **装好 App**：拿到测试包（iOS TestFlight / Android apk），装到手机上。
2. **网络**：连正常 Wi-Fi 或 4G/5G。
3. **测试账号**：用一个 Google 账号登录（建议专门的测试 Google 账号，不要用私人常用号——因为后面要测「注销」会删数据）。
4. **语言**：App 支持**英文(EN)**和**印尼语(ID)**。界面会出现这两种文字，**正常情况绝不应出现中文**——看到中文就是 bug，记下来。
5. **准备几张图**：手机相册里准备 3-5 张宠物照片（猫/狗都行），后面发内容、问诊要用。

### 怎么记录问题（每发现一个填一行）

| 截图 | 在哪个页面 | 你做了什么 | 期望看到 | 实际看到 | 严重度 |
|---|---|---|---|---|---|
| （贴图） | 例：首页 | 点了「+」发布 | 弹出发布页 | 闪退了 | 严重 |

> 严重度：**闪退/卡死/数据没了/红色急症页能点出付费** = 最严重；**功能用不了** = 严重；**文案错/显示乱** = 一般；**小瑕疵** = 轻微。

## 二、试用顺序总览

按这个顺序走，前面是后面的基础：

1. 登录 & 新手引导 → 2. 建宠物档案 → 3. 首页 & 社区 → 4. 发布内容 → 5. 成长档案 & 名片 → 6. AI 问诊 → 7. 在线兽医咨询 → 8. 消息通知 → 9. 里程碑成就 → 10. 个人中心 / 语言 / 注销 → 11.（可选）兽医端 → 12. 双语扫雷 → 13. 异常体验

> 每个场景一张表。**步骤列从上往下做。**

## 场景 6 · AI 问诊（重点：红色急症体验）

> 这是安全攸关功能，**红色结果页绝不能出现任何收费/导流**，重点盯。

| # | 你要做的操作 | 应该看到 | 正常? | 备注 |
|---|---|---|---|---|
| 6.1 | 首页「Konsultasi Kilat」或底部「Consult」进问诊 | 问诊入口页，可上传图文描述症状 | □ | |
| 6.2 | 上传宠物照片 + 描述一个**轻微**问题（如「有点没胃口」） | 提交后显示「分析中」，稍等出结果 | □ | |
| 6.3 | 轻微问题的结果 | **绿色**结果：日常建议，不吓人 | □ | |
| 6.4 | 再测一个**中等**问题 | 可能是**黄色**：建议观察，可能带「条件倒计时」提示 | □ | |
| 6.5 | 再测一个**明显危急**描述（如「呼吸困难/抽搐/中毒」） | **红色半屏强提醒**：明确告知尽快就医 | □ | |
| 6.6 | ⚠️ 重点看红色页 | **不能有任何**「升级付费咨询」「找兽医」「立即购买」之类入口；只有等约 5 秒后出现一个「我已知晓」按钮关掉 | □ | |
| 6.7 | 关掉红色遮罩 | 回到结果页，结果还在 | □ | |
| 6.8 | 试一个「带否定」的描述（如「**没有**呼吸困难，只是打喷嚏」） | 不应被误判成急症升红（除非还有别的危急词） | □ | |
| 6.9 | 问诊过程中断网 | 有「加载失败/重试」提示，不白屏不闪退 | □ | |

> 🔴 **如果红色页出现了任何收费或导流入口 → 立即标记为「最严重」并截图。**

## 三、试用收尾

1. 把所有 × 的问题汇总成一张表（用上面「怎么记录问题」的模板）。
2. **最严重的优先**：闪退、数据丢失、**红色急症页出现收费/导流**、注销没删干净、出现中文。
3. 连同截图一起反馈给研发。
4. 注明：你用的是 **iOS 还是 Android**、**哪个语言**、**测试账号**。
