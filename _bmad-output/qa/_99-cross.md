## X2. 🌐 i18n 双语专项（en / id）

> 核心铁律：**App 绝不渲染中文**；后端显示串按 code/typeSlug 本地化，App 不直接渲染后端显示串。

| # | 验收点 | 做法 | 依据 |
|---|---|---|---|
| I18N-1 | 全屏中文扫雷 | 逐屏切 `en` 与 `id`，任何界面**不得出现中文字符**（含里程碑标题、问诊结果、推送、错误提示、空态） | 7.2 · 全局 |
| I18N-2 | 里程碑标题本地化 | 标题按 `milestone_titles.dart` 的 `code` + `typeSlug` 本地化，不渲染后端串；猫/狗/通用清单标题 en/id 各验 | 8.x |
| I18N-3 | 语言切换即时生效 + 持久化 | `/me/language` 切换立即全局生效、重启保持；「跟随系统」随系统语言变 | 7.2 |
| I18N-4 | 硬编码印尼语现状记录 | 部分页面硬编码印尼语（如首页「Konsultasi Kilat」「Untukmu」、档案「Linimasa Tumbuh Kembang」）—— 记录为现状，en locale 下仍显示印尼语属已知债 | i18n 债 |
| I18N-5 | 印尼语翻译复核 | 里程碑等专业术语初稿待母语者复核（`perawatan`/`steril` 等）—— 列为上线前 L2 审校项 | 8.x 盲区 |
| I18N-6 | 推送/深链文案双语 | 各类推送标题/正文 en/id；点击落地正确 | 6.x |

## X3. 🔗 E2E 主链路脚本（端到端贯穿，L2）

> 一条龙跑通核心价值链。建议真后端 + 真机，至少跑 1 遍 en、1 遍 id。

**主链路 A —— 新用户全生命周期：**
1. 游客浏览 Feed → 滑到第 3 页触发软登录浮层
2. Google 登录 → 新用户分流进引导
3. 昵称确认 → 宠物状态选 `HAS_PET` → 建档（含 `pet_type`）
4. **建档成功庆祝页**（头像+名字 / 分享名片副 CTA / 开始探索主 CTA）
5. 庆祝页后弹**推送权限**（双时机之一）→ 授权
6. 进首页 → 快捷入口「Catat」记录每日 → 发布成长日历内容（带 event_date）
7. 成长档案：信息卡 + 统计栏 + 里程碑进度 + 时间线/日历双视图
8. **首次问诊**：上传图文 → AI 分诊三态（绿/黄/红）
9. （黄/需进一步）→ 升级兽医 → 排队 → 兽医抢单 → 腾讯 IM 图文会话
10. 会话结束 → **评分门** 评分
11. 问诊记录承接进成长档案健康事件时间线
12. 触发**里程碑自动完成** → 三级庆祝动效 → （L 级）推送 + 分享卡
13. 名片 H5 对外分享 → 浏览器打开（含失效 card_gone 态）
14. 通知中心铃铛查看全部通知 + 深链跳转
15. 「我的」→ 语言切换 en↔id → **账号注销**（逐表核对 SC-5）

**主链路 B —— 兽医侧：**
1. ADMIN 后台开兽医户 → 兽医 `/vet/login` 登录
2. 工作台上线 → 收件箱收到新请求推送
3. 请求详情（3 分钟预览超时回队列）→ 抢单（多兽医并发，先到先得）
4. AI 上下文带入（vet_ai_context_card）→ IM 图文回复
5. 结束会话 → 用户评分门 → 问诊历史整合
6. 异常：兽医被封 → 进行中会话中断处理；用户等待期 kill → 自动取消；对话中 kill → 30 分钟保护窗口恢复
