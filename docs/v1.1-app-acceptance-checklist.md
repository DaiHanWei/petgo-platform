# v1.1 App 验收测试清单（真机 · staging）

> 用途：v1.1 在 **App 端可测的功能点 / 边界 / 达标预期**，供真机手动验收。
> 环境：**staging `api-stag.tailtopia.id`**（GemPay sandbox；数据克隆自生产）。包=debug、真实 Google 登录、装于真机。
> 后台治理（Epic 9）在 **web** `ops-stag.tailtopia.id`，不在本 App 清单（见 §9）。
> 价格/额度类：Story 9.2 起**运行时读 DB**（`pricing_config`/`pawcoin_config`），验收对齐**后台当前配置**，不对硬编码数字。

## 0. 全局护栏（贯穿所有功能的边界，任一违反即不达标）

| 护栏 | 达标预期 |
|---|---|
| **红色态零变现** | 宠物红色态（已故/走失等）下，一切收费/充值/解锁入口**关闭或不可点**，绝不产生扣费 |
| **安全规则只升不降** | 触发安全规则的内容/操作只会更严，不会因重试/绕路被放行 |
| **游客只读 + 受控入口门控** | 未登录可浏览只读内容；点收费/发布/我的等受控入口 → 弹登录门控，不越权 |
| **注销级联** | 注销后：UGC 匿名化/隐藏、PawCoin 余额作废、健康/个人数据删除；同账号再登录是新账号（不复活） |
| **双语** | 界面按 `users.locale` 渲染（默认印尼语 id / 英文 en）；App 不渲染后端显示串，按 code 本地化 |
| **不可枚举 token** | 分享/订单/支付对外用 token，URL 不暴露自增 id |

## 1. Epic 1 — PawCoin 钱包与充值（本次重点 · GemPay 已上 staging）

| # | 功能点 | 边界 | 达标预期 |
|---|---|---|---|
| 1a | **充值下单（QRIS/GemPay）**：进充值页 → 选档 → 拿二维码 | 仅 **QRIS** 走外部网关；`PAWCOIN`（站内余额支付）不走 GemPay；金额低于渠道最低额应被前置拦（不发到网关 P13） | 选档后返回可渲染的 QRIS 二维码载荷 + 生成一条 `PENDING` 支付意图；下单幂等（重复点/重放同 Idempotency-Key 不重复建单） |
| 1b | **付款到账**：完成 sandbox mock 付款 → GemPay 回调 | mock 二维码需在浏览器完成付款（sandbox）；回调走 `api-stag/pay/callback`，验签用确认公式 | 回调验签通过 → 意图转 `PAID` → 钱包**到账对应 koin**（按用户支付 `amount`，admin_fee 为平台成本）；回调+轮询**双通道只到账一次**（幂等） |
| 1c | **余额与流水页**：查看 PawCoin 余额 + 流水 | 流水按用户维度、倒序；换账号后不串号 | 余额=双分录总账推导；流水条目与充值/消费一一对应；余额**非负** |
| 1d | **充值失败/取消/暂停态**：付款失败/取消/网关未回 | 未支付余额不变；暂停开关（浮存门槛）由运营手动置 | 失败/取消 → 意图 `FAILED/EXPIRED`、余额不变、页面显对应态；**暂停态**下充值入口显示「暂停」且不可下单 |
| 1e | **状态轮询**：App 查支付状态 | 只读本地库、不主动刷网关（Story 1.5 契约） | 轮询返回当前意图状态；到账后状态变 PAID 并反映到余额 |
| 1f | **注销作废 PawCoin** | 与 §0 注销级联联动 | 注销 → 写 FORFEITURE 终结分录归零 + 物理删钱包/流水 |

> ⚠️ sandbox 完整闭环需在浏览器完成 mock 付款才触发回调；真机上 App 只负责下单 + 展示二维码 + 轮询到账。

## 2. Epic 2 — AI 问诊解锁 / Paywall

| # | 功能点 | 边界 | 达标预期 |
|---|---|---|---|
| 2a | **每月免费额度**：首次 AI 问诊解锁 | 额度运行时读 DB（`monthly_free_quota`，clamp[0,35]，0=全付费）；按月/按账号 | 免费额度内解锁不扣费；额度耗尽后进入付费 |

## 8. 数据 / 环境注意（测试前必读）

- **staging 数据克隆自生产**：用户/帖子/管理员都在，可直接用真实账号登录。
- ⚠️ **OSS 同桶**：**别在 staging 删除克隆来的老帖子/媒体**（会删共享桶真实对象，影响生产）。staging 新上传带 `stag/` 前缀。
- **Redis DB3 隔离**：staging 用逻辑 DB 3，与生产 DB2 隔离；运维**禁 `FLUSHALL`**。
- **GemPay = sandbox**：真实付款用 sandbox mock QR（浏览器完成）；不产生真实扣款。
- **公网入口被 CF 拦脚本**：脚本直连 `api-stag` 会遇 Cloudflare 1010；App 走正常 UA 不受影响。

## 10. 一次冒烟建议顺序（真机，~15 分钟）

1. 真实 Google 登录 → 落成长档案/首页（游客门控先验一遍）。
2. **PawCoin 充值**：选 QRIS 档 → 拿二维码 → 浏览器完成 mock 付款 → 回余额页看到账（Epic 1 核心）。
3. AI 问诊：发起 → 看锁定态 → 解锁 → 显详建（Epic 2）。
4. 兽医问诊：发起 → （兽医账号）接单 → 限时支付 → 会话 → 结束进历史（Epic 3）。
5. 订单中心：看上面几笔是否聚合、各态正确（Epic 5）。
6. 身份证：看 KTP 卡 + 分享面板（Epic 6）。
7. 健康记录 + 新手任务：录一条 → 看里程碑/任务进度（Epic 7）。

> 每步对照对应章节的「达标预期」；任一全局护栏（§0）被破即记为不达标。
