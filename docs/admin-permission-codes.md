# TailTopia 运营后台 · 权限码说明

> 适用页面：**后台账号** → 创建账号 / 改权限 里的「模块权限」勾选项。
> 权威来源：`petgo-backend/.../admin/account/domain/AdminPermissions.java`（PRD 附录 B）。新增权限码以该文件为准。

## 先理解两级权限

后台账号分两种类型：

| 账号类型 | 权限 | 是否需要勾选 |
|---|---|---|
| **SUPER_ADMIN（超级管理员）** | **隐式拥有全部权限** | ❌ 不用勾，勾了也无意义——系统对超管一律放行 |
| **STAFF（普通运营）** | **只有被勾选的权限码** | ✅ 勾哪些就能用哪些，未勾的页面/操作会 403 |

> 所以：账号页上那排勾选框**只对 STAFF 账号生效**。给某人开 STAFF 账号后，按 TA 的岗位勾对应权限即可。

权限码格式统一为 `<模块>.<动作>` 全小写点分（如 `vet.create`），它本身就是后端的鉴权标识。

## 权限码逐项说明

### 🩺 兽医管理（Epic 2）

| 权限码 | 含义 | 勾选后能做什么 |
|---|---|---|
| `vet.view` | 查看兽医 | 查看兽医账号列表与详情 |
| `vet.create` | 开通兽医账号 | 为兽医「开户」、创建兽医登录账号 |
| `vet.ban` | 封禁兽医 | 封禁 / 变更兽医封禁状态（封禁会立即踢下线并中断其进行中会话） |
| `vet.reset_password` | 重置兽医密码 | 为兽医重置登录密码（初始密码线下交付，系统不存明文） |
| `vet.qualify` | 兽医资质管理 | 录入 / 审核 / 续期兽医执业资质证件 |

### 👤 用户账号治理（Epic 3）

| 权限码 | 含义 | 勾选后能做什么 |
|---|---|---|
| `user.view` | 查看用户 | 查看 App 用户列表与详情 |
| `user.deactivate` | 停用用户 | 停用用户账号（即时不可登录、强关进行中问诊） |
| `user.delete` | 删除用户 | 删除用户账号（级联删除 / 匿名化，不可逆） |

### 📝 内容审核（Epic 4）

| 权限码 | 含义 | 勾选后能做什么 |
|---|---|---|
| `content.view_reports` | 查看举报队列 | 查看待处理 / 已下架 / 已驳回的举报工单 |
| `content.takedown` | 下架（处理举报） | 对被举报内容执行下架，或驳回举报 |
| `content.restore` | 恢复内容 | 把已下架内容恢复上架 |
| `content.proactive_takedown` | 主动下架 | 不经举报、主动对全量内容执行下架 |

> 说明：平台**无自动下架**，所有下架 / 驳回均由运营人工决定。

| `consult.view_sessions` | 查看会话 | 查询问诊会话记录 |

### ⚙️ 后台账号与审计（Epic 1）

| 权限码 | 含义 | 勾选后能做什么 |
|---|---|---|
| `admin.create_account` | 管理后台账号 | 创建后台账号、修改账号的模块权限 |
| `admin.deactivate` | 停用后台账号 | 停用 / 启用后台账号 |
| `admin.view_logs` | 查看审计日志 | 查看后台操作审计日志 |

> ⚠️ 这三项是**高危管理权限**（能管账号、改权限、看全量审计）。给 STAFF 勾选前请慎重——通常只留给超管。

## 常见问答

**Q：我勾了权限，对方还是进不去某个页面？**
A：① 确认 TA 是 STAFF（超管本就全权，与勾选无关）；② 权限是登录时加载的，**改完权限后对方要重新登录才生效**。

**Q：想让某人能用所有功能怎么办？**
A：直接把账号类型设为 SUPER_ADMIN，不用逐个勾权限。

**Q：页面上为什么只显示 `vet.create` 这种英文码、没有中文？**
A：当前账号页直接渲染权限码字符串。本文档即对照表；如需在 UI 上显示中文，需后续给每个码补 i18n 文案（可作为一个小改进项）。
